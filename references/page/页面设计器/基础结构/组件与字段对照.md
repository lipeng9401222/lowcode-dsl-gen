# 组件与字段对照

本页是 `page` 资产生成的落盘级补充参考，服务于 `scripts/add_page.py` 和页面设计器 JSON。

它不是移动端专用文档。当前 `lowcode-dsl-gen` 的 page 示例和生成器都以 PC 页面为主，`Sform*` 是运行态组件族，落盘 JSON 通过 `type`、`componentType`、`componentName` 组合表达。

## 1. 字段含义

### 页面顶层字段

| 字段 | 含义 | 说明 |
|------|------|------|
| `schemaVersion` | 协议版本 | 固定为 `core-1.0` |
| `kind` | 资产类型 | 固定为 `page` |
| `id` | 页面稳定标识 | 生成器可显式写入，需在页面内唯一 |
| `pagetag` | 页面业务标识 | workflow / module 引用的关键值，当前生成器会写入 |
| `title` | 页面标题 | 列表、表单、详情页名称 |
| `pageId` | 运行时页面 id | 主要用于表单 / 详情页，列表页通常不需要 |
| `viewport` | 画布配置 | `desktop` / `mobile` / `tablet` / `embedded` |
| `theme` | 页面主题 | 页面级默认视觉变量 |
| `models` | 页面数据模型 | `record` / `collection` |
| `resources` | 外部资源注册区 | endpoint、dictionary、workflow 等 |
| `actions` | 行为注册区 | 顶层 action id |
| `events` | 页面级事件映射 | 值引用顶层 `actions` |
| `children` | 根视图树 | 页面唯一根节点数组 |
| `props` | 页面级附加配置 | 例如表单页 `labelPosition` |

### 节点字段

| 字段 | 含义 | 说明 |
|------|------|------|
| `componentType` | 节点实例类型 | 例如 `input-name`、`form-item-1`、`table-column-3` |
| `componentId` | 节点实例唯一 id | 通常为 `{componentType}_{pageId}` |
| `id` | 节点短 id | 稳定、可读的实例标识 |
| `componentName` | 运行态注册名 | 例如 `ev-input-1100`、`ev-table100` |
| `type` | 运行态组件类型 | 例如 `input`、`table`、`toolbar` |
| `version` | 组件版本 | 当前示例均为 `1.0.0` |
| `isRegister` | 是否注册到运行态 | 当前示例均为 `true` |
| `parentContainerId` | 父容器 id | 指向父节点 `componentId` |
| `slotName` | 具名插槽名 | 如 `title`、`button`、`search`、`actions`、`main` |
| `props` | 组件配置 | 组件语义参数，不放业务动作 |
| `children` | 默认子节点 | 普通树结构子节点 |
| `slots` | 具名子节点 | 例如 toolbar 的 `title/button/search/actions` |
| `events` | 节点事件映射 | 事件值引用顶层 `actions` |

## 2.1 页面组件选型总规则

skill 生成页面时，先判断页面层级结构，再判断字段语义，最后才补齐 props。不要先想“字段类型是什么控件”，再临时拼页面骨架。

### 设计稿到组件库的优先映射规则

当输入包含设计稿、截图、原型图或 PRD 页面图时，组件选择必须从可见 UI 元素出发，而不是从 MIS 字段出发。执行顺序：

1. 抽取设计稿可见元素：标题、按钮、查询区、表格列、分组表头、行操作、分页、示例数据。
2. 在本组件对照和组件文档中找到对应组件。
3. 记录 `设计稿元素 -> 组件库组件 -> page JSON type/componentName -> JSON 落点`。
4. 再决定字段绑定和模型字段是否需要补充。

如果设计稿和 MIS 字段冲突，优先还原设计稿可见结构；MIS 只作为数据绑定来源，不得反向改写页面列结构。

### 列表页默认结构

| 区域 | 默认组件 | 必要结构/props | 说明 |
|------|------|------|------|
| 页面根 | `list-container` | `children=[toolbar, table]` | 当前生成器稳定输出的列表根容器 |
| 查询区 | `toolbar` + `search` slot | `showSearchSlot=true` | 查询控件放工具栏 `search` 插槽，不单独起一块 form-layout |
| 标题区 | `toolbar-title` | `showTitleSlot` 按需开启 | 列表默认可关闭，避免页面标题重复 |
| 主操作区 | `add-button` / `del-button` / `toolbar-more` | `showButtonSlot=true` / `showActionsSlot=true` | 新增、批量删除、更多动作都挂在工具栏 |
| 数据区 | `table` | `modelValue`、`idField`、`pageSize` | 列表主数据承载组件 |
| 普通列 | `table-column` | `field`、`label` | 只读展示列 |
| 可编辑列 | `table-column` + `table-column-editor` | editor 节点 `type` 与字段控件一致 | 只有明确表内编辑时才加 editor |
| 操作列 | `table-action-column` + `table-action` | 操作列固定独立节点 | 查看、编辑、删除不要伪装成普通列 |
| 分组列头 | `table-column-group` | `children` 只能挂列或操作列 | 多级表头必须用分组节点 |

### 设计稿常见元素映射

| 设计稿元素 | 推荐组件 | page JSON 落点 | 备注 |
|---|---|---|---|
| 左上页面标题 | `toolbar-title` 或页面标题容器 | `toolbar.slots.title` / 页面标题区 | 若运行端已有外部标题，可关闭 toolbar title |
| 导出 | `add-button`/普通按钮形态的 `SformButton` | `toolbar.slots.button` | 文案保持“导出”，不要替换为新增 |
| 刷新 | toolbar action / icon button | `toolbar.slots.actions` | 文案或图标按设计稿 |
| 文本搜索框 | `search-control` 或 `input` | `toolbar.slots.search` | 占位文案按需求或设计稿 |
| 人员/组织搜索框 | `search-control` 或对应选择控件 | `toolbar.slots.search` | 数据源不明确时先按文本搜索，并标注待确认 |
| 搜索按钮 | `SformButton` | `toolbar.slots.search` 或 `toolbar.slots.actions` | 不要只生成高级筛选图标 |
| 勾选列 | `SformTable` selection | `table.props.showSelectionColumn` | 设计稿显示 checkbox 时必须开启 |
| 序号列 | table index / sequence column | `table.props.showIndexColumn` 或 `table.children` | 设计稿列名为“序”时需显式表达 |
| 分组表头 | `table-column-group` | `table.children` | 子列继续用 `table-column` 或 `table-action-column` |
| 链接列 | `table-column` 或 `table-action` | 对应表格列或分组子列 | 仅在需求明确为可点击时使用 |
| 行内操作 | `table-action-column` + `table-action` | `table.children` | 操作文案按需求或设计稿 |
| 底部分页 | `SformTable` pagination | `table.props` | 记录 pageSize 和位置 |

### 表单 / 详情 / 办理页默认结构

| 区域 | 默认组件 | 必要结构/props | 说明 |
|------|------|------|------|
| 页面根 | `layout-manager` | `slots.main` / `slots.bottom` | 表单页默认使用区域布局 |
| 主内容容器 | `div` | 包裹 `collapse` 或其他布局容器 | 当前示例使用普通容器承接主体 |
| 分组容器 | `collapse` + `collapse-item` | `children` 中挂 `form-layout` | 默认一个折叠组；字段显式分组时再拆多个 |
| 表单布局 | `form-layout` | `itemSpan`、`gutter` | 默认双列布局 |
| 表单项 | `form-item` | `label`、`span` | 所有表单控件必须挂在表单项内 |
| 底部按钮区 | `toolbar` | `showButtonSlot=true` | 保存、取消、提交等动作放底部工具栏 |
| 工作流动作区 | `workflow-button` / `workflow-history` / `workflow-right` | 仅工作流场景启用 | 普通 CRUD 不默认生成 |

### 选型顺序

1. 先按页面类型选结构容器：`list` 用 `toolbar + table`，`form/detail` 用 `layout-manager + collapse + form-layout`。
2. 再按字段语义选控件：人员、组织、字典、日期、布尔、长文本、附件优先走专用控件，不退化成普通 `input`。
3. 最后补最少 props：占位文案、是否可清空、是否多选、格式、范围、字典来源、交互方式。

## 2.2 字段语义到组件的默认映射

下表是 skill 应优先执行的默认选型矩阵。只有在用户、示例 JSON 或运行态约束明确要求时，才偏离这些默认值。

| 字段/场景特征 | 默认组件 | page JSON `type` | 必带 props | 何时改用别的组件 |
|------|------|------|------|------|
| 普通短文本，如名称、编号、标题 | `SformInput` | `input` | `placeholder`、`maxlength` | 需要多行描述时改 `textarea`；有固定枚举时改 `select` |
| 长文本，如备注、说明、原因 | `SformInput` | `input` | `type='textarea'`、`rows` 或 `autosize`、`maxlength` | 需要富文本排版时改 `editor-wrapper` |
| 数值、金额、比例、数量 | `SformInputNumber` | `input-number` | `min`、`max`、`step`；金额/比例补 `precision` | 只是展示不编辑时仍可作为表格普通列，不必强行带 editor |
| 布尔值，如是否启用、是否有效 | `SformSwitch` | `switch` | `activeValue`、`inactiveValue` | 只有当业务明确要求“是/否枚举”时才改单选或下拉 |
| 单选字典字段，且有代码项/显示字段 | `SformSelect` | `select` | `codeItem` 或资源绑定、`placeholder`、`clearable` | 选项很少且要求平铺时可改 `radio-group` |
| 多选字典字段 | `SformSelect` | `select` | `multiple=true`、`clearable=true`、必要时 `filterable=true` | 有明确树层级时改 `tree-select` 或 `cascader` |
| 日期字段 | `SformDatePicker` | `date-picker` | `type='date'`、`format`、`clearable` | 要时间改 `time-picker`；需要起止区间改 `daterange` |
| 日期区间查询 | `SformDatePicker` | `date-picker` | `type='daterange'`、`startPlaceholder`、`endPlaceholder` | 如果设计只需要单个日期，不要硬生生拆成区间 |
| 时间字段 | `SformTimePicker` | `time-picker` | `placeholder`、必要时时间格式 | 纯日期不要用时间选择器 |
| 人员字段，值就是人 | `SformPersonPicker` 或 `SformOrganizationSelect` | `person-picker` / `organization-select` | 单人/多人、占位文案、可清空 | 需要专门人员选择体验时优先 `person-picker`；当前生成器默认落 `organization-select` 的 people 模式 |
| 部门字段 | `SformOrganizationSelect` | `organization-select` | `mode='dept'`、`range`、`interactive` | 没有组织树语义不要误用 |
| 人员/部门混合选择 | `SformOrganizationSelect` | `organization-select` | `mode`、`multiple`、`range`、`interactive` | 只选人时可用 `person-picker`，不要把组织选择器当默认万金油 |
| 需要树形层级选择的分类 | `SformTreeSelect` | `tree-select` | `placeholder`、树资源绑定、必要时 `multiple` | 只有平铺枚举时改 `select` |
| 父子级联类行政区、品类 | `SformCascader` | `cascader` | 数据源、`placeholder`、必要时 `multiple` | 不存在层级关系时不要用级联 |
| 需要弹窗选取外部业务对象 | `SformButtonEdit` | `button-edit` | `dialogConfigGuid` 或 `url`、`pageName`、`openType` | 仅字典/树选择不要升级成弹窗 |
| 附件字段 | `SformFileUpload` | `file-upload` | 上传资源、是否多文件、大小/数量限制 | 纯图片资料优先 `image-upload` |
| 图片字段 | `SformImageUpload` | `image-upload` | 上传资源、数量限制 | 非图片附件不要误用图片上传 |
| 富文本正文 | `SformEditorWrapper` | `editor-wrapper` | 编辑器配置、占位文案 | 普通备注不要默认上富文本 |
| 只读文本展示 | `SformText` | `text` | 文本来源 | 可编辑场景不要用展示控件代替输入控件 |

## 2.3 当前生成器的默认控件推断规则

`add_page.py` 现在的自动推断并不是“组件全量智能选型”，而是一组保守默认值。skill 编写页面时应先参考这些默认行为，再决定是否手工覆盖：

| 页面模式 | 生成器默认规则 |
|------|------|
| `query` | 数值字段用 `input-number`；日期/时间字段用 `date-picker`；有 `displayField` 的字段用 `select`；其余用 `input` |
| `form` | `personSelect=true` 的字段用 `organization-select`；数值用 `input-number`；日期/时间用 `date-picker`；布尔用 `switch`；`multiple=true` 用 `select`；其余用 `input` |
| `table` | 数值用 `input-number`；日期/时间用 `date-picker`；有 `displayField` 的字段用 `select`；其余用 `input` |

补充约束：

- 如果字段语义明显是“人员”，但生成器只会落 `organization-select`，skill 应在文档和输出里明确这是“people 模式组织选择器”，不是普通下拉。
- 如果字段语义要求 `tree-select`、`cascader`、`button-edit`、`file-upload`、`image-upload` 等专用控件，当前不能只依赖默认推断，必须显式给字段 `widget` 或手工调整布局配置。
- 不要把“示例里没自动生成”理解为“组件不可用”；要区分“运行态组件存在”和“生成器默认不会替你挑它”。

## 3.1 表格分组补充规则

- 多级表头、分组列头必须使用 `table-column-group`，不要用父级 `table-column + children` 代替。
- `table-column-group` 的子节点可以是：
  - `table-column`
  - `table-action-column`
- 如果把分组父节点错误落成 `table-column`，常见现象是 JSON 校验通过，但运行页只渲染一级标题，二级表头不展开。

## 3.2 已稳定支持与需显式确认的边界

### 已稳定支持

- 列表页：工具栏、查询控件、表格、操作列、分组表头。
- 表单页：布局管理器、折叠面板、表单布局、表单项、底部按钮栏。
- 常见字段控件：`input`、`input-number`、`select`、`date-picker`、`switch`、`organization-select`。

### 需显式确认或手工补充

- `person-picker`：运行态组件已存在，但生成器默认更常落成 `organization-select`。
- `tree-select`、`cascader`、`button-edit`：需要明确字段语义或显式 `widget`，不要靠猜。
- `file-upload`、`image-upload`：除控件外还要确认上传资源、限制和回填字段。
- `workflow-button`、`workflow-history`、`workflow-right`：仅工作流页面使用，普通 CRUD 不默认加。
- `tabs`、`tab-pane`、复杂栅格/Flex：组件文档存在，但当前页面示例和默认骨架未大规模覆盖，优先在有明确设计稿时使用。

## 4. 常用 props / events

### 输入类控件

- `input` 常用 `modelValue`、`placeholder`、`maxlength`、`showWordLimit`、`width`、`state`、`type`、`tooltipPosition`、`tooltipContent`
- `input-number` 常用 `min`、`max`、`step`、`precision`、`showControls`、`clearable`
- `select` 常用 `codeItem`、`placeholder`、`clearable`、`multiple`、`filterable`
- `date-picker` 常用 `format`、`type`、`minDate`、`maxDate`、`clearable`
- `time-picker` 常用 `placeholder`、`clearable`、时间格式配置
- `switch` 常用 `activeValue`、`inactiveValue`
- `person-picker` 常用 `placeholder`、`multiple`、`clearable`、`url`、`morePageUrl`
- `organization-select` 常用 `mode`、`range`、`interactive`、`multiple`、`placeholder`
- `button-edit` 常用 `dialogConfigGuid`、`url`、`pageName`、`openType`
- `file-upload` / `image-upload` 常用上传资源、文件数量限制、回填字段约束

### 布局类控件

- `toolbar` 常用 `showTitleSlot`、`showButtonSlot`、`showSearchSlot`、`showActionsSlot`、`position`
- `table` 常用 `modelValue`、`idField`、`tableType`、`pageSize`、`showSelectionColumn`、`showIndexColumn`
- `form-layout` 常用 `gutter`、`itemSpan`
- `form-item` 常用 `label`、`labelWidth`、`labelPosition`、`span`
- `collapse` 常用 `accordion`、`showNav`、`showIndex`、`navAffix`
- `layout-manager` 常用 `slots.top/left/right/bottom/main` 区域分配

## 4.1 组件最少 props 基线

skill 生成 JSON 时，不要只写 `type`，以下 props 至少要补齐到可用状态：

| 组件类型 | 最少 props 基线 |
|------|------|
| `input` | `placeholder`；长文本再补 `type='textarea'` + `rows/autosize` |
| `input-number` | `min`、`max`、`step`；金额/比例再补 `precision` |
| `select` | `placeholder`、`clearable`；枚举来源用 `codeItem` 或资源绑定；多选补 `multiple=true` |
| `date-picker` | `type`、`format`、`placeholder`；区间再补 `startPlaceholder/endPlaceholder` |
| `switch` | `activeValue`、`inactiveValue` |
| `organization-select` | `mode`、`range`、`interactive`、`placeholder`；多人补 `multiple=true` |
| `person-picker` | `placeholder`、`multiple`、`clearable`；必要时补 `url/morePageUrl` |
| `button-edit` | `dialogConfigGuid` 或 `url`，以及 `pageName`、`openType` |
| `table` | `modelValue`、`idField`、`pageSize`；表内编辑再补 `tableType` |
| `toolbar` | 根据插槽内容显式设置 `showTitleSlot/showButtonSlot/showSearchSlot/showActionsSlot` |
| `form-layout` | `itemSpan`、`gutter` |
| `form-item` | `label`；跨列字段补 `span` |

### 事件约定

- 页面级事件通常是 `load`
- 列表页常见节点事件是 `click`、`pageChange`、`sortChange`、`selectionChange`
- 表单页常见节点事件是 `click`、`change`、`submit`、`reset`
- 节点事件值必须引用顶层 `actions`，不要内联动作步骤

## 5. 结论

`组件文档.md` 可以直接作为 `props/events` 参考，但要真正用于 page 生成，还必须结合本页的字段含义和组件映射。

如果要补全 skill，优先按这三件事维护：

1. 组件名和 page JSON `type/componentType` 对照。
2. 页面顶层字段和节点字段字典。
3. 字段语义到组件的默认选型矩阵。
4. 每类组件的最少 props 基线。
5. 生成器当前覆盖的结构节点清单和边界。
