# 列表页场景示例

> 适用于“列表、台账、查询、管理页”。本文件说明列表页生成规则；完整组件结构参考 [`../../设计器 Schema 规范定义/列表示例.md`](../../设计器%20Schema%20规范定义/列表示例.md)。

## 使用脚本一键生成

```bash
python3 scripts/add_page.py \
  --app-root /path/to/<apptag> \
  --type list \
  --title 项目列表 \
  --pagetag projectinfo_list \
  --model projectinfo \
  --sql-table projectinfo \
  --form-page-id page_1781580614416 \
  --fields-json '[{"name":"projectname","label":"项目名称","type":"string"}]' \
  --query-json '[{"name":"projectname","label":"项目名称","type":"string"}]'
```

## 生成前提

- 列表页使用 `models[].type=collection`。
- `models[].source` 优先使用 `mis`，并填写真实 `sqlTableName`。
- 查询字段来自用户确认或 MIS 字段，不虚构查询条件。
- 当列表包含新增、修改、查看按钮时，必须先确定目标表单页 JSON 的顶层 `pageId`。

## pageId 引用规则

列表页打开表单/详情弹窗时，URL 中的 `pageId` 必须取自目标表单页 JSON 顶层 `pageId` 字段。真实设计器值通常是 `page_<时间戳>`：

```json
{
  "pageId": "page_1781580614416",
  "id": "projectinfo-form-page",
  "pagetag": "projectinfo_form",
  "kind": "page",
  "title": "项目信息表单"
}
```

列表页按钮引用同一个值：

```json
{
  "componentType": "add-button",
  "props": {
    "buttonText": "新增记录",
    "url": "vuepagedesigner/renderer/add?pageId=page_1781580614416",
    "openType": "dialog"
  }
}
```

```json
{
  "componentType": "table-action",
  "props": {
    "actionType": "detail",
    "label": "查看",
    "url": "vuepagedesigner/renderer/detail?pageId=page_1781580614416",
    "openType": "dialog"
  }
}
```

列表页本身不需要顶层 `pageId`。列表页顶层 `id` 使用稳定 schema id，例如 `projectinfo-list-page`；不要把列表页自己的 `id` 当成表单弹窗目标。

## 推荐结构

```json
{
  "pagetag": "projectinfo_list",
  "schemaVersion": "core-1.0",
  "kind": "page",
  "id": "projectinfo-list-page",
  "title": "项目列表",
  "models": [
    {
      "alias": "projectinfo",
      "source": "mis",
      "type": "collection",
      "primaryKey": "rowguid",
      "sqlTableName": "projectinfo",
      "fields": {}
    }
  ],
  "resources": {},
  "actions": {},
  "events": {},
  "children": []
}
```

## 组件要点

- 根组件使用 `list-container`。
- 顶部工具栏使用 `toolbar`，默认 `showTitleSlot=false`，避免列表左上角重复显示页面标题。
- 新增按钮使用 `add-button`，`buttonText=新增记录`，`openType=dialog`。
- 批量删除按钮使用 `del-button`，`buttonText=删除选中`。
- 表格使用 `table`，数据模型绑定 `collection` 模型。
- 表格 `props.modelValue` 使用 `{ "$expr": "$model.<modelAlias>" }`，表格列 `props.modelValue` 使用 `{ "$expr": "$model.<modelAlias>.<fieldName>" }`。
- 表格最后一列追加 `table-action-column`，包含修改、查看、删除。
- 修改和查看都打开目标表单页；删除留在当前列表，URL 为空并使用删除确认。

## 常见按钮 URL

| 场景 | URL |
|------|-----|
| 新增 | `vuepagedesigner/renderer/add?pageId=<form-page-id>` |
| 修改 | `vuepagedesigner/renderer/add?pageId=<form-page-id>` |
| 查看 | `vuepagedesigner/renderer/detail?pageId=<form-page-id>` |
| 删除 | 空字符串，使用当前行删除动作 |

其中 `<form-page-id>` 必须等于目标表单 JSON 的顶层 `pageId`。

## 禁止事项

- 不生成 `<apptag>/pagedesigne/` 或老 metadata 目录。
- 不主动生成 event 来承载标准新增、修改、查看、删除。
- 不在没有接口依据时虚构 `endpoint`、`resources` 或 `actions`。
- 列表页不写顶层 `pageId`；按钮 URL 中的 `pageId` 必须使用目标表单页真实 `page_<时间戳>`。
