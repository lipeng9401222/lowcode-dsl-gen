# 工作流列表页场景示例

> 适用于“工作流列表、流程发起入口、审批业务台账”。普通列表完整组件结构参考 [`../../设计器 Schema 规范定义/列表示例.md`](../../设计器%20Schema%20规范定义/列表示例.md)，工作流列表结构差异参考 [`../../设计器 Schema 规范定义/工作流列表结构.md`](../../设计器%20Schema%20规范定义/工作流列表结构.md)。

## 生成前提

- 已存在或计划生成目标工作流表单页，并能取得真实顶层 `pageId`。
- 已存在或计划生成 workflow 资产，并能取得真实 `processGuid`。
- 列表页使用 `models[].type=collection`。
- 列表字段、查询条件、操作列仍按普通列表规则生成。

## 使用脚本一键生成

当前 `add_page.py` 若只支持普通列表骨架，应先生成普通列表，再按本场景规则把新增、修改、查看入口调整为工作流入口。

```bash
python3 scripts/add_page.py \
  --app-root /path/to/<apptag> \
  --type list \
  --title 立项管理列表 \
  --pagetag projectinfo_list \
  --model projectinfo \
  --sql-table projectinfo \
  --form-page-id page_1780888036004 \
  --fields-json '[{"name":"projectname","label":"项目名称","type":"string"}]' \
  --query-json '[{"name":"projectname","label":"项目名称","type":"string"}]'
```

生成后必须把打开表单的 URL 补为：

```text
vuepagedesigner/renderer/add?pageId=<formPageId>&processGuid=<processGuid>
vuepagedesigner/renderer/detail?pageId=<formPageId>&processGuid=<processGuid>
```

## 组件要点

- 根组件仍使用 `list-container`。
- 新增按钮位于 `toolbar.slots.button`。
- 工作流新增入口使用 `type: "process-button"`，不是普通 `add-button`。
- 修改、查看仍使用 `table-action`，但 URL 必须带 `processGuid`。
- 删除仍留在当前列表页执行，`url` 为空，不追加 `processGuid`。
- 工作流示例中 `openType` 使用 `modal`；若项目已有统一弹窗规范，以真实现有页面为准。

## 常见按钮 URL

| 场景 | URL |
|------|-----|
| 新增流程 | `vuepagedesigner/renderer/add?pageId=<formPageId>&processGuid=<processGuid>` |
| 修改流程表单 | `vuepagedesigner/renderer/add?pageId=<formPageId>&processGuid=<processGuid>` |
| 查看流程表单 | `vuepagedesigner/renderer/detail?pageId=<formPageId>&processGuid=<processGuid>` |
| 删除 | 空字符串，使用当前行删除动作 |

## 与普通列表的判断

- 用户只说“列表、查询、管理页”：默认普通列表。
- 用户说“工作流列表、发起流程、流程台账、审批列表”：使用工作流列表入口。
- 用户只要求列表展示流程状态，但不从列表发起流程：可以保留普通列表，只展示状态字段。
- 用户要求从列表新增时进入流程：必须使用 `process-button` 和 `processGuid`。

## 禁止事项

- 不手工编造 `processGuid`。
- 不把列表页自己的 `id` 当成目标表单 `pageId`。
- 不主动生成 event 来承载标准新增、修改、查看、删除。
- 不复制参考示例里的业务字段、组件 id、页面 id 或流程 id。
