# 表单页场景示例

> 适用于“新增、编辑、详情、申请、办理页”。本文件说明表单页生成规则；完整组件结构参考 [`../../设计器 Schema 规范定义/表单示例.md`](../../设计器%20Schema%20规范定义/表单示例.md)。

- 如果用户提供了设计稿、截图或明确布局要求，优先按用户要求生成。
- 只有在没有明确说明时，才使用默认规则：`单个折叠区 + 默认双列`。

## 使用脚本一键生成

```bash
python3 scripts/add_page.py \
  --app-root /path/to/<apptag> \
  --type form \
  --title 项目信息表单 \
  --pagetag projectinfo_form \
  --page-id page_1781580614416 \
  --model projectinfo \
  --sql-table projectinfo \
  --fields-json '[{"name":"projectname","label":"项目名称","type":"string","required":true}]'
```

## 生成前提

- 表单页使用 `models[].type=record`。
- `models[].source` 优先使用 `mis`，并填写真实 `sqlTableName`。
- 字段、必填、只读、隐藏、默认值必须有来源记录。
- 表单页顶层必须生成真实设计器格式的 `pageId`，供列表页新增/修改/查看按钮引用。
- 默认只生成一个 `collapse-item` 承载整页表单，除非字段显式配置 `group`，否则不要拆成多个折叠区。

## pageId 生成规则

表单页的 `pageId` 是运行时打开弹窗的目标 id，真实设计器值通常是 `page_<时间戳>`；它不要求和页面顶层 `id` 一致：

```json
{
  "pageId": "page_1781580614416",
  "id": "projectinfo-form-page",
  "pagetag": "projectinfo_form",
  "kind": "page",
  "title": "项目信息表单"
}
```

推荐取值规则：

| 页面类型 | pagetag 示例 | pageId 示例 |
|----------|--------------|-------------|
| 列表页 | `projectinfo_list` | `page_1781592006000` |
| 表单页 | `projectinfo_form` | `page_1781580614416` |
| 详情页 | `projectinfo_detail` | `page_1781592007000` |

生成列表页时，新增、修改、查看 URL 中的 `pageId` 必须读取目标表单 JSON 的 `pageId`，例如：

```json
{
  "url": "vuepagedesigner/renderer/add?pageId=page_1781580614416"
}
```

落盘时必须使用真实 `page_<时间戳>`。

## 推荐结构

```json
{
  "pagetag": "projectinfo_form",
  "schemaVersion": "core-1.0",
  "kind": "page",
  "pageId": "page_1781580614416",
  "id": "projectinfo-form-page",
  "title": "项目信息表单",
  "models": [
    {
      "alias": "projectinfo",
      "source": "mis",
      "type": "record",
      "primaryKey": "rowguid",
      "sqlTableName": "projectinfo",
      "fields": {}
    }
  ],
  "resources": {},
  "actions": {},
  "events": {},
  "children": [],
  "props": {
    "labelPosition": "right"
  }
}
```

## 组件要点

- 根布局使用 `layout-manager-1`，底部工具栏承载保存/取消等按钮。
- 表单内容放在 `sform-div-1`、`collapse`、`collapse-item`、`form-layout` 内。
- `form-layout` 默认使用 `itemSpan=12` 形成左右双列，长文本字段单独占满一行时再设为 `span=24`。
- 每个字段使用 `form-item` 包裹真实输入控件。
- 输入控件的 `modelValue` 必须用表达式单元绑定到 `record` 模型字段，例如 `{ "$expr": "$model.projectinfo.projectname" }`。
- 长文本字段可使用 textarea；普通文本使用 input；数字、日期、下拉等按字段类型选择对应控件。
- 页面级默认 `props.labelPosition` 使用 `right`，避免标签上置导致与示例不一致。
- 详情页或只读浏览页复用表单结构，但控件应只读，并且不生成保存按钮。

## 工作流页面

- 工作流申请/审批页仍使用 `pagetag` 被 workflow 引用。
- 列表页弹窗使用 `pageId` 打开表单页。
- 同一页面可以同时有 `pagetag` 和 `pageId`，两者用途不同：`pagetag` 面向 workflow/module，`pageId` 面向页面设计器弹窗 URL。

## 禁止事项

- `pageId` 必须使用真实 `page_<时间戳>`。
- 不在表单页里生成列表页表格或批量删除按钮。
- 不主动生成 event 来承载标准保存、取消、查看。
- 没有接口依据时，只生成页面结构，不虚构保存接口。
