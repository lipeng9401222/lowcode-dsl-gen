# 规则与表达式

规则用于表达字段的必填、提交、脱敏、校验等控制逻辑。表达式是受限字符串，不允许任意 JavaScript 执行。

## 规则声明位置

字段规则声明入口为 `models[*].fields.<field>.rules`，其中模型通过 `alias` 被节点和动作引用。字段提交、校验和脱敏规则只出现在字段定义内。

节点显隐使用 `ViewNode.visible`。它是视图级规则，只控制节点是否渲染，不替代字段级 `required`、`submit`、`mask` 或 `validate`。

跨字段、跨上下文逻辑写在字段规则表达式中，或拆分到 action 的 `when` 条件中。节点状态类逻辑可通过 `state(path)` 读取运行时注册的状态。

## 字段级 rules

```json
{
  "models": [
    {
      "alias": "employee",
      "type": "record",
      "fields": {
        "salary": {
          "type": "number",
          "label": "薪资",
          "rules": {
            "required": "includes(user.roles, 'hr')",
            "submit": "includes(user.roles, 'hr')",
            "mask": "!includes(user.roles, 'hr')",
            "validate": [
              { "type": "number", "options": { "min": 0 }, "message": "不能为负数", "when": "notEmpty(model.employee.salary)" }
            ]
          }
        }
      }
    }
  ]
}
```

支持字段：

- `required`：是否必填。
- `submit`：是否进入提交 payload。
- `mask`：是否脱敏展示。
- `validate`：校验规则数组，每项 `{ type?, expression?, validator?, message?, when?, options? }`。`type` 表示预设校验规则类型，`expression` 表示表达式校验规则，`validator` 引用顶层 `actions` 中的自定义校验动作，`options` 承载规则配置，`when` 控制单条校验规则是否生效。

## validate.when 示例

`validate[].when` 只控制当前这一条校验规则是否执行；省略时默认执行。复杂条件建议拆成多条规则，让每条规则只表达一个边界或一个业务约束。

### 有值时才校验格式

邮箱不是必填，但只要填写就必须符合邮箱格式：

```json
{
  "type": "string",
  "label": "邮箱",
  "rules": {
    "validate": [
      { "type": "email", "message": "请输入正确的邮箱", "when": "notEmpty(model.employee.email)" }
    ]
  }
}
```

### 动态数字范围

默认数字范围是 `0-200`。当 `model.employee.useStrictMin` 为 `true` 时，下限改为 `20`；当 `model.employee.useStrictMax` 为 `true` 时，上限改为 `50`。两个条件可以同时生效，同时生效时范围自然变成 `20-50`。

```json
{
  "type": "number",
  "label": "评分",
  "rules": {
    "required": true,
    "validate": [
      { "type": "number", "options": { "min": 0 }, "message": "评分不能小于 0", "when": "model.employee.useStrictMin != true" },
      { "type": "number", "options": { "min": 20 }, "message": "严格下限开启时，评分不能小于 20", "when": "model.employee.useStrictMin == true" },
      { "type": "number", "options": { "max": 200 }, "message": "评分不能大于 200", "when": "model.employee.useStrictMax != true" },
      { "type": "number", "options": { "max": 50 }, "message": "严格上限开启时，评分不能大于 50", "when": "model.employee.useStrictMax == true" }
    ]
  }
}
```

### 跨字段日期范围

结束日期不是独立判断，只有开始日期和结束日期都存在时才校验结束日期不得早于开始日期：

```json
{
  "type": "date",
  "label": "结束日期",
  "rules": {
    "validate": [
      {
        "expression": "model.employee.endDate >= model.employee.startDate",
        "message": "结束日期不能早于开始日期",
        "when": "notEmpty(model.employee.startDate) && notEmpty(model.employee.endDate)"
      }
    ]
  }
}
```

### 自定义校验动作

复杂校验可以引用顶层 `actions` 中注册的动作，由运行态执行动作并返回校验结果：

```json
{
  "type": "string",
  "label": "员工编号",
  "rules": {
    "validate": [
      {
        "validator": "checkEmployeeNo",
        "message": "员工编号不可用",
        "when": "notEmpty(model.employee.employeeNo)"
      }
    ]
  }
}
```

写法建议：`required` 负责字段是否必填，`validate[].when` 负责单条校验何时启用。范围、格式、跨字段关系优先拆成多条校验规则，不把所有逻辑塞进一条表达式。

## 表达式

表达式可出现在：

- 字段规则：`models[*].fields.*.rules.required`、`models[*].fields.*.rules.submit`、`models[*].fields.*.rules.mask`、`models[*].fields.*.rules.validate[].when`、`models[*].fields.*.rules.validate[].expression`。
- 视图规则：`ViewNode.visible`。
- 动作：`steps[].when`、`assign` 右侧、`params`、`body`。

允许引用：

- `model.*`：业务数据模型。
- `params.*`：当前 action/页面入参。
- `response.*`：上一步响应。
- `user.*`：当前用户上下文。
- `env.*`：运行环境信息。
- `meta.*`：页面元信息。

允许运算符：

```text
== != > >= < <= && || ! + - * / %
```

允许函数：

- `empty()`
- `notEmpty()`
- `includes()`
- `len()`
- `sum()`
- `concat()`
- `today()`
- `now()`
- `state(path)`：读取控件或页面运行时状态。

### state(path)

`state(path)` 用于读取不属于业务模型的运行时状态，比如渲染器注册的节点状态，或页面级的 `loading`、当前 tab 等。

`path` 形如：

- `state('<nodeId>.<key>')`：读取节点运行时状态，`key` 为渲染器注册的运行时键。
- `state('page.<key>')`：读取页面运行时状态。

示例：

```json
{
  "when": "state('employeeForm.ready') == true && state('page.loading') == false"
}
```

`state` 只读，不能出现在 `assign` 左侧；对未注册的 path 返回 `undefined`。

禁止能力：

- 赋值。
- 循环。
- 对象构造。
- 原型访问。
- 网络请求。
- 动态函数调用。
- 任意 JavaScript 执行。
