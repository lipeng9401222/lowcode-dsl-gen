# 工作流列表结构

本文件维护工作流列表页的页面设计器 JSON 结构规律。工作流列表页本质上仍是普通列表页，差异集中在打开表单的按钮和 URL：新增、修改、查看入口需要携带目标工作流表单 `pageId` 和流程 `processGuid`。

生成新页面时只参考结构规律，不复制示例中的业务字段、标题、`pageId`、`processGuid`、模型名或节点 id。

## 顶层结构

工作流列表页使用 `collection` 模型，根节点仍为 `list-container`。

```json
{
  "models": [
    {
      "alias": "projectinfo",
      "source": "mis",
      "type": "collection",
      "fields": {},
      "sqlTableName": "projectinfo"
    }
  ],
  "schemaVersion": "core-1.0",
  "children": [
    {
      "componentType": "list-container",
      "children": [
        {
          "componentType": "toolbar",
          "slots": {
            "button": [],
            "search": [],
            "title": [],
            "actions": []
          },
          "type": "toolbar"
        },
        {
          "componentType": "table",
          "children": [],
          "type": "table"
        }
      ],
      "type": "list-container"
    }
  ],
  "kind": "page",
  "title": "工作流列表",
  "actions": {}
}
```

## 结构层级

```text
page
+-- list-container
    +-- toolbar
    |   +-- slots.button
    |   |   +-- process-button
    |   |   +-- del-button
    |   +-- slots.search
    |   +-- slots.title
    |   +-- slots.actions
    +-- table
        +-- table-column
        +-- table-action-column
            +-- edit
            +-- detail
            +-- delete
```

## 新增入口

工作流列表的新增按钮在真实导出中仍放在 `toolbar.slots.button`，但 `type` 使用 `process-button`，URL 比普通新增多 `processGuid`。

```json
{
  "slotName": "button",
  "componentType": "add-button",
  "id": "add-button",
  "componentName": "sform-add-button100",
  "type": "process-button",
  "props": {
    "buttonText": "新增记录",
    "carryParentPageParam": false,
    "plain": false,
    "dialogHeight": "400px",
    "dialogTitle": "新增记录",
    "dialogWidth": "1000px",
    "state": "default",
    "text": false,
    "type": "primary",
    "params": [],
    "url": "vuepagedesigner/renderer/add?pageId=<formPageId>&processGuid=<processGuid>",
    "openType": "modal"
  }
}
```

## 行操作入口

修改、查看操作仍使用 `table-action`，区别是 URL 带 `processGuid`，并打开同一个目标工作流表单 `pageId`。

```json
[
  {
    "id": "table-action-1",
    "componentName": "ev-table-action-1100",
    "type": "table-action",
    "props": {
      "actionType": "edit",
      "icon": "Edit",
      "label": "修改",
      "url": "vuepagedesigner/renderer/add?pageId=<formPageId>&processGuid=<processGuid>",
      "openType": "modal"
    }
  },
  {
    "id": "table-action-2",
    "componentName": "ev-table-action-2100",
    "type": "table-action",
    "props": {
      "actionType": "detail",
      "icon": "Search",
      "label": "查看",
      "url": "vuepagedesigner/renderer/detail?pageId=<formPageId>&processGuid=<processGuid>",
      "openType": "modal"
    }
  },
  {
    "id": "table-action-3",
    "componentName": "ev-table-action-3100",
    "type": "table-action",
    "props": {
      "actionType": "delete",
      "icon": "Delete",
      "label": "删除",
      "url": "",
      "openType": "dialog"
    }
  }
]
```

## 与普通列表的差异

| 位置 | 工作流列表 | 普通列表 |
|------|------------|----------|
| 新增按钮 `type` | `process-button` | `add-button` |
| 新增 URL | `renderer/add?pageId=<formPageId>&processGuid=<processGuid>` | `renderer/add?pageId=<formPageId>` |
| 修改 URL | `renderer/add?pageId=<formPageId>&processGuid=<processGuid>` | `renderer/add?pageId=<formPageId>` |
| 查看 URL | `renderer/detail?pageId=<formPageId>&processGuid=<processGuid>` | `renderer/detail?pageId=<formPageId>` |
| 打开方式 | 真实工作流示例使用 `modal` | 普通示例常用 `dialog` |
| 删除动作 | 不带流程 URL | 不带流程 URL |

## 约束

- 列表页本身不需要顶层 `pageId`。
- `<formPageId>` 必须来自目标工作流表单页真实顶层 `pageId`。
- `<processGuid>` 必须来自 workflow 资产或已有流程配置，不能凭空编造。
- 标准删除不需要携带 `processGuid`。
- 查询区、表格列、选择列、分页等普通列表能力仍按 `列表示例.md` 生成。
