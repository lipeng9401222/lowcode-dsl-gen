# 数据模型

`models` 声明页面数据。模型用于表单绑定、列表绑定、查询条件、规则表达式和 action 参数。

`models` 声明两类需要后端请求的业务数据模型：

- `record`：单条业务数据（表单、查询条件、详情对象）。
- `collection`：多条业务数据（列表、子表、重复组）。

UI 临时状态（loading、当前 tab、选中行等）由组件内部状态或运行时存储承担，不属于业务数据模型。静态展示数据直接写在节点 `props` 中。

## 示例

```json
{
  "models": [
    {
      "alias": "search",
      "type": "record",
      "fields": {
        "keyword": {
          "type": "string",
          "label": "关键词"
        }
      }
    },
    {
      "alias": "employee",
      "modelGuid": "xxx-xxx",
      "source": "mis",
      "sqlTableName": "hr_employee",
      "type": "record",
      "primaryKey": "id",
      "fields": {
        "id": { "type": "string", "label": "ID" },
        "name": {
          "type": "string",
          "label": "姓名",
          "rules": { "required": true }
        },
        "deptId": {
          "type": "string",
          "label": "部门"
        }
      },
      "initial": {
        "name": "",
        "deptId": null
      }
    },
    {
      "alias": "employeeList",
      "type": "collection",
      "primaryKey": "id",
      "fields": {
        "id": { "type": "string", "label": "ID" },
        "name": { "type": "string", "label": "姓名" },
        "deptId": { "type": "string", "label": "部门" }
      }
    }
  ]
}
```

## 模型类型

`record` 与 `collection` 结构完全对齐，仅 `type` 不同：

```ts
interface BaseModel {
  /** 模型唯一标识，用于向服务端请求数据 */
  modelGuid?: string;
  /** 数据来源标识，例如："mis", "interface" */
  source?: string;
  /** 对应后端表名 */
  sqlTableName?: string;
  /** 模型别名，用于在表达式中引用，如 `modelStore.useDataModel('<alias>.a.b')` */
  alias?: string;
  primaryKey?: string;
  fields: Record<string, FieldDefinition>;
  /** 子模型，支持嵌套 */
  children?: Array<RecordModel | CollectionModel>;
}

interface RecordModel extends BaseModel {
  type: 'record';
  initial?: Record<string, JsonValue>;
}

interface CollectionModel extends BaseModel {
  type: 'collection';
  parentSqlTableName?: string;
  initial?: Array<Record<string, JsonValue>>;
}

type ModelDefinition = RecordModel | CollectionModel;
```

### 字段说明

- `modelGuid`：模型唯一标识，向服务端请求数据时使用
- `source`：数据来源标识，如 "mis"、"interface"
- `sqlTableName`：对应后端表名
- `alias`：模型别名，用于在表达式中引用，如 `modelStore.useDataModel('employee.name')`
- `children`：子模型数组，支持嵌套模型定义

### record vs collection

- `record`：运行时值为对象。`initial` 为单条字段默认值对象。
- `collection`：运行时值为数组（一般由 action 写入，初始值缺省为空数组）。`initial` 为多条记录初始值数组，每项为一条记录的字段值对象。

## 字段类型

- `string`
- `number`
- `boolean`
- `date`
- `object`
- `array`

## 字段元信息

- `type`：基础类型。
- `label`：字段显示名。
- `rules`：字段级规则统一入口。

资源来源不写在字段定义上。下拉、树选择、上传等组件需要资源时，通过视图节点的 `source` 引用顶层 `resources`。

字段级 `rules` 支持：

```json
{
  "rules": {
    "required": true,
    "readonly": false,
    "submit": true,
    "mask": false,
    "validate": [
      { "type": "email", "message": "请输入正确的邮箱", "when": "notEmpty(model.employee.email)" }
    ]
  }
}
```

- `required`：是否必填，布尔值或表达式。
- `readonly`：是否只读，布尔值或表达式。
- `submit`：是否进入提交 payload。
- `mask`：是否脱敏展示。
- `validate`：校验规则数组，每项可包含 `type`、`expression`、`validator`、`message`、`when` 和 `options`。`type` 为预设校验规则类型；`expression` 为表达式校验规则；`validator` 引用顶层 `actions` 中的自定义校验动作；`options` 为规则配置对象；`when` 为表达式，结果为 `true` 时该单条校验规则生效，省略时默认生效。

节点显隐使用 `ViewNode.visible`。跨上下文临时控制使用字段规则表达式或 action 的 `when` 条件表达；节点运行时状态可通过 `state(path)` 只读引用。
