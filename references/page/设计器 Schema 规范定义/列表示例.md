# 列表示例

本文件维护列表页真实设计器 Core Schema 示例。后续新增列表页示例时，按“示例三、示例四...”继续追加，并优先放真实设计器导出的完整 JSON。

## 示例一：项目列表

- 页面类型：列表页
- 模型类型：`collection`
- 模型别名：`projectinfo`
- 适合参考：
  - `list-container -> toolbar + table` 的列表页基础骨架
  - 工具栏按钮、查询区、标题区和更多操作区的 slot 组织方式
  - 产品/项目名称、立项人、立项时间、状态等常见业务列
  - `table-column`、`table-action-column` 和行内修改/查看/删除动作
  - 列表按钮引用表单页 `pageId` 的写法

生成新页面时只参考结构规律，不复制项目列表里的业务字段、标题、模型名、页面 id 或节点 id。

## 示例二：任务列表

- 页面类型：列表页
- 模型类型：`collection`
- 模型别名：`taskinfo`
- 适合参考：
  - 无查询区或弱查询区的列表工具栏结构
  - 任务名称、负责人、预计开始/结束时间、项目名称、任务描述、工时等字段列
  - 日期列、数值列、人员显示字段的绑定方式
  - `showSelectionColumn`、`showIndexColumn`、`pageSize`、`pageSizeType` 等表格分页与选择列配置
  - 表格最后操作列中修改、查看、删除的标准组织方式

生成新页面时只参考结构规律，不复制任务列表里的业务字段、标题、模型名、页面 id 或节点 id。

## 使用规则

1. 列表页优先参考本文件的真实 JSON 示例，再结合 `../页面设计器/基础结构/组件与字段对照.md` 做组件选型。
2. 表格数据模型使用 `collection`，表格 `props.modelValue` 绑定到模型根，例如 `{ "$expr": "$model.projectinfo" }`。
3. 查询控件放在 `toolbar.slots.search`，按钮放在 `toolbar.slots.button`，刷新/更多操作放在 `toolbar.slots.actions`。
4. 表格列使用 `table-column`，操作列使用 `table-action-column` + `table-action`，多级表头使用 `table-column-group`。
5. 有新增、修改、查看弹窗时，按钮 URL 中的 `pageId` 必须来自目标表单 JSON 的真实顶层 `pageId`。
6. 没有真实接口依据时，不要虚构 `resources`、`actions` 或页面 `load` 事件。
