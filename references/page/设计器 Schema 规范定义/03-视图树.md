# 视图树

`children` 是 Core Schema 的通用页面视图树入口。`ViewNode` 只保留跨运行态通用的结构、绑定、资源、样式和事件字段；PC/mobile 的具体组件节点可以继承 `ViewNode` 后扩展自己的根属性。

## 节点结构

```json
{
  "id": "employeeName",
  "name": "姓名输入框",
  "version": "1.0.0",
  "type": "input",
  "source": "deptOptions",
  "props": {
    "label": "姓名",
    "modelValue": { "$expr": "$model.employee.name" }
  },
  "style": {},
  "events": {},
  "visible": { "$expr": "$model.employee.enabled == true" },
  "children": []
}
```

## 字段规则

- `id`：节点稳定标识。省略时可由 normalizer 生成。
- `name`：控件中文名，可选。用于识别不同控件。
- `version`：节点版本号，可选。用于标记节点协议或物料版本。
- `type`：运行态组件类型标识，Core Schema 不枚举具体 PC/mobile 组件集合。
- `source`：资源 id，只引用顶层 `resources`。
- `props`：组件语义配置，不放渲染器私有运行时状态。属性值类型为 `SchemaValue = JsonValue | ExpressionCell`，因此可以写普通 JSON 字面量，也可以写 `{ "$expr": "..." }` 由运行时计算。展示文本、标题、间距等简单配置可以放在这里；强结构、强父子依赖的配置由具体组件扩展根属性表达。节点主数据绑定通过 `props.modelValue`（可选）表达，通常写为 `{ "$expr": "$model.employee.name" }`；下拉、树选择等存在 text/value 对的控件可通过 `props.text`（可选）保存展示文本。
- `style`：作者显式指定的样式。
- `events`：事件到顶层 action 的引用。
- `visible`：视图级显隐规则，类型为 `RuleValue = boolean | ExpressionCell`。布尔字面量直接写 `true` / `false`；需要动态逻辑时用 `{ "$expr": "..." }` 包装。省略时默认可见。
- `children`：直接子节点，语义上相当于 default 插槽。
- `slots`：具名插槽，键为插槽名，值为该插槽下的子节点数组。与 `children` 互斥的组件应在扩展类型中使用 `children?: never` 禁用。

`slots` 示例：

```json
{
  "type": "card",
  "props": { "title": "员工信息" },
  "slots": {
    "header": [
      { "type": "text", "props": { "content": "卡片头部" } }
    ],
    "default": [
      { "type": "input", "props": { "modelValue": { "$expr": "$model.employee.name" } } }
    ]
  }
}
```

## 组件扩展

Core Schema 不把 `label`、`title`、`columns`、`items`、`gap` 等控件专用字段放在基础 `ViewNode` 上。简单展示配置可以通过 `props` 约定：

```json
{
  "type": "text",
  "props": { "content": "今日待办" }
}
```

```json
{
  "type": "button",
  "props": { "label": "保存" },
  "events": { "click": "saveEmployee" }
}
```

强结构字段由具体组件类型继承 `ViewNode` 后声明为根属性。`columns` 属于表格结构的一部分，只在 table 上有完整语义，不作为普通 `children`，也不降级到 `props.columns`：

```ts
type TableColumn =
  | { type: 'field'; field: string; label?: string }
  | { type: 'actions'; label?: string; items: Array<{ label: string }> };

interface TableNode extends ViewNode {
  type: 'table';
  children?: never;
  columns: TableColumn[];
}
```

## 文本绑定（text）

`props.text` 与 `props.modelValue` 成对出现，典型用于下拉、树选择等控件在提交数据时同时保留 value 与展示文本：

```json
{
  "type": "select",
  "source": "deptOptions",
  "props": {
    "label": "部门",
    "modelValue": { "$expr": "$model.employee.deptId" },
    "text": { "$expr": "$model.employee.deptName" }
  }
}
```

`props.text` 必须能解析到同一视图上下文可写的 model 字段。
