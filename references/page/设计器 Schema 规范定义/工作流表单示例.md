# 工作流表单示例

本文件维护工作流表单页真实设计器 Core Schema 示例。后续新增工作流申请、办理、审批、浏览类表单示例时，按“示例二、示例三...”继续追加，并优先放真实设计器导出的完整 JSON。

## 示例一：项目工作流表单

- 页面类型：工作流表单页
- 模型类型：`record`
- 模型别名：`projectinfo`
- 运行时页面 id：`page_1780888036004`
- 适合参考：
  - 工作流表单 `pageId` 与 `pagetag` 同时存在的写法
  - 顶部 `toolbar -> workflow-button / workflow-right` 的流程操作区结构
  - `layout-manager -> scrollbar -> collapse -> collapse-item -> form-layout -> form-item -> control` 的表单主体结构
  - `workflow-history` 放在表单主体后的流程历史结构
  - 附件上传 `file-upload` 在工作流材料场景下的控件结构

## 参考要点

1. 工作流表单的主提交入口是顶部 `workflow-button`，不是普通底部 `save-button`。
2. `workflow-right` 放在工具栏 `actions` 区，配合流程办理场景展示右侧流程辅助信息。
3. `workflow-history` 放在表单主体后，用于展示审批历史和流转记录。
4. 附件上传是本示例中的工作流材料字段；新页面只有在需求、截图、流程材料或已有页面要求附件时才生成。
5. `pageId` 用于列表按钮 URL，`pagetag` 用于 workflow `handleurl`，两者不要混用。
