# 表单示例

本文件维护表单页真实设计器 Core Schema 示例。后续新增表单页示例时，按“示例三、示例四...”继续追加，并优先放真实设计器导出的完整 JSON。

## 示例一：项目表单

- 页面类型：表单页
- 模型类型：`record`
- 模型别名：`projectinfo`
- 运行时页面 id：`page_1781580614416`
- 适合参考：
  - `layout-manager -> slots.main -> div -> collapse -> collapse-item -> form-layout -> form-item -> control` 的表单主体结构
  - `slots.bottom` 底部按钮栏结构
  - 普通输入、日期、长文本等基础控件的字段绑定
  - 双列表单和跨整行字段的 `span` 配置
  - 列表页新增、修改、详情按钮引用表单页 `pageId` 的方式

- 模型类型：`record`
- 模型别名：`taskinfo`

  - 任务类字段较多时的表单分组和字段排列

## 使用规则

1. 表单、详情和工作流办理页优先参考本文件的真实 JSON 示例，再结合 `../页面设计器/基础结构/组件与字段对照.md` 做组件选型。
2. 表单数据模型使用 `record`，控件 `props.modelValue` 绑定到具体字段，例如 `{ "$expr": "$model.projectinfo.projectname" }`。
3. 主体结构优先使用 `layout-manager`，表单内容放在 `slots.main`，保存/取消等按钮放在 `slots.bottom`。
4. 所有表单控件必须放在 `form-item` 内，字段 label、span、labelWidth 等配置写在 `form-item.props`。
5. 新增、编辑、详情默认复用同一个表单页 `pageId`，由运行时打开模式区分编辑或只读详情。
6. 没有真实接口或动作流信息时，只生成页面结构和字段绑定，不虚构保存、提交或审批动作。
