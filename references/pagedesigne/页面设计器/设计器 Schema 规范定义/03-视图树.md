# 视图树

`children` 是 Core Schema 的通用页面视图树入口。`ViewNode` 只保留跨运行态通用的结构、绑定、资源、样式和事件字段；PC/mobile 的具体组件节点可以继承 `ViewNode` 后扩展自己的根属性。

## 节点结构

```json
{
  "id": "employeeName",
  "version": "1.0.0",
  "type": "input",
  "model": "employee.name",
  "textModel": "employee.deptName",
  "source": "deptOptions",
  "props": {
    "label": "姓名"
  },
  "style": {},
  "events": {},
  "visible": "model.employee.enabled == true",
  "children": []
}
```

## 字段规则

- `id`：节点稳定标识。省略时可由 normalizer 生成。
- `version`：节点版本号，可选。用于标记节点协议或物料版本。
- `type`：运行态组件类型标识，Core Schema 不枚举具体 PC/mobile 组件集合。
- `model`：短 model 路径，例如 `employee.name`，代表节点主数据绑定（存储的 value）。
- `textModel`：可选的文本侧绑定，代表节点的展示文本。用于下拉、树选择等存在 text/value 对的控件，将用户可见文本保存到独立字段。
- `source`：资源 id，只引用顶层 `resources`。
- `props`：组件语义配置，不放渲染器私有运行时状态。展示文本、标题、间距等简单配置可以放在这里；强结构、强父子依赖的配置由具体组件扩展根属性表达。
- `style`：作者显式指定的样式。
- `events`：事件到顶层 action 的引用。
- `visible`：视图级显隐规则，支持布尔值或受限表达式字符串。省略时默认可见。
- `children`：子节点。

## 组件扩展

Core Schema 不把 `label`、`title`、`columns`、`items`、`gap` 等控件专用字段放在基础 `ViewNode` 上。简单展示配置可以通过 `props` 约定：

```json
{
  "type": "text",
  "props": { "content": "今日待办" }
}
```

```json
{
  "type": "button",
  "props": { "label": "保存" },
  "events": { "click": "saveEmployee" }
}
```

强结构字段由具体组件类型继承 `ViewNode` 后声明为根属性。`columns` 属于表格结构的一部分，只在 table 上有完整语义，不作为普通 `children`，也不降级到 `props.columns`：

```ts
type TableColumn =
  | { type: 'field'; field: string; label?: string }
  | { type: 'actions'; label?: string; items: Array<{ label: string }> };

interface TableNode extends ViewNode {
  type: 'table';
  children?: never;
  columns: TableColumn[];
}
```

## textModel

`textModel` 与 `model` 成对出现，典型用于下拉类控件在提交数据时同时保留文本：

```json
{
  "type": "select",
  "model": "employee.deptId",
  "textModel": "employee.deptName",
  "source": "deptOptions",
  "props": { "label": "部门" }
}
```

`textModel` 必须能解析到同一视图上下文可写的 model 字段。
