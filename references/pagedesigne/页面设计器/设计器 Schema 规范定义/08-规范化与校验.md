# 规范化与校验

`normalize` 负责补齐和统一，`validate` 负责保存或发布前阻断静态错误。两者的输入输出都保持 Core Schema 结构。

## normalize

职责：

- 补齐 `schemaVersion: "core-1.0"`。
- 补齐 `kind: "page"`。
- 为缺失 `id` 的页面和节点生成稳定 id。
- 把节点短路径 `employee.name` 解析为完整引用索引 `model.employee.name`。
- 把事件字符串或字符串数组规范成统一内部事件索引，覆盖顶层 `events` 和节点 `events`。
- 补齐必要结构，如 `children: []`、`props: {}`、`style: {}`。
- 把字段 `rules` 规范成运行时规则索引。
- 保持 `ViewNode.visible` 为布尔值或受限表达式，不提升到 `props`。
- 基础 `ViewNode` 不承载 `label`、`title`、`columns`、`items`、`gap` 等控件专用字段；组件扩展节点可以声明自己的根属性。

接口：

```ts
function normalizeCoreSchema(schema: CorePageSchema): CorePageSchema;
```

## validate

校验规则：

- `schemaVersion` 必须是 `core-1.0`。
- `kind` 必须是 `page`。
- 顶层 `children` 必须是数组。
- 节点 `type` 必须是非空字符串；具体值由运行态或物料包约定。
- 节点 `events` 和顶层 `events` 只能引用存在的顶层 `actions`。
- `events` value 只能是字符串或字符串数组。
- `visible` 只能是布尔值或受限表达式字符串。
- `model` 引用必须能解析到 `models` 中的模型或字段。
- `textModel` 引用必须能解析到同一上下文可写的 model 字段。
- 节点 `source` 必须引用存在且类型兼容的 `resources`。
- `actions.steps[].use` 必须引用存在的 `resource.operation`。
- `actions.steps[].run` 不能形成循环依赖。
- `actions.steps[].function.name` 必须引用已注册且允许调用的自定义 JS 方法。
- `assign` 左侧必须是可写引用。
- `fields.*.rules` 的表达式必须通过白名单解析，包含 `validate[].when` 和 `validate[].expression`。
- `fields.*.rules.validate[].validator` 必须引用存在的顶层 `actions`。
- `state()` 表达式的 path 必须能解析到已注册的节点 id 或 `page.*` 键。
- 同一页面显式 id 不得重复。
- 基础 `ViewNode` 根属性不得出现组件专用字段；具体组件扩展根属性由对应运行态校验。

接口：

```ts
function validateCoreSchema(schema: CorePageSchema): CoreSchemaValidationResult;

interface CoreSchemaValidationResult {
  valid: boolean;
  errors: CoreSchemaError[];
}

interface CoreSchemaError {
  path: string;
  code: string;
  message: string;
}
```

错误示例：

```json
{
  "valid": false,
  "errors": [
    {
      "path": "children[1].events.click",
      "code": "unknown_action",
      "message": "事件引用的 action 不存在：saveEmployee"
    }
  ]
}
```
