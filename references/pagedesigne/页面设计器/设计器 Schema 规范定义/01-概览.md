# 概览

Core Schema 是统一作者态页面协议，用于描述页面结构、数据模型、外部资源、行为、规则和校验约束。它以页面树为核心，吸收企业表单和列表场景需要的数据、资源、动作、规则能力，同时保持字段和节点语义轻量。

## 适用范围

Core Schema 用于：

- AI 根据自然语言生成初始页面。
- 设计器读取、编辑和保存页面。
- `normalize` 补齐默认值、生成稳定 id、建立引用索引。
- `validate` 在保存或发布前阻断静态错误。
- 渲染器消费规范化后的 Schema 并完成运行时展示和交互。

## 设计原则

- 页面树优先：`children` 是唯一视图树。
- AI 友好：字段名和节点类型使用通用语义，接近 HTML、表单和业务页面概念。
- 企业能力内聚：数据、资源、动作放在顶层注册区，节点通过短引用使用。
- 字段保持轻量：字段元信息包含基础类型、显示名、资源引用和规则入口。
- 事件统一：页面级 `events` 与节点 `events` 同构，值均引用顶层 `actions`。
- 规则就近声明：字段规则统一写在 `models.<model>.fields.<field>.rules`，ViewNode 通过 `model`、`textModel`、`visible`、`events` 等字段引用数据和行为。
- 运行态隔离：Schema 不包含缓存、接口响应状态、设计器临时状态或框架私有字段。

## 协议边界

- Schema 字段集合不等同于某个 UI 库、Vue、React 或 FormKit 的组件 props 集合。
- 不允许任意 JavaScript 表达式或动态函数执行。
- 不承载运行时组件快照、接口响应缓存、设计器临时状态或适配层私有字段。

## 模块关系

```text
CorePageSchema
  |- viewport/theme
  |- models        // record / collection
  |- resources
  |- actions
  |- events        // 页面级事件，引用 actions
  `- children      // 节点树，节点 events 引用 actions
```

各顶层注册区互相通过 id 或短路径引用。节点不内联资源调用步骤，也不内联 action steps。
