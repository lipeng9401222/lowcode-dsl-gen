# 动作与事件

`actions` 是行为注册表，用来表达接口调用、赋值、校验、弹窗、列表刷新、导航、提示和工作流能力。节点不内联 action steps，只通过 `events` 引用顶层 action。

## actions

```json
{
  "actions": {
    "refreshEmployees": {
      "steps": [
        {
          "use": "employeeApi.list",
          "params": {
            "keyword": "model.search.keyword"
          },
          "assign": {
            "model.employeeList": "response.items"
          }
        }
      ],
      "onError": {
        "notify": "加载失败",
        "stop": true
      }
    }
  }
}
```

## step 白名单

- `use`：调用资源操作，例如 `employeeApi.list`。
- `assign`：给模型赋值。
- `run`：执行另一个 action。
- `validate`：校验模型、字段或节点。
- `reset`：重置模型或字段。
- `log`：在当前 action 上下文中求值表达式，并通过 `console.log(value)` 输出调试信息。
- `notify`：提示消息。
- `open`：打开弹窗、抽屉或页面片段。
- `close`：关闭弹窗、抽屉或页面片段。
- `confirm`：二次确认。
- `workflow`：调用工作流资源能力。
- `navigate`：跳转路由或链接。
- `function`：执行已注册的自定义 JS 方法。

每个 step 可通过 `when` 表达式控制条件执行。

```json
{
  "when": "model.employee.type == 'external'",
  "assign": {
    "model.employee.vendorRequired": true
  }
}
```

调试时可使用 `log` step 输出表达式求值结果，不会中断后续 step。

```json
{
  "log": "model.employee.name"
}
```

`function.name` 必须引用运行时已注册并允许调用的方法，不支持在 Schema 中内联任意 JS 代码。返回值通过步骤结果 `result` 暴露给后续 `assign` 或表达式。

## events

节点和页面都通过 `events` 引用顶层 `actions`，结构相同：

```json
{
  "type": "button",
  "props": { "label": "保存" },
  "events": {
    "click": "saveEmployee"
  }
}
```

事件值只允许字符串或字符串数组，每个字符串必须引用顶层 `actions` 中存在的 action id。

```json
{
  "events": {
    "change": ["clearVendorName", "refreshVendorOptions"]
  }
}
```

## 页面级 events

页面级 `events` 挂在 `CorePageSchema` 根上，用于页面加载、离开等生命周期节点：

```json
{
  "events": {
    "load": "refreshEmployees",
    "beforeLeave": ["confirmDirty"]
  }
}
```

事件名不强制枚举，`EventName` 是通用字符串类型。常见自定义约定包含 `load`、`beforeLeave`、`show`、`hide`、`resize` 等，具体由运行时负责触发。

## 常见节点事件名约定

- `init`
- `change`
- `click`
- `submit`
- `reset`
- `open`
- `close`
- `pageChange`
- `sortChange`
- `selectionChange`
