# 条件判断节点 (if-else)

## 概述

条件判断节点用于根据条件表达式判断执行流程走向，实现流程分支控制。支持单条件和多条件组合判断。

## 节点配置

**⚠️ 重要：字段顺序必须严格按照以下顺序配置**

```yaml
data:
  cases:                           # 分支情况（必须放在第一位）
    - logical_operator: "and"      # 逻辑运算符
      case_id: "true"              # 分支ID
      id: "true"                   # 条件分支标识
      conditions:                  # 条件配置
      - varType: "string"          # 变量类型
        variable_selector:         # 变量选择器
        - "1765857159183"
        - "body"
        - "projectinfo"
        comparison_operator: "empty"  # 比较运算符
        id: "6746810b-d97e-4e52-a2ac-724287217a29"  # 条件ID（必需）
        value: ""                  # 比较值（必需）
  type: "if-else"                  # 节点类型（放在cases之后）
  title: "判断项目信息在参数中是否存在"  # 节点标题
  selected: false                  # 选中状态
  desc: ""                         # 描述
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "if-else" |
| title | string | 节点显示标题 |
| logical_operator | string | 逻辑运算符：and / or |
| conditions | array | 条件列表 |
| cases | array | 分支情况配置 |

## 逻辑运算符 (logical_operator)

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `and` | 所有条件都满足时为 true | 条件1 AND 条件2 |
| `or` | 任一条件满足时为 true | 条件1 OR 条件2 |

## 条件配置 (conditions)

### 条件结构

```yaml
conditions:
- id: "条件唯一ID"                    # 条件ID（必需，通常为UUID）
  varType: "string"                  # 变量类型（必需）
  variable_selector:                 # 变量选择器（必需）
  - "节点ID"
  - "输出路径"
  - "字段名"
  comparison_operator: "="           # 比较运算符（必需）
  value: "比较值"                     # 比较值（必需，即使为空也要设置为 ""）
```

**⚠️ 必需字段说明**：
- `id`：条件的唯一标识符，通常使用UUID格式，如 `"94c94df3-e51d-41eb-9cf4-340f7c65a20e"`
- `value`：比较值，即使不需要比较值（如 `empty` 运算符），也必须设置为空字符串 `""`

### 变量类型 (varType)

| 类型 | 说明 | 示例 |
|------|------|------|
| `string` | 字符串 | "Hello World" |
| `number` | 数字 | 123, 45.67 |
| `object` | 对象 | {"key": "value"} |
| `array` | 数组 | [1, 2, 3] |
| `boolean` | 布尔值 | true, false |

### 变量选择器 (variable_selector)

变量选择器是一个数组，用于指定要判断的变量路径：

```yaml
# 格式
variable_selector:
- "源节点ID"
- "路径1"
- "路径2"
- "字段名"

# 示例1：判断 Webhook 请求体中的字段
variable_selector:
- "1765857159183"      # Webhook 节点ID
- "body"               # 请求体
- "projectinfo"        # 项目信息对象

# 示例2：判断查询结果
variable_selector:
- "1765867634751"      # 查询节点ID
- "CG_ProjectInfo"     # 表名
- "projectno"          # 字段名
```

## 比较运算符 (comparison_operator)

### 通用运算符

| 运算符 | 说明 | 适用类型 |
|--------|------|----------|
| `empty` | 为空 | 所有类型 |
| `not empty` | 不为空 | 所有类型 |

### 相等性运算符

| 运算符 | 说明 | 适用类型 |
|--------|------|----------|
| `=` / `is` | 等于 | 字符串、数字、布尔值 |
| `≠` / `is not` | 不等于 | 字符串、数字、布尔值 |

### 字符串运算符

| 运算符 | 说明 | 适用类型 |
|--------|------|----------|
| `contains` | 包含 | 字符串、数组 |
| `not contains` | 不包含 | 字符串、数组 |
| `start with` | 以...开始 | 字符串 |
| `end with` | 以...结束 | 字符串 |

### 数字运算符

| 运算符 | 说明 | 适用类型 |
|--------|------|----------|
| `>` | 大于 | 数字 |
| `<` | 小于 | 数字 |
| `≥` | 大于等于 | 数字 |
| `≤` | 小于等于 | 数字 |

## 分支配置 (cases)

条件判断节点有两个分支：

```yaml
cases:
- case_id: "true"       # True 分支（必需）
  id: "true"            # 分支ID（必需，固定为 "true"）
  conditions: [...]     # 满足条件时执行
  
- case_id: "false"      # False 分支（隐式）
  id: "false"
  # 不满足条件时执行
```

## 配置示例

### 场景一：判断字段是否为空

```yaml
data:
  type: "if-else"
  title: "检查项目信息"
  logical_operator: "and"
  conditions:
  - varType: "object"
    variable_selector:
    - "1765857159183"
    - "body"
    - "projectinfo"
    comparison_operator: "not empty"
    id: "cond-001"
    value: ""
```

**说明**：判断 Webhook 请求体中的 projectinfo 对象是否存在。

### 场景二：判断数值大小

```yaml
data:
  type: "if-else"
  title: "检查金额"
  logical_operator: "and"
  conditions:
  - varType: "number"
    variable_selector:
    - "webhook_node_id"
    - "body"
    - "amount"
    comparison_operator: ">"
    id: "cond-002"
    value: "1000"
```

**说明**：判断金额是否大于 1000。

### 场景三：判断字符串相等

```yaml
data:
  type: "if-else"
  title: "检查状态"
  logical_operator: "and"
  conditions:
  - varType: "string"
    variable_selector:
    - "query_node_id"
    - "CG_ProjectInfo"
    - "status"
    comparison_operator: "="
    id: "cond-003"
    value: "approved"
```

**说明**：判断项目状态是否为 "approved"。

### 场景四：多条件 AND 判断

```yaml
data:
  type: "if-else"
  title: "检查用户权限"
  logical_operator: "and"
  conditions:
  - varType: "string"
    variable_selector:
    - "query_node_id"
    - "FRAME_USER"
    - "role"
    comparison_operator: "="
    id: "cond-004"
    value: "admin"
  - varType: "boolean"
    variable_selector:
    - "query_node_id"
    - "FRAME_USER"
    - "isactive"
    comparison_operator: "="
    id: "cond-005"
    value: "true"
```

**说明**：判断用户角色是 admin 且账号激活。

### 场景五：多条件 OR 判断

```yaml
data:
  type: "if-else"
  title: "检查紧急等级"
  logical_operator: "or"
  conditions:
  - varType: "string"
    variable_selector:
    - "webhook_node_id"
    - "body"
    - "priority"
    comparison_operator: "="
    id: "cond-006"
    value: "high"
  - varType: "string"
    variable_selector:
    - "webhook_node_id"
    - "body"
    - "priority"
    comparison_operator: "="
    id: "cond-007"
    value: "urgent"
```

**说明**：判断优先级是 high 或 urgent。

### 场景六：字符串包含判断

```yaml
data:
  type: "if-else"
  title: "检查关键词"
  logical_operator: "and"
  conditions:
  - varType: "string"
    variable_selector:
    - "webhook_node_id"
    - "body"
    - "content"
    comparison_operator: "contains"
    id: "cond-008"
    value: "urgent"
```

**说明**：判断内容是否包含 "urgent" 关键词。

## 连接分支

条件判断节点的输出有两个连接点：

### True 分支

满足条件时执行的节点。

```yaml
edges:
- source: "if_else_node_id"
  sourceHandle: "true"
  target: "next_node_id"
  targetHandle: "target"
```

### False 分支

不满足条件时执行的节点。

```yaml
edges:
- source: "if_else_node_id"
  sourceHandle: "false"
  target: "error_handler_node_id"
  targetHandle: "target"
```

## 实际应用场景

### 场景一：参数验证

```
Webhook 触发
    ↓
条件判断：参数是否完整
    ↓ True          ↓ False
处理业务逻辑    返回错误信息
```

### 场景二：数据存在性检查

```
查询数据库
    ↓
条件判断：数据是否存在
    ↓ True          ↓ False
更新数据        插入新数据
```

### 场景三：权限验证

```
查询用户信息
    ↓
条件判断：用户是否有权限
    ↓ True          ↓ False
执行操作        返回权限不足
```

### 场景四：业务规则判断

```
接收订单
    ↓
条件判断：金额是否超过审批阈值
    ↓ True          ↓ False
进入审批流程    自动批准
```

## 最佳实践

### 1. 条件设计

- **简单明确**：优先使用简单明确的条件，避免过于复杂的逻辑嵌套
- **单一职责**：一个条件判断节点只负责一个明确的判断逻辑
- **避免深层嵌套**：超过 3 层的条件嵌套应考虑重构

### 2. 空值检查

处理用户输入或外部数据时，优先进行 `empty` / `not empty` 检查：

```yaml
# 先检查字段存在性
conditions:
- comparison_operator: "not empty"
  variable_selector: ["node_id", "body", "field"]
```

### 3. 分支处理

- **完整覆盖**：确保 true 和 false 分支都有明确的后续处理节点
- **错误处理**：false 分支应包含适当的错误处理或提示
- **避免空分支**：每个分支都应有实际的处理逻辑

### 4. 性能优化

- **常见条件优先**：将最常见的条件放在前面
- **快速失败**：使用 AND 时，将最可能失败的条件放在前面
- **减少判断**：合并相似的判断逻辑

### 5. 可读性

- **有意义的标题**：使用清晰描述判断逻辑的标题
- **添加描述**：在 desc 字段中说明判断的业务意义
- **变量命名**：引用的变量路径要清晰易懂

## 调试技巧

### 1. 输出调试信息

在 true 和 false 分支后分别添加代码节点，输出当前状态：

```groovy
def main() {
    println "条件判断结果：True"
    return [status: "true"]
}
```

### 2. 检查变量值

在条件判断前添加代码节点，检查变量的实际值和类型：

```groovy
def main(checkValue) {
    println "变量值：$checkValue"
    println "变量类型：${checkValue?.getClass()?.name}"
    return [value: checkValue]
}
```

### 3. 测试边界条件

- 空值：null、""、[]、{}
- 边界值：0、-1、最大值、最小值
- 特殊字符：空格、换行、特殊符号

## 常见问题

### 1. 条件总是走 False 分支

**原因**：
- 变量选择器路径错误
- 变量类型不匹配
- 比较值格式错误

**解决**：
- 检查 variable_selector 路径是否正确
- 确认 varType 与实际数据类型一致
- 验证 value 字段的值是否正确

### 2. 多条件判断不符合预期

**原因**：
- logical_operator 设置错误（and/or）
- 条件之间的逻辑关系理解错误

**解决**：
- 明确业务逻辑要求
- 使用括号或分级判断简化复杂逻辑
- 分步测试每个条件

### 3. 数字比较不生效

**原因**：
- 数字被当作字符串处理
- 小数精度问题

**解决**：
- 确保 varType 设置为 "number"
- 注意浮点数比较的精度问题
