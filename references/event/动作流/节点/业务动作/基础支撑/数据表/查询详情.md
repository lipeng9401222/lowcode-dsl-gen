# 数据表查询节点 (biz-action - misfindaction)

## 概述

数据表查询节点是业务动作节点的一种，用于查询数据表详情。支持复杂条件查询和多字段选择。

## 节点配置

```yaml
data:
  provider_type: "frame"              # 提供者类型
  error_handle_mode: "terminated"     # 错误处理模式
  paramSchemas: []                    # 参数模式
  icon: "./logo/icon-tool9.png"       # 节点图标
  start_node_id: ""                   # 开始节点ID
  type: "biz-action"                  # 节点类型
  title: "查询用户详情"                # 节点标题
  output_selector: []                 # 输出选择器
  iterator_selector: []               # 迭代选择器
  actionData:                         # 动作数据
    panelPath: "frame-datasource/data-query-detail"  # 面板路径
    action: "misfindaction"           # 动作名称：查询详情
    formData:                         # 表单数据
    - name: "sqlTableName"            # 查询表名
      valueMode: "fixed"              # 值模式
      value: "FRAME_USER"             # 表名
    - name: "conditions"              # 查询条件
      valueMode: "fixed"              # 值模式
      value:
        conditionData:                # 条件数据
          name: "and"                 # 条件组合方式
          type: "group"               # 类型：组
          items:                      # 条件项
          - opt: "=="                 # 操作符
            left:                     # 左侧字段
            - dataType: "string"      # 数据类型
              formula: {}             # 公式
              label: "userguid"       # 字段标签
              type: "tableField"      # 类型：表字段
              value: "userguid"       # 字段值
            right:                    # 右侧值
            - dataType: "string"      # 数据类型
              formula: {}             # 公式
              label: "Webhook触发.Body.userguid"  # 引用标签
              type: "field"           # 类型：字段
              value: "s1767841926276Context.data.body.userguid"  # 变量引用
        sqlData:                      # SQL数据（高级模式）
          name: ""
          value: []
        type: "simple"                # 条件类型：简单模式
    - name: "queryFields"             # 查询字段列表
      valueMode: "fixed"              # 值模式
      value:                          # 字段配置数组
      - name: "userguid"              # 字段名
        tableId: 2099122154           # 表ID
        label: "userguid"             # 字段标签
        type: "string"                # 字段类型
      - name: "loginid"
        tableId: 2099122154
        label: "loginid"
        type: "string"
      - name: "displayname"
        tableId: 2099122154
        label: "displayname"
        type: "string"
    category: "biz_action"            # 类别
    panelType: ""                     # 面板类型
  tool_description: "查询数据详情"     # 工具描述
  isAction: true                      # 是否为动作节点
  provider_name: "数据表"              # 提供者名称
  editurl: ""                         # 编辑URL
  selected: false                     # 是否选中
  parallel_nums: 10                   # 并行数量
  tool_name: "misfindaction"          # 工具名称
  variables: []                       # 变量
  params: []                          # 参数
  actionContext: []                   # 动作上下文
  real_type: "frame"                  # 实际类型
  provider_id: "数据表"                # 提供者ID
  output_schema: []                   # 输出模式
  is_parallel: false                  # 是否并行
  is_team_authorization: true         # 是否团队授权
  desc: ""                            # 描述
  tool_label: "查询详情"               # 工具标签
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "biz-action" |
| actionData.action | string | 动作名称，固定为 "misfindaction" |
| tool_name | string | 工具名称，固定为 "misfindaction" |
| provider_name | string | 提供者名称，"数据表" |
| error_handle_mode | string | 错误处理模式：terminated / continue |

## formData 配置

### 1. sqlTableName（表名）

指定要查询的数据表名称。

```yaml
- name: "sqlTableName"
  valueMode: "fixed"
  value: "FRAME_USER"        # 数据表名
```

### 2. conditions（查询条件）

定义查询的条件表达式。

```yaml
- name: "conditions"
  valueMode: "fixed"
  value:
    type: "simple"           # 条件类型：simple / sql
    conditionData: {}        # 简单条件配置
    sqlData: {}              # SQL 条件配置
```

### 3. queryFields(查询字段)

指定需要查询的字段列表。

```yaml
- name: "queryFields"
  valueMode: "fixed"
  value:
  - name: "userguid"
    tableId: 2099122154      # ⚠️ 必填：数据表ID,从 query_table 返回的 tableID 获取
    label: "userguid"
    type: "string"
```

⚠️ **重要提示**：
- `tableId` 字段为**必填项**，缺少会导致导入报错
- 必须使用 `mcp_LowcodeMCP_query_table` 工具查询表结构，从返回结果的 `tableID` 字段获取
- 所有 `queryFields` 数组中的每一项都必须包含 `tableId` 字段

## 条件查询配置

### 简单条件模式 (type: "simple")

使用可视化方式配置查询条件。

```yaml
conditionData:
  name: "and"                 # 逻辑连接符：and / or
  type: "group"               # 类型：group
  items:                      # 条件项数组
  - opt: "=="                 # 操作符
    left:                     # 左侧操作数
    - dataType: "string"
      type: "tableField"      # 表字段
      value: "userguid"       # 字段名
      label: "userguid"
    right:                    # 右侧操作数
    - dataType: "string"
      type: "field"           # 引用其他节点输出
      value: "s{节点ID}Context.data.body.userguid"
      label: "Webhook触发.Body.userguid"
```

#### 操作符 (opt)

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | `userguid == "123"` |
| `!=` | 不等于 | `status != "deleted"` |
| `>` | 大于 | `age > 18` |
| `<` | 小于 | `amount < 1000` |
| `>=` | 大于等于 | `score >= 60` |
| `<=` | 小于等于 | `count <= 100` |
| `like` | 模糊匹配 | `name like "%张%"` |
| `in` | 在列表中 | `status in ("active", "pending")` |
| `not in` | 不在列表中 | `status not in ("deleted")` |
| `is null` | 为空 | `remark is null` |
| `is not null` | 不为空 | `email is not null` |

#### 左侧操作数 (left)

通常是数据表字段。

```yaml
left:
- dataType: "string"          # 数据类型
  type: "tableField"          # 类型：表字段
  value: "userguid"           # 字段名
  label: "userguid"           # 显示标签
  formula: {}                 # 公式（可选）
```

#### 右侧操作数 (right)

可以是固定值或变量引用。

**固定值**：
```yaml
right:
- dataType: "string"
  type: "fixed"               # 类型：固定值
  value: "active"             # 固定值
  label: "active"
```

**引用变量**：
```yaml
right:
- dataType: "string"
  type: "field"               # 类型：字段引用
  value: "s1767841926276Context.data.body.userguid"
  label: "Webhook触发.Body.userguid"
```

### SQL 条件模式 (type: "sql")

使用自定义 SQL WHERE 条件。

```yaml
sqlData:
  name: "customSQL"
  value:
  - "userguid = '${s1767841926276Context.data.body.userguid}'"
  - "AND isenabled = 1"
```

## 查询字段配置

`queryFields` 数组定义需要查询的字段列表。

```yaml
queryFields:
- name: "userguid"            # 字段名
  tableId: 2099122154         # ⚠️ 必填：数据表ID（从 query_table 返回的 tableID 字段获取）
  label: "userguid"           # 显示标签
  type: "string"              # 数据类型
- name: "loginid"
  tableId: 2099122154         # ⚠️ 必填：每个字段都需要配置
  label: "loginid"
  type: "string"
- name: "displayname"
  tableId: 2099122154
  label: "displayname"
  type: "string"
- name: "age"
  tableId: 2099122154
  label: "age"
  type: "int"
- name: "createtime"
  tableId: 2099122154
  label: "createtime"
  type: "datetime"
```

### 获取 tableId 的步骤

1. **调用 MCP 工具查询表结构**：
   ```
   mcp_LowcodeMCP_query_table(query="CG_ProjectInfo")
   ```

2. **从返回结果中提取 tableID**：
   ```json
   {
     "tableID": 201000,           // ← 使用这个值
     "tableName": "CG_ProjectInfo",
     "columns": [...]
   }
   ```

3. **配置到 queryFields**：
   ```yaml
   queryFields:
   - name: "rowguid"
     tableId: 201000             // ← 配置 tableID
     label: "项目GUID"
     type: "string"
   ```

### 字段类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `string` | 字符串 | "张三" |
| `int` | 整数 | 123 |
| `decimal` | 小数 | 45.67 |
| `datetime` | 日期时间 | "2024-01-08 10:30:00" |
| `boolean` | 布尔值 | true / false |
| `text` | 长文本 | 大量文字 |

## 结果输出

查询结果存储在节点上下文的 `data` 对象中，以表名作为键。

```yaml
# 格式
s{节点ID}Context.data.{表名}

# 示例
s1767841950223Context.data.FRAME_USER
```

### 输出结构

查询成功时返回对象：
```json
{
  "userguid": "123456",
  "loginid": "zhangsan",
  "displayname": "张三",
  "age": 28
}
```

查询无结果时返回 `null` 或空对象。

## 配置示例

### 场景一：根据 ID 查询单条记录

```yaml
actionData:
  action: "misfindaction"
  formData:
  - name: "sqlTableName"
    value: "FRAME_USER"
  - name: "conditions"
    value:
      type: "simple"
      conditionData:
        name: "and"
        type: "group"
        items:
        - opt: "=="
          left:
          - type: "tableField"
            value: "userguid"
            dataType: "string"
          right:
          - type: "field"
            value: "s{webhook_id}Context.data.body.userguid"
            dataType: "string"
  - name: "queryFields"
    value:
    - name: "userguid"
      tableId: 2099122154      # ⚠️ 必填：数据表ID
      label: "userguid"
      type: "string"
    - name: "loginid"
      tableId: 2099122154
      label: "loginid"
      type: "string"
    - name: "displayname"
      tableId: 2099122154
      label: "displayname"
      type: "string"
```

### 场景二：多条件查询（AND）

```yaml
conditionData:
  name: "and"
  type: "group"
  items:
  - opt: "=="
    left:
    - type: "tableField"
      value: "isenabled"
      dataType: "int"
    right:
    - type: "fixed"
      value: "1"
      dataType: "int"
  - opt: "like"
    left:
    - type: "tableField"
      value: "displayname"
      dataType: "string"
    right:
    - type: "field"
      value: "s{webhook_id}Context.data.body.keyword"
      dataType: "string"
```

**说明**：查询已启用且名称包含关键词的用户。

### 场景三：多条件查询（OR）

      dataType: "string"

**说明**：通过登录名或邮箱查询用户。

## 最佳实践

### 1. 条件配置

- **优先使用简单模式**：简单条件模式更易维护，除非需要复杂 SQL
- **索引字段查询**：使用有索引的字段作为查询条件，提高性能
- **避免模糊查询**：尽量使用精确匹配，必要时在前端加通配符

### 2. 字段选择

- **只查询需要的字段**：避免查询所有字段 `SELECT *`，影响性能
- **合理选择字段类型**：确保字段类型与数据库一致
- **避免大字段**：如非必要，不要查询 BLOB、TEXT 等大字段

### 3. 变量引用

- **使用完整路径**：在条件中引用变量时，使用完整的上下文路径
- **检查变量存在性**：在查询前先判断变量是否存在（使用条件判断节点）
- **注意数据类型**：确保引用的变量类型与字段类型匹配

### 4. 错误处理

- **配置错误处理模式**：根据业务需求选择 terminated 或 continue
- **处理空结果**：使用条件判断节点检查查询结果是否为空
- **记录查询失败**：在错误分支记录失败信息

### 5. 结果使用

- **检查结果格式**：查询结果是对象，不是数组
- **引用结果字段**：使用 `s{节点ID}Context.data.{表名}.{字段名}` 格式
- **处理空值**：查询无结果时，结果可能为 null，需要判断

## 性能优化

1. **使用索引字段**：查询条件使用有索引的字段
2. **限制查询字段**：只查询必要的字段
3. **避免复杂条件**：简化查询条件，减少数据库负担
4. **缓存查询结果**：对不常变化的数据，可以缓存查询结果
5. **批量查询**：多次查询考虑使用 IN 条件批量查询

## 常见问题

### 0. 导入报错：queryFields 缺少 tableId

**错误现象**：DSL 导入时提示字段配置错误

**原因**：`queryFields` 数组中的字段缺少 `tableId` 配置

**解决方法**：
```yaml
# ❌ 错误配置（缺少 tableId）
queryFields:
- name: "rowguid"
  label: "项目GUID"
  type: "string"

# ✅ 正确配置
queryFields:
- name: "rowguid"
  tableId: 201000      # 必须添加 tableId
  label: "项目GUID"
  type: "string"
```

**获取 tableId**：
1. 调用 `mcp_LowcodeMCP_query_table(query="表名")` 查询表结构
2. 从返回结果的 `tableID` 字段获取数值
3. 配置到每个字段的 `tableId` 属性

### 1. 查询无结果

**原因**：
- 条件不匹配
- 表名错误
- 字段名错误
- 引用的变量值错误

**解决**：
- 检查条件配置是否正确
- 确认表名和字段名拼写
- 使用代码节点打印变量值调试

### 2. 查询结果引用错误

**原因**：引用路径不正确

**解决**：
使用正确的格式：`s{节点ID}Context.data.{表名}.{字段名}`

### 3. 类型不匹配

**原因**：字段类型配置与实际不符

**解决**：
- 检查 queryFields 中的 type 配置
- 确保条件中的 dataType 正确
