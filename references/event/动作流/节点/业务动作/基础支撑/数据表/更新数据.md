# 数据表更新节点 (biz-action - misupdateaction)

## 概述

数据表更新节点用于更新数据表中的记录。支持条件更新、批量更新，以及固定值和公式计算。

## 节点配置

```yaml
data:
  provider_type: "frame"              # 提供者类型
  error_handle_mode: "terminated"     # 错误处理模式
  paramSchemas: []                    # 参数模式

  start_node_id: ""                   # 开始节点ID
  type: "biz-action"                  # 节点类型
  title: "更新数据"                    # 节点标题
  output_selector: []                 # 输出选择器
  iterator_selector: []               # 迭代选择器
  actionData:                         # 动作数据
    panelPath: "frame-datasource/data-update"  # 面板路径
    action: "misupdateaction"         # 动作名称：更新数据
    formData:                         # 表单数据
    - name: "sqlTableName"            # 表名
      valueMode: "fixed"              # 值模式
      value: "FRAME_USER"             # 数据表名
    - name: "updateType"              # 更新类型
      valueMode: "fixed"              # 值模式
      value: "condition"              # condition: 条件更新 / primary: 主键更新
    - name: "conditions"              # 更新条件
      valueMode: "fixed"              # 值模式
      value:
        conditionData:                # 条件数据
          name: "and"                 # 条件组合方式

          - opt: "=="                 # 操作符

        sqlData:                      # SQL数据（高级模式）
          name: ""

        type: "simple"                # 条件类型：简单模式
    - name: "updateFields"            # 更新字段列表
      valueMode: "fixed"              # 值模式
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "biz-action" |
| actionData.action | string | 动作名称，固定为 "misupdateaction" |
| tool_name | string | 工具名称，固定为 "misupdateaction" |
| provider_name | string | 提供者名称，"数据表" |
| error_handle_mode | string | 错误处理模式：terminated / continue |

## formData 配置

### 1. sqlTableName（表名）

指定要更新数据的表名。

```yaml
- name: "sqlTableName"
  valueMode: "fixed"
  value: "FRAME_USER"    # 数据表名
```

### 2. updateType（更新类型）

指定更新方式。

```yaml
- name: "updateType"
  valueMode: "fixed"
  value: "condition"     # condition: 条件更新 / primary: 主键更新
```

| 值 | 说明 | 使用场景 |
|----|------|----------|
| `condition` | 条件更新 | 根据查询条件更新记录 |
| `primary` | 主键更新 | 根据主键直接更新单条记录 |

### 3. conditions（更新条件）

定义更新条件（updateType 为 condition 时必须）。

```yaml
- name: "conditions"
  valueMode: "fixed"
  value:
    type: "simple"           # 条件类型：simple / sql
    conditionData: {}        # 简单条件配置
    sqlData: {}              # SQL 条件配置
```

### 4. updateFields（更新字段）

定义要更新的字段及其值。

```yaml
- name: "updateFields"
  valueMode: "fixed"
  value:
  - name: "email"            # 字段名
    valueMode: "fixed"       # 值模式：fixed / formula
    value: "123@qq.com"      # 字段值
```

## 条件配置

### 简单条件模式 (type: "simple")

使用可视化方式配置更新条件。

```yaml
conditionData:
  name: "and"                 # 逻辑连接符：and / or
  type: "group"               # 类型：group
  items:                      # 条件项数组
  - opt: "=="                 # 操作符
    left:                     # 左侧操作数
    - dataType: "string"
      type: "tableField"      # 表字段
      value: "userguid"       # 字段名
      label: "userguid"
    right:                    # 右侧操作数
    - dataType: "string"
      type: "field"           # 引用其他节点输出
      value: "s{节点ID}Context.data.body.userguid"
      label: "Webhook触发.Body.userguid"
```

#### 操作符 (opt)

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | `userguid == "123"` |
| `!=` | 不等于 | `status != "deleted"` |
| `>` | 大于 | `age > 18` |
| `<` | 小于 | `amount < 1000` |
| `>=` | 大于等于 | `score >= 60` |
| `<=` | 小于等于 | `count <= 100` |
| `like` | 模糊匹配 | `name like "%张%"` |
| `in` | 在列表中 | `status in ("active", "pending")` |
| `not in` | 不在列表中 | `status not in ("deleted")` |
| `is null` | 为空 | `remark is null` |
| `is not null` | 不为空 | `email is not null` |

#### 左侧操作数 (left)

通常是数据表字段。

```yaml
left:
- dataType: "string"          # 数据类型
  type: "tableField"          # 类型：表字段
  value: "userguid"           # 字段名
  label: "userguid"           # 显示标签
  formula: {}                 # 公式（可选）
```

#### 右侧操作数 (right)

可以是固定值或变量引用。

**固定值**：
```yaml
right:
- dataType: "string"
  type: "fix"                 # 类型：固定值
  value: "active"             # 固定值
  label: "固定值"
```

**引用变量**：
```yaml
right:
- dataType: "string"
  type: "field"               # 类型：字段引用
  value: "s1767841926276Context.data.body.userguid"
  label: "Webhook触发.Body.userguid"
```

### SQL 条件模式 (type: "sql")

使用自定义 SQL WHERE 条件。

```yaml
sqlData:
  name: "customSQL"
  value:
  - "userguid = '${s1767841926276Context.data.body.userguid}'"
  - "AND isenabled = 1"
```

## 更新字段配置

`updateFields` 数组定义要更新的字段及其新值。

### 字段值配置模式 (valueMode)

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| `fixed` | 固定值 | 直接指定常量值 |
| `formula` | 公式 | 使用表达式计算值或引用变量 |

### 固定值模式

```yaml
- name: "email"
  valueMode: "fixed"
  value: "123@qq.com"
```

### 公式模式

```yaml
- name: "email"
  valueMode: "formula"
  formula:
    expression: "RuleParamUtil.parserParam(s1767841926276Context,\"data.body.email\")"
    header: ""
    text: "Webhook触发.Body.email"
    value:
    - name: "Webhook触发.Body.email"
      id: "uuid;s1767841926276Context.data.body.email"
      value: "s1767841926276Context.data.body.email"
```

### 更新为当前时间

```yaml
- name: "updatetime"
  valueMode: "formula"
  formula:
    expression: "DateFunctionRule.now(DateFunctionRule_instance)"
    header: "let DateFunctionRule_instance = new com.epoint.rule.engine.customer.function.date.DateFunctionRule();\n"
    text: "now()"
    value:
    - fun: "now"
    - "()"
```

### 引用其他节点输出

```yaml
- name: "displayname"
  valueMode: "formula"
  formula:
    expression: "RuleParamUtil.parserParam(s{query_id}Context,\"data.FRAME_USER.displayname\")"
    header: ""
    text: "查询用户.FRAME_USER.displayname"
    value:
    - name: "查询用户.FRAME_USER.displayname"
      id: "uuid;s{query_id}Context.data.FRAME_USER.displayname"
      value: "s{query_id}Context.data.FRAME_USER.displayname"
```

## 结果输出

更新操作的结果存储在节点上下文中。

```yaml
# 格式
s{节点ID}Context.data.updateCount

# 示例
s1768441678089Context.data.updateCount
```

### 输出结构

更新成功时返回影响的行数：

```json
{
  "updateCount": 5,        // 更新的记录数
  "success": true          // 是否成功
}
```

更新失败时：

```json
{
  "updateCount": 0,
  "success": false,
  "message": "错误信息"
}
```

## 最佳实践

### 1. 更新条件

- **必须配置条件**：避免误更新全表数据
- **使用主键或唯一键**：精确定位要更新的记录
- **索引字段作为条件**：使用有索引的字段提高性能
- **更新前先查询**：使用查询节点确认要更新的记录

### 2. 字段更新

- **只更新需要的字段**：不要更新所有字段
- **更新时间字段**：建议同时更新 updatetime 字段
- **验证数据有效性**：更新前验证数据格式和范围
- **保持数据一致性**：关联字段一起更新

### 3. 软删除

- **重要数据使用软删除**：标记删除而不是物理删除
- **记录删除信息**：更新 isdeleted、deletetime、deleteuser 等字段
- **查询时过滤**：查询时添加 isdeleted = 0 条件

### 4. 批量更新

- **分批处理**：大量数据分批更新，避免长时间锁表
- **使用迭代节点**：批量更新使用迭代节点逐条处理
- **考虑性能**：更新操作会影响数据库性能

### 5. 事务处理

- **相关数据一起更新**：关联数据在同一个流程中完成
- **错误处理**：配置 `error_handle_mode: "terminated"` 保证事务一致性
- **回滚机制**：更新失败时考虑回滚已更新的数据

### 6. 错误处理

- **配置错误处理模式**：根据业务需求选择 terminated 或 continue
- **检查更新结果**：使用条件判断节点检查 updateCount
- **记录失败信息**：在错误分支记录失败原因

### 7. 数据验证

- **验证数据类型**：确保更新值的类型与字段类型匹配
- **字符串长度**：注意 VARCHAR 字段的长度限制
- **数值范围**：INT、DECIMAL 字段注意数值范围
- **日期格式**：确保日期时间格式正确

## 性能优化

1. **使用索引字段**：WHERE 条件使用有索引的字段
2. **减少更新字段**：只更新必要的字段
3. **分批更新**：大量数据分批更新，每批 1000-10000 条
4. **避免全表扫描**：确保更新条件能利用索引
5. **异步处理**：大量数据更新使用异步任务
6. **并行更新**：多个独立更新可以使用并行节点

## 常见问题

### 1. 更新失败

**原因**：
- 条件配置错误
- 字段类型不匹配
- 字段不存在
- 权限不足
- 数据验证失败

**解决**：
- 检查更新条件是否正确
- 验证字段值类型
- 确认字段名拼写
- 检查数据库用户权限
- 验证数据库约束

### 2. 更新结果为 0

**原因**：没有符合条件的记录

**解决**：
- 检查更新条件是否正确
- 使用查询节点验证记录是否存在
- 检查变量引用是否正确
- 确认数据是否已被更新

### 3. 误更新数据

**原因**：
- 更新条件不准确
- 未配置条件
- 条件中使用了错误的变量

**解决**：
- 更新前先使用查询节点验证
- 必须配置更新条件
- 在测试环境先验证
- 使用精确的主键条件

### 4. 字段值类型错误

**原因**：更新值的类型与字段类型不匹配

**解决**：
- 检查字段数据类型
- 确保更新值格式正确
- 使用类型转换函数
- 验证数值范围

### 5. 更新超时

**原因**：
- 更新数据量过大
- 未使用索引字段
- 数据库锁等待

**解决**：
- 分批更新
- 在索引字段上添加更新条件
- 避免在高峰期执行大量更新
- 优化数据库索引

### 6. 并发更新冲突

**原因**：多个流程同时更新同一记录

**解决**：
- 使用乐观锁（版本号）
- 使用悲观锁（FOR UPDATE）
- 添加重试机制
- 避免并发更新同一记录
