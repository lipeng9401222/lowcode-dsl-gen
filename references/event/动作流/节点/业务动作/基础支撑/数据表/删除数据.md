# 数据表删除节点 (biz-action - misdeleteaction)

## 概述

数据表删除节点用于删除数据表中的记录。支持条件删除和批量删除，建议谨慎使用。

## 节点配置

```yaml
data:
  provider_type: "frame"              # 提供者类型
  error_handle_mode: "terminated"     # 错误处理模式
  paramSchemas: []                    # 参数模式

  start_node_id: ""                   # 开始节点ID
  type: "biz-action"                  # 节点类型
  title: "删除数据"                    # 节点标题
  output_selector: []                 # 输出选择器
  iterator_selector: []               # 迭代选择器
  actionData:                         # 动作数据
    panelPath: "frame-datasource/data-delete"  # 面板路径
    action: "misdeleteaction"         # 动作名称：删除数据
    formData:                         # 表单数据
    - name: "sqlTableName"            # 表名
      valueMode: "fixed"              # 值模式
      value: "FRAME_USER"             # 数据表名
    - name: "conditions"              # 删除条件
      valueMode: "fixed"              # 值模式
      value:
        conditionData:                # 条件数据
          name: "and"                 # 条件组合方式
          type: "group"               # 类型：组

            - dataType: "string"      # 数据类型
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "biz-action" |
| actionData.action | string | 动作名称，固定为 "misdeleteaction" |
| tool_name | string | 工具名称，固定为 "misdeleteaction" |
| provider_name | string | 提供者名称，"数据表" |
| error_handle_mode | string | 错误处理模式：terminated / continue |

## formData 配置

### 1. sqlTableName（表名）

指定要删除数据的表名。

```yaml
- name: "sqlTableName"
  valueMode: "fixed"
  value: "FRAME_USER"    # 数据表名
```

### 2. conditions（删除条件）

定义删除条件，**必须配置条件**，避免误删除全表数据。

```yaml
- name: "conditions"
  valueMode: "fixed"
  value:
    type: "simple"           # 条件类型：simple / sql
    conditionData: {}        # 简单条件配置
    sqlData: {}              # SQL 条件配置
```

## 条件配置

### 简单条件模式 (type: "simple")

使用可视化方式配置删除条件。

```yaml
conditionData:
  name: "and"                 # 逻辑连接符：and / or
  type: "group"               # 类型：group
  items:                      # 条件项数组
  - opt: "=="                 # 操作符
    left:                     # 左侧操作数
    - dataType: "string"
      type: "tableField"      # 表字段
      value: "userguid"       # 字段名
      label: "userguid"
    right:                    # 右侧操作数
    - dataType: "string"
      type: "field"           # 引用其他节点输出
      value: "s{节点ID}Context.data.body.userguid"
      label: "Webhook触发.Body.userguid"
```

#### 操作符 (opt)

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | `userguid == "123"` |
| `!=` | 不等于 | `status != "active"` |
| `>` | 大于 | `age > 18` |
| `<` | 小于 | `amount < 1000` |
| `>=` | 大于等于 | `score >= 60` |
| `<=` | 小于等于 | `count <= 100` |
| `like` | 模糊匹配 | `name like "%test%"` |
| `in` | 在列表中 | `status in ("deleted", "expired")` |
| `not in` | 不在列表中 | `status not in ("active")` |
| `is null` | 为空 | `remark is null` |
| `is not null` | 不为空 | `userguid is not null` |

#### 左侧操作数 (left)

通常是数据表字段。

```yaml
left:
- dataType: "string"          # 数据类型
  type: "tableField"          # 类型：表字段
  value: "userguid"           # 字段名
  label: "userguid"           # 显示标签
  formula: {}                 # 公式（可选）
```

#### 右侧操作数 (right)

可以是固定值或变量引用。

**固定值**：
```yaml
right:
- dataType: "string"
  type: "fix"                 # 类型：固定值
  value: "deleted"            # 固定值
  label: "固定值"
```

**引用变量**：
```yaml
right:
- dataType: "string"
  type: "field"               # 类型：字段引用
  value: "s1767841926276Context.data.body.userguid"
  label: "Webhook触发.Body.userguid"
```

### SQL 条件模式 (type: "sql")

使用自定义 SQL WHERE 条件。

```yaml
sqlData:
  name: "customSQL"
  value:
  - "auditstatus = '1'"
  - "AND createtime < '2024-01-01'"
```

## 结果输出

删除操作的结果存储在节点上下文中。

```yaml
# 格式
s{节点ID}Context.data.deleteCount

# 示例
s1768442149626Context.data.deleteCount
```

### 输出结构

删除成功时返回影响的行数：

```json
{
  "deleteCount": 5,        // 删除的记录数
  "success": true          // 是否成功
}
```

删除失败时：

```json
{
  "deleteCount": 0,
  "success": false,
  "message": "错误信息"
}
```

## 配置示例

### 场景一：根据主键删除单条记录

```yaml
actionData:
  action: "misdeleteaction"
  formData:
  - name: "sqlTableName"
    value: "FRAME_USER"
  - name: "conditions"
    value:
      type: "simple"
      conditionData:
        name: "and"
        type: "group"
        items:
        - opt: "=="
          left:
          - type: "tableField"
            value: "userguid"
            dataType: "string"
          right:
          - type: "field"
            value: "s{webhook_id}Context.data.body.userguid"
            dataType: "string"
```

### 场景二：根据状态批量删除

```yaml
conditionData:
  name: "and"
  type: "group"
  items:
  - opt: "=="
    left:
    - type: "tableField"
      value: "status"
      dataType: "string"
    right:
    - type: "fix"
      value: "deleted"
      dataType: "string"
  - opt: "=="
    left:
    - type: "tableField"
      value: "auditstatus"
      dataType: "string"
    right:
    - type: "fix"
      value: "1"
      dataType: "string"
```

**说明**：删除状态为 "deleted" 且审核状态为 1 的记录。

### 场景三：删除过期数据

```yaml
conditionData:
  name: "and"
  type: "group"
  items:
  - opt: "<"
    left:
    - type: "tableField"
      value: "expiretime"
      dataType: "datetime"
    right:
    - type: "field"
      value: "s{code_node_id}Context.data.currentTime"
      dataType: "datetime"
  - opt: "=="
    left:
    - type: "tableField"
      value: "isdeleted"
      dataType: "int"
    right:
    - type: "fix"
      value: "0"
      dataType: "int"
```

**说明**：删除已过期且未被标记删除的记录。

### 场景四：根据外键删除关联数据

```yaml
conditionData:
  name: "and"
  type: "group"
  items:
  - opt: "=="
    left:
    - type: "tableField"
      value: "projectguid"
      dataType: "string"
    right:
    - type: "field"
      value: "s{query_node_id}Context.data.CG_ProjectInfo.projectguid"
      dataType: "string"
```

**说明**：删除指定项目下的所有子记录。

### 场景五：IN 条件批量删除

```yaml
conditionData:
  name: "and"
  type: "group"
  items:
  - opt: "in"
    left:
    - type: "tableField"
      value: "userguid"
      dataType: "string"
    right:
    - type: "field"
      value: "s{webhook_id}Context.data.body.userguids"
      dataType: "string"
```

**说明**：根据用户 GUID 列表批量删除用户（userguids 格式："guid1,guid2,guid3"）。

### 场景六：条件删除（多条件 OR）

```yaml
conditionData:
  name: "or"
  type: "group"
  items:
  - opt: "=="
    left:
    - type: "tableField"
      value: "status"
      dataType: "string"
    right:
    - type: "fix"
      value: "expired"
      dataType: "string"
  - opt: "=="
    left:
    - type: "tableField"
      value: "status"
      dataType: "string"
    right:
    - type: "fix"
      value: "cancelled"
      dataType: "string"
```

**说明**：删除状态为 "expired" 或 "cancelled" 的记录。

## 最佳实践

### 1. 安全性

- **必须配置条件**：永远不要删除无条件的记录（会删除全表）
- **谨慎使用 OR 条件**：可能删除超出预期的记录
- **删除前先查询**：使用查询节点确认要删除的记录
- **使用事务**：关联数据删除使用事务保证一致性
- **记录删除日志**：重要数据删除前记录日志

### 2. 软删除优先

对重要数据，建议使用软删除（标记删除）而不是物理删除。

```yaml
# 推荐：软删除（更新状态）
- title: "标记删除"
  actionData:
    action: "misupdateaction"
    formData:
    - name: "sqlTableName"
      value: "FRAME_USER"
    - name: "conditions"
      value:
        conditionData:
          items:
          - opt: "=="
            left:
            - type: "tableField"
              value: "userguid"
            right:
            - type: "field"
              value: "s{webhook_id}Context.data.body.userguid"
    - name: "updateFields"
      value:
      - name: "isdeleted"
        valueMode: "fixed"
        value: "1"
      - name: "deletetime"
        valueMode: "formula"
        formula:
          expression: "DateFunctionRule.now(DateFunctionRule_instance)"
```

### 3. 级联删除

- **先删子表再删主表**：避免外键约束错误
- **检查关联数据**：删除前检查是否有关联数据
- **使用事务**：多表删除使用事务保证原子性

### 4. 批量删除

- **分批处理**：大量数据分批删除，避免长时间锁表
- **使用 IN 条件**：批量删除使用 IN 操作符
- **考虑性能**：删除操作会影响数据库性能

### 5. 条件配置

- **使用主键或唯一键**：精确定位要删除的记录
- **索引字段作为条件**：使用有索引的字段提高性能
- **验证条件准确性**：确保条件能准确匹配目标记录

### 6. 错误处理

- **配置错误处理模式**：根据业务选择 terminated 或 continue
- **检查删除结果**：使用条件判断节点检查 deleteCount
- **记录失败信息**：在错误分支记录失败原因

### 7. 数据备份

- **重要数据先备份**：删除重要数据前先备份
- **定期备份**：定期备份数据库
- **可恢复性**：考虑数据的可恢复性

## 性能优化

1. **使用索引字段**：WHERE 条件使用有索引的字段
2. **分批删除**：大量数据分批删除，每批 1000-10000 条
3. **避免全表扫描**：确保删除条件能利用索引
4. **清理索引**：删除大量数据后重建索引
5. **异步处理**：大量数据删除使用异步任务

## 安全检查清单

在执行删除操作前，请确认：

- [ ] 已配置明确的删除条件
- [ ] 删除条件准确，不会误删其他记录
- [ ] 已在测试环境验证删除条件
- [ ] 重要数据已备份
- [ ] 考虑使用软删除代替物理删除
- [ ] 已检查级联关系，确保关联数据处理
- [ ] 已配置错误处理
- [ ] 已记录删除操作日志

## 常见问题

### 1. 删除失败

**原因**：
- 外键约束冲突
- 条件配置错误
- 权限不足
- 表名或字段名错误

**解决**：
- 先删除子表数据，再删除主表数据
- 检查删除条件是否正确
- 确认数据库用户权限
- 验证表名和字段名拼写

### 2. 误删除数据

**原因**：
- 删除条件不准确
- 未配置条件
- 条件中使用了错误的变量

**解决**：
- 删除前先使用查询节点验证
- 必须配置删除条件
- 在测试环境先验证
- 重要数据使用软删除

### 3. 外键约束错误

**原因**：有子表数据引用主表记录

**解决**：
- 先删除子表关联数据
- 使用级联删除
- 检查数据库外键约束配置

### 4. 删除超时

**原因**：
- 删除数据量过大
- 未使用索引字段
- 数据库锁等待

**解决**：
- 分批删除
- 在索引字段上添加删除条件
- 避免在高峰期执行大量删除

### 5. 删除结果为 0

**原因**：没有符合条件的记录

**解决**：
- 检查删除条件是否正确
- 使用查询节点验证记录是否存在
- 检查变量引用是否正确
- 确认数据是否已被删除

### 6. 删除后无法回滚

**原因**：物理删除无法恢复

**解决**：
- 重要数据使用软删除
- 删除前备份数据
- 从数据库备份恢复
