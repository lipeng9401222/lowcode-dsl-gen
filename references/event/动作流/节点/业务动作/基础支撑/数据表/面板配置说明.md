# 数据表节点面板配置说明

## 使用场景

当MCP服务查询到业务动作节点包含 `panelPath` 字段时，根据面板路径查询本文档，获取对应的节点配置模板。

## ⚠️ 重要约束

**表名验证规则**：
- ✅ **必须**：使用 `query_table` 工具验证表名是否存在
- ✅ **必须**：只能配置 MCP 工具能查询到的数据表
- ❌ **禁止**：配置查询不到的表名
- ❌ **禁止**：假设或猜测表名

**验证流程**：
1. 使用 `query_table(query="表名")` 查询表结构
2. 确认返回结果中包含该表
3. 使用返回结果中的 `tableName` 字段（如 `FRAME_USER`）作为配置值

## 五大面板配置

### 1. 查询列表 (data-query-list)

```yaml
panelPath: "frame-datasource/data-query-list"
action: "misfindlistaction"
tool_name: "misfindlistaction"
tool_label: "查询列表"

formData:
  - name: "sqlTableName"      # 表名（⚠️ 必须使用 query_table 验证存在）
  - name: "conditions"        # 查询条件（可选）
  - name: "queryFields"       # 查询字段（⚠️ 必须从 query_table 返回的 columns 中选择，且每个字段必须包含 tableId）
  - name: "pageIndex"         # 页码（0=不分页）
  - name: "pageSize"          # 每页大小（0=全部）
```

**配置要求**：
- `sqlTableName`：必须先使用 `query_table` 验证表存在，使用返回的 `tableName` 值
- `queryFields`：字段名必须从 `query_table` 返回的 `columns[].name` 中选择
- **⚠️ 关键**：`queryFields` 数组中每个字段对象必须包含 `tableId` 字段（从 `query_table` 返回的 `tableID` 获取）

### 2. 查询详情 (data-query-detail)

```yaml
panelPath: "frame-datasource/data-query-detail"
action: "misfindaction"
tool_name: "misfindaction"
tool_label: "查询详情"

formData:
  - name: "sqlTableName"      # 表名（⚠️ 必须使用 query_table 验证存在）
  - name: "conditions"        # 查询条件（必填）
  - name: "queryFields"       # 查询字段（⚠️ 必须从 query_table 返回的 columns 中选择，且每个字段必须包含 tableId）
```

**配置要求**：
- `sqlTableName`：必须先使用 `query_table` 验证表存在，使用返回的 `tableName` 值
- `queryFields`：字段名必须从 `query_table` 返回的 `columns[].name` 中选择
- **⚠️ 关键**：`queryFields` 数组中每个字段对象必须包含 `tableId` 字段（从 `query_table` 返回的 `tableID` 获取）

### 3. 新增数据 (data-add)

```yaml
panelPath: "frame-datasource/data-add"
action: "misaddaction"
tool_name: "misaddaction"
tool_label: "插入数据"

formData:
  - name: "sqlTableName"      # 表名（⚠️ 必须使用 query_table 验证存在）
  - name: "entities"          # 插入字段配置数组（⚠️ 字段名必须从 query_table 返回的 columns 中选择）
```

**配置要求**：
- `sqlTableName`：必须先使用 `query_table` 验证表存在，使用返回的 `tableName` 值
- `entities` 数组中的字段名：必须从 `query_table` 返回的 `columns[].name` 中选择
- 主键字段：如果 `columns[].isPrimaryKey` 为 `true`，需要生成 UUID
- 必填字段：如果 `columns[].required` 为 `true`，必须提供值

### 4. 更新数据 (data-update)

```yaml
panelPath: "frame-datasource/data-update"
action: "misupdateaction"
tool_name: "misupdateaction"
tool_label: "更新数据"

formData:
  - name: "sqlTableName"      # 表名（⚠️ 必须使用 query_table 验证存在）
  - name: "updateType"        # 更新类型（condition/primary）
  - name: "conditions"        # 更新条件（必填，字段名必须验证）
  - name: "updateFields"      # 更新字段配置数组（⚠️ 字段名必须从 query_table 返回的 columns 中选择）
```

**配置要求**：
- `sqlTableName`：必须先使用 `query_table` 验证表存在，使用返回的 `tableName` 值
- `conditions` 中的字段名：必须从 `query_table` 返回的 `columns[].name` 中选择
- `updateFields` 数组中的字段名：必须从 `query_table` 返回的 `columns[].name` 中选择

### 5. 删除数据 (data-delete)

```yaml
panelPath: "frame-datasource/data-delete"
action: "misdeleteaction"
tool_name: "misdeleteaction"
tool_label: "删除数据"

formData:
  - name: "sqlTableName"      # 表名（⚠️ 必须使用 query_table 验证存在）
  - name: "conditions"        # 删除条件（必填，字段名必须验证）
```

**配置要求**：
- `sqlTableName`：必须先使用 `query_table` 验证表存在，使用返回的 `tableName` 值
- `conditions` 中的字段名：必须从 `query_table` 返回的 `columns[].name` 中选择
- ⚠️ **安全警告**：删除操作必须配置条件，避免误删除全表数据

## 节点配置模板

基于 `panelPath`，生成完整的节点配置：

```yaml
data:
  provider_type: "frame"
  provider_name: "数据表"
  provider_id: "数据表"
  type: "biz-action"
  isAction: true
  real_type: "frame"
  is_team_authorization: true
  error_handle_mode: "terminated"
  
  # 根据 panelPath 填充以下字段
  title: "{tool_label}"
  tool_name: "{tool_name}"
  tool_label: "{tool_label}"
  tool_description: "{tool_description}"
  
  actionData:
    panelPath: "{panelPath}"
    action: "{action}"
    formData: []              # 根据上面的 formData 配置
    category: "biz_action"
    panelType: ""

## 字段补充说明（仅适用于基础支撑-数据表）

### data 级别
- `tool_name`：必须等于面板对应的 action（如查询详情为 `misfindaction`）
- `provider_name`、`provider_id`：固定为 `数据表`

### actionData 级别
- `category`：固定为 `biz_action`
```

## 配置流程示例

### 完整配置步骤（查询节点）

**步骤 1：查询表结构**
```
调用：mcp_LowcodeMCP_query_table(query="CG_ProjectInfo")
```

**步骤 2：提取关键信息**
```json
返回结果：
{
  "tableID": 201000,              // ← 提取 tableID
  "tableName": "CG_ProjectInfo",   // ← 提取 tableName
  "columns": [
    {"name": "rowguid", "isPrimaryKey": true, "required": true},
    {"name": "projectname", "required": false},
    {"name": "projectno", "required": false}
  ]
}
```

**步骤 3：配置节点**
```yaml
formData:
- name: "sqlTableName"
  valueMode: "fixed"
  value: "CG_ProjectInfo"        # ← 使用 tableName

- name: "queryFields"
  valueMode: "fixed"
  value:
  - name: "rowguid"              # ← 从 columns[].name 获取
    tableId: 201000              # ← 必填：使用 tableID
    label: "项目GUID"
    type: "string"
  - name: "projectname"
    tableId: 201000              # ← 必填：每个字段都要配置
    label: "项目名称"
    type: "string"
  - name: "projectno"
    tableId: 201000              # ← 必填：统一使用同一个 tableID
    label: "项目编号"
    type: "string"
```

### ❌ 常见错误

```yaml
# 错误1：缺少 tableId
queryFields:
  - name: "rowguid"              # ❌ 缺少 tableId
    label: "项目GUID"
    type: "string"

# 错误2：tableId 错误或不一致
queryFields:
  - name: "rowguid"
    tableId: 999999              # ❌ 使用了错误的 tableId
    label: "项目GUID"
    type: "string"

# 错误3：未验证表名就配置
sqlTableName: "MY_CUSTOM_TABLE"  # ❌ 未使用 query_table 验证
```

### ✅ 正确示例

```yaml
# 正确：先用 query_table 验证，使用返回的实际名称
sqlTableName: "CG_ProjectInfo"   # ✅ 从 query_table 返回的 tableName

queryFields:
  - name: "rowguid"               # ✅ 从 query_table 返回的 columns[].name
    tableId: 201000               # ✅ 从 query_table 返回的 tableID（必填）
    label: "项目GUID"
    type: "string"
  - name: "projectname"
    tableId: 201000               # ✅ 每个字段都必须配置 tableId
    label: "项目名称"
    type: "string"
```

## 核心约束总结

### ✅ 必须遵守

1. **表名验证**：任何数据表节点配置前，必须先使用 `query_table` 验证表存在
2. **字段验证**：所有字段名必须从 `query_table` 返回的 `columns[].name` 中选择
3. **tableId 配置**：`queryFields` 数组中每个字段对象都必须包含 `tableId` 字段，值从 `query_table` 返回的 `tableID` 获取
4. **使用实际名称**：使用 MCP 工具返回的实际表名和字段名，不得假设或猜测
5. **主键处理**：主键字段（`isPrimaryKey: true`）需要生成 UUID
6. **必填字段**：必填字段（`required: true`）不能为空

### ❌ 严格禁止

1. **禁止配置未验证的表名**：未使用 `query_table` 验证的表名不得配置
2. **禁止使用不存在的字段**：未在 `columns` 中出现的字段名不得使用
3. **禁止遗漏 tableId**：`queryFields` 中的字段缺少 `tableId` 配置会导致导入失败
4. **禁止假设表结构**：不得根据业务需求假设表名和字段名
5. **禁止跳过验证**：任何情况下都不得跳过 `query_table` 验证步骤

### ⚠️ 特别注意

**tableId 是必填项**：
- 所有查询类节点（查询列表、查询详情）的 `queryFields` 配置中，每个字段对象都必须包含 `tableId` 字段
- 缺少 `tableId` 会导致 DSL 导入时报错
- `tableId` 的值必须从 `mcp_LowcodeMCP_query_table` 工具返回结果的 `tableID` 字段获取
- 同一个表的所有字段使用相同的 `tableId` 值

**获取 tableId 示例**：
```
调用：mcp_LowcodeMCP_query_table(query="CG_ProjectInfo")
返回：{"tableID": 201000, ...}  // ← 使用 201000 作为 tableId
```
