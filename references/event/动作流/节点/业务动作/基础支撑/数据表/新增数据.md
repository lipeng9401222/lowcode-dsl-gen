# 数据表插入节点 (biz-action - misaddaction)

## 概述

数据表插入节点用于向数据表中插入新记录。支持固定值、公式计算和变量引用。

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "biz-action" |
| actionData.action | string | 动作名称，固定为 "misaddaction" |
| tool_name | string | 工具名称，固定为 "misaddaction" |
| provider_name | string | 提供者名称，"数据表" |
| error_handle_mode | string | 错误处理模式：terminated / continue |
| isInIteration | boolean | 是否在迭代节点中 |
| iteration_id | string | 所属迭代节点ID（在迭代中时） |

## formData 配置

### 1. sqlTableName（表名）

指定要插入数据的表名。

```yaml
- name: "sqlTableName"
  valueMode: "fixed"
  value: "CG_BiaoDuanInfo"    # 数据表名
```

### 2. entities（实体数据）

定义要插入的字段及其值。

```yaml
- name: "entities"
  valueMode: "fixed"
  value:                      # 字段配置数组
  - name: "字段名"
    valueMode: "fixed"/"formula"
    value: "值"
    formula: {}               # 公式配置（valueMode为formula时）
```

## 字段值配置模式 (valueMode)

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| `fixed` | 固定值 | 直接指定常量值 |
| `formula` | 公式 | 使用表达式计算值或引用变量 |

### 固定值模式

```yaml
- name: "status"
  valueMode: "fixed"
  value: "active"
```

### 公式模式

### 公式结构

```yaml

  value: []                   # 值数组
```

## 常用公式函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `StringFunctionRule.getUUID()` | 生成 UUID | 用于主键字段 |
| `RuleParamUtil.parserParam()` | 解析参数 | 引用其他节点输出 |
| `DateFunctionRule.now()` | 获取当前时间 | 时间戳字段 |
| `DateFunctionRule.formatDate()` | 格式化日期 | 日期格式转换 |

## 结果输出

插入成功后，结果存储在节点上下文中：

```yaml
# 格式
s{节点ID}Context.data.{表名}

# 示例
s1765867634751Context.data.CG_ProjectInfo
```

### 输出结构

插入成功时返回插入的记录数据（包括自动生成的字段值）：

## 最佳实践

### 1. 主键生成

- **始终使用 UUID**：使用 `StringFunctionRule.getUUID()` 生成唯一主键
- **不要使用固定值**：主键必须保证唯一性
- **自增主键**：如果数据库支持自增，可以不插入主键字段

### 2. 必填字段

- **确保所有必填字段都有值**：检查表结构的 NOT NULL 字段
- **提供默认值**：对可选字段提供合理的默认值
- **验证数据完整性**：在插入前使用条件判断节点验证数据

### 3. 数据类型

- **注意字段数据类型**：确保插入的值类型与字段类型匹配
- **字符串长度**：注意 VARCHAR 字段的长度限制
- **数值精度**：DECIMAL 字段注意精度和小数位数
- **日期格式**：确保日期时间格式正确

### 4. 关联字段

- **先插入主表**：有外键关系时，先插入主表获取主键
- **正确引用主键**：使用 `s{节点ID}Context.data.{表名}.{主键字段}` 引用
- **检查引用有效性**：确保引用的字段确实存在值

### 5. 迭代插入

- **使用正确的上下文**：迭代内插入使用 `s{迭代节点ID}Context.data.item.{字段名}`
- **配置 isInIteration**：在迭代中设置 `isInIteration: true`
- **设置 iteration_id**：指定所属的迭代节点ID
- **考虑性能**：大量数据考虑启用并行执行

### 6. 时间字段

- **使用当前时间**：创建时间使用 `DateFunctionRule.now()`
- **统一时间格式**：更新时间也使用相同的时间函数
- **时区处理**：注意服务器时区设置

### 7. 事务处理

- **相关数据一起插入**：关联数据在同一个流程中完成
- **错误处理**：配置 `error_handle_mode: "terminated"` 保证事务一致性
- **回滚机制**：插入失败时考虑删除已插入的相关数据

## 性能优化

1. **批量插入**：如果系统支持，使用批量插入接口
2. **减少字段数量**：只插入必要的字段
3. **异步处理**：非关键数据可以异步插入
4. **索引优化**：确保插入的字段有合适的索引
5. **并行执行**：迭代插入时启用并行（注意数据库连接池）

## 常见问题

### 1. 插入失败

**原因**：
- 主键冲突
- 必填字段缺失
- 字段类型不匹配
- 外键约束违反

**解决**：
- 使用 UUID 生成主键
- 检查所有必填字段
- 验证字段值类型
- 确保外键引用正确

### 2. 引用变量错误

**原因**：引用路径不正确或引用的值不存在

**解决**：
- 使用正确的格式：`s{节点ID}Context.data.{表名/item}.{字段名}`
- 在插入前使用条件判断节点检查变量是否存在

### 3. 迭代插入失败

**原因**：
- 未设置 isInIteration
- iteration_id 错误
- 迭代项引用路径错误

**解决**：
- 确保配置 `isInIteration: true`
- 正确设置 `iteration_id`
- 使用正确的迭代项引用格式

### 4. 时间字段格式错误

**原因**：时间格式与数据库不匹配

**解决**：
- 使用 `DateFunctionRule.now()` 获取当前时间
- 使用 `DateFunctionRule.formatDate()` 格式化
