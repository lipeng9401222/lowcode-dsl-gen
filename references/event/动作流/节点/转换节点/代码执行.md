# 代码节点 (Code - Groovy)

## 概述

代码节点允许使用 Groovy 语言编写自定义逻辑，用于数据转换、流式处理、复杂计算等场景。

## 节点配置

```yaml
data:
  outputs:                 # 输出配置
    result:               # 结果输出
      type: "string"      # 输出类型
  variables:              # 变量配置
  - value_selector: "{{#1753410080199.text#}}"  # 值选择器
    variable_type: "constant"  # 变量类型
    variable: "arg1"      # 变量名
    value: "{{#1753410080199.text#}}"  # 变量值
  code_language: "groovy"  # 代码语言
  code: "import com.epoint.rule.action.StreamingIterator..."  # 代码内容
  type: "code"            # 节点类型
  title: "流式输出拼接"     # 节点标题
  selected: false
  desc: ""
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "code" |
| title | string | 节点显示标题 |
| code_language | string | 代码语言，目前支持 "groovy" |
| code | string | 代码内容 |
| variables | array | 输入变量配置 |
| outputs | object | 输出配置 |

## 变量配置

### 输入变量 (variables)

```yaml
variables:
- variable: "arg1"                           # 变量名
  variable_type: "constant"                  # 变量类型
  value_selector: "{{#前置节点ID.输出字段#}}"  # 值选择器
  value: "{{#前置节点ID.输出字段#}}"          # 变量值
```

| 字段 | 说明 |
|------|------|
| variable | 在代码中使用的变量名 |
| variable_type | 变量类型：constant(常量) / variable(变量) |
| value_selector | 引用前置节点输出的选择器 |
| value | 变量值（与 value_selector 相同） |

### 输出配置 (outputs)

```yaml
outputs:
  result:                # 输出字段名
    type: "string"       # 输出类型：string / number / object / array
  data:
    type: "object"
  count:
    type: "number"
```

## 代码编写

### 基本结构

```groovy
def main(arg1, arg2) {
    // 你的代码逻辑
    
    return [
        result: "输出结果",
        data: ["key": "value"],
        count: 123
    ]
}
```

### 返回值

代码必须返回一个 Map 对象，键名对应 outputs 配置中的字段名。

```groovy
return [
    result: "处理结果",
    status: "success"
]
```

## 常用场景

### 场景一：流式输出拼接

使用 `StreamingIterator` 处理 LLM 的流式输出。

```groovy
import com.epoint.rule.action.StreamingIterator
import java.util.ArrayList
import java.util.List

def main(arg1) {
    def list = []
    
    def chgline = { line ->
        def newLine = line
        list << newLine  // 把处理后的行加入 list
        return newLine
    }
    
    def ret = new StreamingIterator(arg1, chgline)
    
    // 触发处理
    ret.each { }
    
    return [
        result: list
    ]
}
```

### 场景二：JSON 解析与拼接

解析流式输出的 JSON 数据并拼接内容。

```groovy
import groovy.json.JsonSlurper

def main(arg1) {
    def slurper = new JsonSlurper()
    
    // 1. 解析每个 JSON 字符串
    def chunks = arg1.collect { slurper.parseText(it) }
    
    // 2. 提取并拼接 content
    def fullText = chunks
        .collect { it.result?.content }   // 获取 content
        .findAll { it }                   // 过滤 null
        .join()                           // 拼接
    
    // 3. 提取并拼接 reasoning_content
    def fullReason = chunks
        .collect { it.result?.reasoning_content }
        .findAll { it }
        .join()
    
    return [
        result: fullText,
        reason: fullReason
    ]
}
```

### 场景三：数据转换

将数组数据转换为不同格式。

```groovy
def main(dataArray) {
    // 转换为 Map
    def dataMap = dataArray.collectEntries { item ->
        [(item.id): item.name]
    }
    
    // 过滤数据
    def filtered = dataArray.findAll { it.status == "active" }
    
    // 映射转换
    def transformed = dataArray.collect { item ->
        [
            id: item.id,
            name: item.name.toUpperCase(),
            timestamp: System.currentTimeMillis()
        ]
    }
    
    return [
        map: dataMap,
        filtered: filtered,
        transformed: transformed
    ]
}
```

### 场景四：字符串处理

```groovy
def main(text) {
    // 分割
    def lines = text.split('\n')
    
    // 清理空白
    def cleaned = lines.collect { it.trim() }.findAll { it }
    
    // 提取关键词
    def keywords = text.tokenize(' ').findAll { it.length() > 3 }
    
    // 统计
    def wordCount = text.tokenize(' ').size()
    def charCount = text.length()
    
    return [
        lines: cleaned,
        keywords: keywords,
        wordCount: wordCount,
        charCount: charCount
    ]
}
```

### 场景五：日期时间处理

```groovy
import java.text.SimpleDateFormat
import java.util.Date

def main(timestamp) {
    def sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss")
    
    // 当前时间
    def now = new Date()
    def nowStr = sdf.format(now)
    
    // 解析时间戳
    def date = new Date(timestamp as Long)
    def dateStr = sdf.format(date)
    
    // 时间计算
    def tomorrow = new Date(now.time + 24 * 60 * 60 * 1000)
    def tomorrowStr = sdf.format(tomorrow)
    
    return [
        now: nowStr,
        date: dateStr,
        tomorrow: tomorrowStr
    ]
}
```

### 场景六：条件逻辑

```groovy
def main(value, threshold) {
    def result
    def level
    
    if (value > threshold) {
        result = "高于阈值"
        level = "high"
    } else if (value == threshold) {
        result = "等于阈值"
        level = "medium"
    } else {
        result = "低于阈值"
        level = "low"
    }
    
    return [
        result: result,
        level: level,
        value: value,
        threshold: threshold
    ]
}
```

## 常用 Groovy 操作

### 集合操作

```groovy
// 遍历
list.each { item -> println item }

// 映射
def newList = list.collect { it * 2 }

// 过滤
def filtered = list.findAll { it > 10 }

// 查找
def found = list.find { it.id == 1 }

// 求和
def sum = list.sum()

// 去重
def unique = list.unique()

// 排序
def sorted = list.sort()
```

### 字符串操作

```groovy
// 拼接
def str = "Hello" + " World"
def str2 = "Hello $name"

// 分割
def parts = str.split(' ')

// 替换
def replaced = str.replace("Hello", "Hi")

// 大小写
def upper = str.toUpperCase()
def lower = str.toLowerCase()

// 判断

def contains = str.contains("Hello")
```

### Map 操作

```groovy
// 创建

// 访问
def value = map.key1
def value2 = map["key2"]

// 遍历
map.each { key, value ->
    println "$key: $value"

// 过滤
def filtered = map.findAll { k, v -> v.startsWith("val") }

// 转换
def list = map.collect { k, v -> [name: k, value: v] }
```

## 性能优化

1. **避免重复计算**：将结果缓存在变量中
2. **使用合适的数据结构**：选择合适的 List/Map/Set
3. **减少循环嵌套**：优化算法复杂度
4. **及时释放资源**：大对象使用后置 null
5. **惰性求值**：使用流式处理大数据集

## 最佳实践

1. **函数式编程**：优先使用 collect、findAll 等函数式方法
2. **代码复用**：将通用逻辑提取为函数
3. **错误处理**：使用 try-catch 处理可能的异常
4. **注释说明**：添加必要的注释说明逻辑
5. **测试验证**：在实际使用前充分测试代码

## 注意事项

1. 代码节点的变量名必须与 main 函数参数名一致
2. 返回值的键名必须与 outputs 配置一致
3. 注意变量类型，Groovy 是动态类型语言
4. 避免使用阻塞操作，影响流式处理性能
5. 大数据量处理时注意内存占用
