# 变量赋值节点

## 概述

变量赋值节点(Assigner)用于设置和管理工作流中的变量,支持固定值赋值、变量引用、公式计算等多种方式。可以将计算结果、函数返回值、节点输出等赋值给环境变量或会话变量。

## 节点配置

### 基本结构

```yaml
- data:
    epoint_formula:
      "4d868a36-1156-41d8-a1ca-3024dce7854f":
        expression: "EpointDateUtil.getCurrentDate()"
        header: "use com.epoint.core.utils.date.EpointDateUtil;\r\n"
        text: "EpointDateUtil_getCurrentDate()"
        value:
        - fun: "EpointDateUtil_getCurrentDate"
        - "()"
    type: "assigner"
    title: "变量赋值"
    version: "2"
    items:
    - write_mode: "over-write"
      variable_selector:
      - "env"
      - "nowdate"
      input_type: "variable"
      value: "{{@4d868a36-1156-41d8-a1ca-3024dce7854f@}} "
      operation: "over-write"
    selected: false
    desc: ""
  id: "1766021638512"
  type: "custom"
```

## 配置项详解

### 1. items (赋值项列表)

定义一组变量赋值操作,每个赋值项配置如下:

#### variable_selector (目标变量)

指定要赋值的变量路径。

**格式**：
- `["env", "变量名"]` - 环境变量
- `["conversation", "变量名"]` - 会话变量

**示例**：

```yaml
variable_selector:
- "env"
- "nowdate"  # 赋值给环境变量 nowdate
```

#### value (赋值内容)

要赋值的值,支持多种形式:

1. **固定值**：

```yaml
value: "固定文本"
```

2. **变量引用**：

```yaml
value: "{{#1766021575240.output#}}"  # 引用其他节点输出
```

3. **公式计算**：

```yaml
value: "{{@4d868a36-1156-41d8-a1ca-3024dce7854f@}}"  # 引用公式结果
```

#### input_type (输入类型)

指定值的类型。

| 类型 | 说明 |
|-----|------|
| `variable` | 变量引用或公式 |
| `constant` | 固定值 |

#### write_mode / operation (写入模式)

定义变量的写入方式。

| 模式 | 说明 |
|------|------|
| `over-write` | 覆盖写入(完全替换原值) |
| `append` | 追加写入(在原值后追加) |
| `clear` | 清空变量 |

**示例**：

```yaml
write_mode: "over-write"
operation: "over-write"
```

### 2. epoint_formula (公式配置)

定义可在变量赋值中使用的公式或函数调用。

#### 基本结构

```yaml
epoint_formula:
  "唯一ID":
    expression: "实际执行的表达式"
    header: "需要导入的包或前置代码"
    text: "公式的可读文本表示"
    value:
    - "函数名或运算符"
    - "参数"
```

#### 示例配置

##### 获取当前日期

```yaml
epoint_formula:
  "4d868a36-1156-41d8-a1ca-3024dce7854f":
    expression: "EpointDateUtil.getCurrentDate()"
    header: "use com.epoint.core.utils.date.EpointDateUtil;\r\n"
    text: "EpointDateUtil_getCurrentDate()"
    value:
    - fun: "EpointDateUtil_getCurrentDate"
    - "()"
```

##### 日期计算(前一天)

```yaml
epoint_formula:
  "f58b270b-8782-440e-ab8e-e984f4538f88":
    expression: "EpointDateUtil.convertDate2String(EpointDateUtil.addDay(EpointDateUtil.convertString2Date(RuleParamUtil.parserParam(nowdate,\"\"),\"yyyy-MM-dd\"),-1),\"yyyy-MM-dd\")"
    header: "use com.epoint.core.utils.date.EpointDateUtil;\r\n"
    text: "EpointDateUtil_convertDate2String(addDay(EpointDateUtil_convertString2Date(ENVIRONMENT.env.nowdate,'yyyy-MM-dd'),-1),'yyyy-MM-dd')"
    value:
    - fun: "EpointDateUtil_convertDate2String"
    - "("
    - fun: "addDay"
    - "("
    - fun: "EpointDateUtil_convertString2Date"
    - "("
    - name: "ENVIRONMENT.env.nowdate"
      id: "4e34e8c1-78ef-43fe-9bab-a2f308f89e61;senvContext.data.env.nowdate"
      value: "senvContext.data.env.nowdate"
    - ",'yyyy-MM-dd'),-1),'yyyy-MM-dd')"
```

##### 清除缓存

```yaml
epoint_formula:
  "727078be-389d-4d7d-952b-fb702a5d82f5":
    expression: "CacheMonitorUtil.clearByCacheName(\"frameuserservice\")"
    header: "use com.epoint.frame.cache.monitor.utils.CacheMonitorUtil;\r\n"
    text: "CacheMonitorUtil_clearByCacheName(\"frameuserservice\",null)"
    value:
    - fun: "CacheMonitorUtil_clearByCacheName"
    - "(\"frameuserservice\",null)"
```

## 常用公式函数

### 日期时间函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `EpointDateUtil.getCurrentDate()` | 获取当前日期 | `2025-12-17` |
| `EpointDateUtil.addDay(date, days)` | 日期加减天数 | `addDay(date, -1)` 前一天 |
| `EpointDateUtil.convertString2Date(str, format)` | 字符串转日期 | `convertString2Date("2025-12-17", "yyyy-MM-dd")` |
| `EpointDateUtil.convertDate2String(date, format)` | 日期转字符串 | `convertDate2String(date, "yyyy-MM-dd")` |

### 缓存函数

| 函数 | 说明 |
|------|------|
| `CacheMonitorUtil.clearByCacheName(name)` | 清除指定名称的缓存 |
| `CacheMonitorUtil.clearByCacheName(name, keys)` | 清除指定缓存的特定键 |

### 字符串函数

| 函数 | 说明 |
|------|------|
| `StringUtil.isEmpty(str)` | 判断字符串是否为空 |
| `StringUtil.concat(str1, str2, ...)` | 拼接字符串 |
| `StringUtil.substring(str, start, end)` | 截取字符串 |

```yaml

        expression: "EpointDateUtil.getCurrentDate()"
        header: "use com.epoint.core.utils.date.EpointDateUtil;\r\n"
        text: "EpointDateUtil_getCurrentDate()"
        value:
        - fun: "EpointDateUtil_getCurrentDate"
        - "()"
    type: "assigner"
    title: "获取当前日期"
    version: "2"
    items:
    - write_mode: "over-write"
      variable_selector:
      - "env"
      - "nowdate"
      input_type: "variable"
      value: "{{@4d868a36-1156-41d8-a1ca-3024dce7854f@}}"
      operation: "over-write"
  id: "1766021638512"
  type: "custom"
```

**说明**：将当前日期赋值给环境变量 `nowdate`。

### 场景二：计算前一天日期

```yaml

        header: "use com.epoint.core.utils.date.EpointDateUtil;\r\n"
        text: "EpointDateUtil_convertDate2String(addDay(EpointDateUtil_convertString2Date(ENVIRONMENT.env.nowdate,'yyyy-MM-dd'),-1),'yyyy-MM-dd')"
        value:
        - fun: "EpointDateUtil_convertDate2String"
        - "("
        - fun: "addDay"
        - "("
        - fun: "EpointDateUtil_convertString2Date"
        - "("
        - name: "ENVIRONMENT.env.nowdate"
          id: "4e34e8c1-78ef-43fe-9bab-a2f308f89e61;senvContext.data.env.nowdate"
          value: "senvContext.data.env.nowdate"
        - ",'yyyy-MM-dd'),-1),'yyyy-MM-dd')"
    type: "assigner"
    title: "计算前一天"
    version: "2"
    items:
    - write_mode: "over-write"
      variable_selector:
      - "env"
      - "fromdate"
      input_type: "variable"
      value: "{{@f58b270b-8782-440e-ab8e-e984f4538f88@}}"
      operation: "over-write"

  type: "custom"
```

**说明**：
1. 读取环境变量 `nowdate`
2. 转换为日期格式
3. 减去1天
4. 转换回字符串格式
5. 赋值给环境变量 `fromdate`

## 变量引用

### 引用其他节点的输出

```yaml
value: "{{#1766024909006.response.body.code#}}"
```

### 引用环境变量

```yaml
value: "{{#env.nowdate#}}"
```

### 引用公式结果

```yaml
value: "{{@4d868a36-1156-41d8-a1ca-3024dce7854f@}}"
```

详细的变量引用规范请参考 [变量引用规范](../进阶指南/变量引用规范.md)。

## 最佳实践

1. **命名规范**：使用有意义的变量名,如 `nowdate`、`fromdate`、`totalCount` 等
2. **公式复用**：将常用的公式定义在 `epoint_formula` 中便于复用
3. **类型一致**：注意变量的数据类型,避免类型不匹配导致的错误
4. **避免覆盖**：谨慎使用 `over-write` 模式,避免误覆盖重要数据
5. **及时清理**：不再使用的变量应及时清空,释放内存

## 常见问题

### 1. 如何在公式中引用变量？

使用 `RuleParamUtil.parserParam()` 函数:

```yaml
expression: "RuleParamUtil.parserParam(nowdate,\"\")"
```

或在 value 配置中直接引用:

```yaml
value:
- name: "ENVIRONMENT.env.nowdate"
  id: "xxx;senvContext.data.env.nowdate"
  value: "senvContext.data.env.nowdate"
```

### 2. 如何处理日期格式？

使用 `EpointDateUtil` 的转换函数:

```groovy
// 字符串转日期
EpointDateUtil.convertString2Date("2025-12-17", "yyyy-MM-dd")

// 日期转字符串
EpointDateUtil.convertDate2String(date, "yyyy-MM-dd HH:mm:ss")
```

### 3. 公式执行失败怎么办?

1. 检查 `header` 是否正确导入所需的包
2. 检查表达式语法是否正确
3. 检查引用的变量是否存在
4. 查看工作流执行日志获取详细错误信息

### 4. 如何清空变量？

```yaml
items:
- write_mode: "over-write"
  variable_selector:
  - "env"
  - "varname"
  input_type: "constant"
  value: ""
  operation: "over-write"
```

或使用 `clear` 操作:

```yaml
items:
- write_mode: "clear"
  variable_selector:
  - "env"
  - "varname"
  operation: "clear"
```

## 注意事项

1. ⚠️ 公式中的 Java 类需要在 `header` 中正确导入
2. ⚠️ 变量赋值是同步操作,复杂计算会影响工作流性能
3. ⚠️ 环境变量在整个工作流中共享,修改会影响后续节点
4. ⚠️ 公式 ID 需要保持唯一性,避免冲突
