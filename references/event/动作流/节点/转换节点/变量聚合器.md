# 变量聚合器节点

## 概述

变量聚合器节点(Variable Aggregator)用于将多个变量聚合为一个结构化的数据对象,支持简单聚合和分组聚合两种模式。可以将分散的变量组合成有层次结构的数据,便于统一管理和传递。

## 节点配置

### 基本结构

```yaml
- sourceId: "1768547930086"
  data:
    variables:                        # 简单变量列表
    - - "env"
      - "aa"
    advanced_settings:                # 高级设置
      groups:                         # 分组配置
      - variables:                    # 分组内的变量
        - - "env"
          - "aa"
        group_name: "Group1"          # 分组名称
        groupId: "5351f429-afef-481f-a482-e4825c98cf95"  # 分组ID
      group_enabled: true             # 是否启用分组
    type: "variable-aggregator"       # 节点类型
    title: "变量聚合器"                # 节点标题
    selected: true
    desc: ""
  sourcePosition: "right"
  type: "custom"
  sourceHandle: "source"
  targetPosition: "left"
  width: 244
  positionAbsolute:
    x: 686
    y: 282
  id: "1768547937646"
  position:
    x: 686
    y: 282
  resultType: 2
  selected: true
  height: 107
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "variable-aggregator" |
| title | string | 节点显示标题 |
| variables | array | 简单模式的变量列表（二维数组） |
| advanced_settings | object | 高级设置，包含分组配置 |
| advanced_settings.group_enabled | boolean | 是否启用分组模式 |
| advanced_settings.groups | array | 分组列表 |
| resultType | integer | 结果类型 |

## 聚合模式

### 模式一：简单聚合

将多个变量聚合为一个扁平的对象结构。

```yaml
data:
  variables:
  - - "env"              # 来源类型
    - "username"         # 变量名
  - - "env"
    - "email"
  - - "1768547922695"    # 节点ID
    - "userId"
  type: "variable-aggregator"
  title: "聚合用户信息"
  advanced_settings:
    group_enabled: false  # 不启用分组
```

**输出结果**：
```json
{
  "username": "张三",
  "email": "zhangsan@example.com",
  "userId": "1001"
}
```

### 模式二：分组聚合

将变量按分组聚合为具有层次结构的对象。

```yaml
data:
  advanced_settings:
    group_enabled: true          # 启用分组
    groups:
    - group_name: "UserInfo"     # 用户信息组
      groupId: "group-001"
      variables:
      - - "env"
        - "username"
      - - "env"
        - "email"
    - group_name: "DeptInfo"     # 部门信息组
      groupId: "group-002"
      variables:
      - - "env"
        - "deptName"
      - - "env"
        - "deptCode"
  type: "variable-aggregator"
  title: "分组聚合信息"
```

**输出结果**：
```json
{
  "UserInfo": {
    "username": "张三",
    "email": "zhangsan@example.com"
  },
  "DeptInfo": {
    "deptName": "技术部",
    "deptCode": "TECH001"
  }
}
```

## 变量配置

### variables (变量列表)

定义要聚合的变量路径。

**格式**：二维数组，每个变量由两个元素组成

```yaml
variables:
- - "来源类型"
  - "变量名称"
```

**来源类型**：
- `env` - 环境变量
- `conversation` - 会话变量
- `节点ID` - 其他节点的输出

**示例**：

```yaml
variables:
# 引用环境变量
- - "env"
  - "username"

# 引用会话变量
- - "conversation"
  - "sessionId"

# 引用其他节点输出
- - "1768547922695"
  - "output.data"
```

### advanced_settings.groups (分组配置)

定义变量的分组结构。

```yaml
groups:
- group_name: "分组名称"           # 分组在输出对象中的键名
  groupId: "唯一标识"              # 分组的唯一ID（UUID）
  variables:                      # 该分组包含的变量列表
  - - "env"
    - "var1"
  - - "env"
    - "var2"
```

#### group_name (分组名称)

- 作为输出对象的键名
- 应使用有意义的名称（如 UserInfo、OrderData 等）
- 支持中文和英文

#### groupId (分组ID)

- UUID 格式的唯一标识
- 用于内部标识和管理分组
- 每个分组必须有唯一的 ID

## 使用场景

### 场景一：聚合用户信息

将分散在环境变量中的用户信息聚合为一个对象。

```yaml
- data:
    variables:
    - - "env"
      - "userId"
    - - "env"
      - "userName"
    - - "env"
      - "userEmail"
    - - "env"
      - "userPhone"
    type: "variable-aggregator"
    title: "聚合用户信息"
    advanced_settings:
      group_enabled: false
  id: "1768547937646"
  type: "custom"
```

**输出**：
```json
{
  "userId": "1001",
  "userName": "张三",
  "userEmail": "zhangsan@example.com",
  "userPhone": "13800138000"
}
```

### 场景二：聚合多个节点的输出

将多个节点的输出结果聚合到一起。

```yaml
- data:
    variables:
    - - "1768547922695"    # 用户查询节点
      - "userData"
    - - "1768547930086"    # 部门查询节点
      - "deptData"
    - - "1768547935000"    # 角色查询节点
      - "roleData"
    type: "variable-aggregator"
    title: "聚合查询结果"
    advanced_settings:
      group_enabled: false
  id: "1768547937647"
  type: "custom"
```

**输出**：
```json
{
  "userData": { ... },
  "deptData": { ... },
  "roleData": { ... }
}
```

### 场景三：按业务对象分组聚合

将变量按业务含义分组聚合。

```yaml
- data:
    advanced_settings:
      group_enabled: true
      groups:
      - group_name: "User"
        groupId: "5351f429-afef-481f-a482-e4825c98cf95"
        variables:
        - - "env"
          - "userId"
        - - "env"
          - "userName"
        - - "env"
          - "userEmail"
      - group_name: "Department"
        groupId: "7462g540-bfhg-592g-b593-f5936d6b0da6"
        variables:
        - - "env"
          - "deptId"
        - - "env"
          - "deptName"
      - group_name: "Metadata"
        groupId: "8573h651-cgih-603h-c604-g6047e7c1eb7"
        variables:
        - - "env"
          - "createTime"
        - - "env"
          - "updateTime"
    type: "variable-aggregator"
    title: "按对象分组聚合"
  id: "1768547937648"
  type: "custom"
```

**输出**：
```json
{
  "User": {
    "userId": "1001",
    "userName": "张三",
    "userEmail": "zhangsan@example.com"
  },
  "Department": {
    "deptId": "DEPT001",
    "deptName": "技术部"
  },
  "Metadata": {
    "createTime": "2025-01-15 10:00:00",
    "updateTime": "2025-01-16 15:30:00"
  }
}
```

### 场景四：准备 API 请求数据

聚合变量为 API 请求所需的数据结构。

```yaml
- data:
    advanced_settings:
      group_enabled: true
      groups:
      - group_name: "requestBody"
        groupId: "api-body-001"
        variables:
        - - "env"
          - "action"
        - - "env"
          - "userId"
        - - "1768547930086"
          - "formData"
      - group_name: "requestHeaders"
        groupId: "api-header-001"
        variables:
        - - "env"
          - "token"
        - - "env"
          - "contentType"
    type: "variable-aggregator"
    title: "准备API请求"
  id: "1768547937649"
  type: "custom"
```

**输出**：
```json
{
  "requestBody": {
    "action": "create",
    "userId": "1001",
    "formData": { ... }
  },
  "requestHeaders": {
    "token": "abc123xyz",
    "contentType": "application/json"
  }
}
```

### 场景五：聚合迭代数据

在迭代过程中聚合当前项的相关数据。

```yaml
# 流程：迭代节点 -> 查询详情 -> 变量聚合器
- data:
    variables:
    - - "1768548000000"    # 迭代节点当前项
      - "item.id"
    - - "1768548100000"    # 查询详情节点输出
      - "detail"
    - - "env"
      - "batchId"
    type: "variable-aggregator"
    title: "聚合迭代数据"
    advanced_settings:
      group_enabled: false
  id: "1768547937650"
  type: "custom"
```

### 场景六：多层级分组聚合

创建复杂的嵌套数据结构（通过多个聚合器实现）。

```yaml
# 第一层：聚合基础信息
- data:
    advanced_settings:
      group_enabled: true
      groups:
      - group_name: "BasicInfo"
        groupId: "group-basic"
        variables:
        - - "env"
          - "id"
        - - "env"
          - "name"
    type: "variable-aggregator"
    title: "聚合基础信息"
  id: "1768547937651"
  type: "custom"

# 第二层：聚合详细信息
- sourceId: "1768547937651"
  data:
    variables:
    - - "1768547937651"    # 引用上一个聚合器的输出
      - "output"
    - - "env"
      - "extendInfo"
    type: "variable-aggregator"
    title: "聚合完整数据"
    advanced_settings:
      group_enabled: false
  id: "1768547937652"
  type: "custom"
```

## 结果输出

变量聚合器节点的输出结果可以通过以下方式引用：

```yaml
# 格式
{{#节点ID.output#}}

# 示例 - 引用整个聚合对象
{{#1768547937646.output#}}

# 示例 - 引用聚合对象中的某个字段（简单聚合）
{{#1768547937646.output.username#}}

# 示例 - 引用聚合对象中的某个分组字段（分组聚合）
{{#1768547937646.output.UserInfo.username#}}
```

输出结果为对象类型：
- **简单聚合**：扁平的 JSON 对象
- **分组聚合**：嵌套的 JSON 对象

## 与其他节点配合

### 与模板转换节点配合

```yaml
# 流程：变量聚合器 -> 模板转换
- id: "1768547937646"
  data:
    type: "variable-aggregator"
    title: "聚合数据"
    
- sourceId: "1768547937646"
  data:
    type: "template-transform"
    title: "格式化输出"
    template: "用户：{{ name }}，部门：{{ dept }}"
    variables:
    - variable: "name"
      value_selector: ["1768547937646", "output.username"]
    - variable: "dept"
      value_selector: ["1768547937646", "output.deptName"]
```

### 与代码执行节点配合

```yaml
# 流程：变量聚合器 -> 代码执行
- id: "1768547937646"
  data:
    type: "variable-aggregator"
    title: "聚合数据"
    
- sourceId: "1768547937646"
  data:
    type: "code"
    title: "处理聚合数据"
    variables:
    - variable: "aggregatedData"
      value_selector: ["1768547937646", "output"]
    code: |
      def main(aggregatedData) {
        // 处理聚合后的数据
        return [result: aggregatedData]
      }
```

### 与 HTTP 请求节点配合

```yaml
# 流程：变量聚合器 -> HTTP请求

  data:
    type: "variable-aggregator"
    title: "准备请求数据"

      group_enabled: true

      - group_name: "body"

        - - "env"
          - "userId"

    type: "http-request"

```

## 最佳实践

### 1. 模式选择

- **简单聚合**：适用于扁平数据结构，变量较少的场景
- **分组聚合**：适用于复杂数据结构，需要分类管理的场景
- **明确目的**：根据后续节点的需求选择合适的聚合模式

### 2. 分组设计

- **语义化命名**：分组名称应清晰表达业务含义
- **合理分组**：按业务逻辑或数据类型分组
- **避免过深嵌套**：建议最多2-3层嵌套
- **保持一致性**：在同一工作流中保持分组命名风格一致

### 3. 变量管理

- **检查变量存在性**：确保引用的变量存在且有值
- **类型一致性**：确保聚合的变量类型符合预期
- **避免重名**：简单聚合模式下避免使用相同的变量名
- **文档说明**：在节点描述中说明聚合的数据结构

### 4. 性能优化

- **按需聚合**：只聚合必要的变量
- **避免大对象**：不要聚合过大的数据对象
- **合理嵌套**：避免过多层级的嵌套结构
- **复用聚合结果**：聚合后的数据可以被多个后续节点使用

### 5. 数据结构

- **扁平优先**：能用简单聚合就不用分组聚合
- **清晰层次**：分组聚合要有清晰的层次结构
- **便于引用**：设计易于后续节点引用的数据结构
- **标准格式**：遵循 JSON 数据格式规范

## 常见问题

### 1. 聚合结果为空对象

**原因**：
- 变量路径配置错误
- 引用的变量不存在
- 变量值为 null 或 undefined

**解决**：
```yaml
# 检查变量路径是否正确
variables:
- - "env"          # 确认来源类型
  - "username"     # 确认变量名拼写

# 在前置节点中验证变量是否存在
# 使用变量赋值节点给变量设置默认值
```

### 2. 分组模式不生效

**原因**：
- `group_enabled` 未设置为 true
- groups 配置为空数组
- 配置格式错误

**解决**：
```yaml
advanced_settings:
  group_enabled: true  # 必须设置为 true
  groups:              # 必须配置至少一个分组
  - group_name: "Group1"
    groupId: "xxx"
    variables:
    - - "env"
      - "var1"
```

### 3. 聚合后的数据无法引用

**原因**：
- 引用路径错误
- 节点 ID 不正确
- 字段路径不完整

**解决**：
```yaml
# 简单聚合 - 引用格式
{{#节点ID.output.变量名#}}
例如：{{#1768547937646.output.username#}}

# 分组聚合 - 引用格式
{{#节点ID.output.分组名.变量名#}}
例如：{{#1768547937646.output.UserInfo.username#}}
```

### 4. 变量值被覆盖

**原因**：
- 简单聚合模式下存在重名变量
- 多个变量引用了相同的路径

**解决**：
```yaml
# 使用分组聚合避免重名
advanced_settings:
  group_enabled: true
  groups:
  - group_name: "Source1"
    groupId: "group-001"
    variables:
    - - "env"
      - "name"      # 这个 name
  - group_name: "Source2"
    groupId: "group-002"
    variables:
    - - "conversation"
      - "name"      # 和这个 name 不会冲突
```

### 5. 嵌套层级过深

**原因**：
- 过多的分组嵌套
- 引用了其他聚合器的输出再次聚合

**解决**：
- 重新设计数据结构，减少嵌套层级
- 使用扁平化的数据结构
- 如果必须嵌套，建议不超过3层

## 注意事项

1. ⚠️ 变量路径必须准确，格式为 `[来源类型, 变量名]`
2. ⚠️ 简单聚合模式下，相同变量名会导致值覆盖
3. ⚠️ 分组聚合需要设置 `group_enabled: true` 才能生效
4. ⚠️ 每个分组的 groupId 必须唯一
5. ⚠️ 聚合后的对象结构应便于后续节点引用
6. ⚠️ 不要聚合过大的数据对象，影响性能
7. ⚠️ 引用聚合器输出时注意区分简单聚合和分组聚合的路径差异
