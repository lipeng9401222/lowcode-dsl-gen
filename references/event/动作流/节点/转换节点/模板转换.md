# 模板转换节点

## 概述

模板转换节点(Template Transform)用于通过模板语法对变量进行格式化输出和组合,支持使用 `{{ }}` 语法引用变量,实现文本拼接、格式化等数据转换功能。

## 节点配置

### 基本结构

```yaml
- sourceId: "1768547922695"
  data:
    template: "{{ arg1 }}"           # 模板内容
    variables:                        # 变量配置
    - value_selector:                 # 变量值选择器
      - "env"
      - "aa"
      variable: "arg1"                # 变量名
    type: "template-transform"        # 节点类型
    title: "模板转换"                  # 节点标题
    selected: false
    desc: ""
  sourcePosition: "right"
  type: "custom"
  sourceHandle: "source"
  targetPosition: "left"
  width: 244
  positionAbsolute:
    x: 383
    y: 282
  id: "1768547930086"
  position:
    x: 383
    y: 282
  resultType: 2
  selected: false
  height: 53
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "template-transform" |
| title | string | 节点显示标题 |
| template | string | 模板内容，使用 `{{ 变量名 }}` 语法引用变量 |
| variables | array | 变量配置列表 |
| resultType | integer | 结果类型，2 表示字符串输出 |

## 变量配置

### variables (变量列表)

定义模板中使用的变量及其数据源。

```yaml
variables:
- variable: "arg1"                   # 变量名（在模板中使用）
  value_selector:                    # 值选择器（数据来源）
  - "env"                            # 来源类型
  - "aa"                             # 变量名称
```

#### value_selector (值选择器)

指定变量的数据来源路径。

**格式**：
- `["env", "变量名"]` - 引用环境变量
- `["conversation", "变量名"]` - 引用会话变量
- `["节点ID", "输出字段"]` - 引用其他节点的输出

**示例**：

```yaml
# 引用环境变量
value_selector:
- "env"
- "username"

# 引用其他节点输出
value_selector:
- "1768547922695"
- "output"
```

## 模板语法

### 基本语法

模板使用双大括号 `{{ }}` 包裹变量名：

```yaml
template: "{{ arg1 }}"
```

### 文本拼接

可以在模板中组合文本和变量：

```yaml
template: "用户名：{{ username }}，年龄：{{ age }}"
```

### 多变量组合

```yaml
template: "{{ greeting }} {{ name }}，欢迎来到 {{ system }}"
variables:
- variable: "greeting"
  value_selector: ["env", "greeting"]
- variable: "name"
  value_selector: ["env", "username"]
- variable: "system"
  value_selector: ["env", "system_name"]
```

### 换行和格式化

模板支持换行和格式化文本：

```yaml
template: |
  姓名：{{ name }}
  部门：{{ department }}
  职位：{{ position }}
```

## 使用场景

### 场景一：简单文本拼接

将多个变量拼接成一段文本。

```yaml
- data:
    template: "{{ firstName }} {{ lastName }}"
    variables:
    - variable: "firstName"
      value_selector: ["env", "firstName"]
    - variable: "lastName"
      value_selector: ["env", "lastName"]
    type: "template-transform"
    title: "拼接姓名"
  id: "1768547930086"
  type: "custom"
```

**输入**：
- firstName: "张"
- lastName: "三"

**输出**：
```
张 三
```

### 场景二：格式化用户信息

将用户信息格式化为可读文本。

```yaml
- data:
    template: |
      用户信息：
      姓名：{{ name }}
      部门：{{ dept }}
      邮箱：{{ email }}
      电话：{{ phone }}
    variables:
    - variable: "name"
      value_selector: ["1768547922695", "userName"]
    - variable: "dept"
      value_selector: ["1768547922695", "deptName"]
    - variable: "email"
      value_selector: ["1768547922695", "email"]
    - variable: "phone"
      value_selector: ["1768547922695", "phone"]
    type: "template-transform"
    title: "格式化用户信息"
  id: "1768547930087"
  type: "custom"
```

**输出示例**：
```
用户信息：
姓名：张三
部门：技术部
邮箱：zhangsan@example.com
电话：13800138000
```

### 场景三：生成 JSON 字符串

通过模板生成 JSON 格式的字符串。

```yaml
- data:
    template: '{"userId":"{{ userId }}","userName":"{{ userName }}","status":"{{ status }}"}'
    variables:
    - variable: "userId"
      value_selector: ["env", "userId"]
    - variable: "userName"
      value_selector: ["env", "userName"]
    - variable: "status"
      value_selector: ["env", "status"]
    type: "template-transform"
    title: "生成JSON"
  id: "1768547930088"
  type: "custom"
```

**输出示例**：
```json
{"userId":"1001","userName":"张三","status":"active"}
```

### 场景四：URL 拼接

拼接 URL 地址和参数。

```yaml
- data:
    template: "{{ baseUrl }}/api/users/{{ userId }}?token={{ token }}"
    variables:
    - variable: "baseUrl"
      value_selector: ["env", "apiBaseUrl"]
    - variable: "userId"
      value_selector: ["1768547922695", "id"]
    - variable: "token"
      value_selector: ["env", "accessToken"]
    type: "template-transform"
    title: "拼接URL"
  id: "1768547930089"
  type: "custom"
```

**输出示例**：
```
http://api.example.com/api/users/1001?token=abc123xyz
```

生成通知消息文本。

```yaml
- data:

      尊敬的 {{ userName }}：
      您的 {{ taskType }} 任务已完成。
      任务编号：{{ taskId }}
      完成时间：{{ completeTime }}
      
      感谢您的使用！
    variables:
    - variable: "userName"

    - variable: "taskType"

```

### 场景六：SQL 语句拼接

```yaml

```

## 结果输出

模板转换节点的输出结果可以通过以下方式引用：

```yaml
# 格式
{{#节点ID.output#}}

# 示例
{{#1768547930086.output#}}
```

输出结果为字符串类型，包含模板渲染后的最终文本。

## 最佳实践

### 1. 模板设计

- **清晰简洁**：模板内容应清晰易读，避免过于复杂
- **语义化变量名**：使用有意义的变量名，如 `userName` 而不是 `arg1`
- **格式规范**：保持一致的格式和缩进
- **注释说明**：在节点标题或描述中说明模板用途

### 2. 变量配置

- **变量完整性**：确保模板中的所有变量都已在 variables 中配置
- **数据源正确**：确认 value_selector 指向正确的数据源
- **类型一致**：确保引用的变量类型符合预期

### 3. 性能优化

- **避免过大模板**：大量文本拼接可以考虑使用代码节点
- **减少变量数量**：合理组织数据结构，避免过多变量
- **缓存常用模板**：对于常用模板可以保存为可复用配置

### 4. 安全考虑

- **转义特殊字符**：在生成 JSON、SQL 等格式时注意特殊字符转义
- **验证输入数据**：在模板转换前验证输入变量的合法性
- **避免敏感信息**：不要在模板中硬编码敏感信息

## 常见问题

### 1. 变量未输出或显示 undefined

**原因**：
- 变量名拼写错误
- value_selector 配置错误
- 数据源不存在或为空

**解决**：
```yaml
# 检查变量名是否与 template 中一致
template: "{{ userName }}"
variables:
- variable: "userName"  # 必须完全匹配

# 检查 value_selector 路径是否正确
value_selector:
- "1768547922695"  # 节点ID
- "output.userName"  # 输出字段路径
```

### 2. 输出结果包含 {{ }}

**原因**：
- 变量未在 variables 中配置
- 变量名不匹配

**解决**：
确保模板中的每个变量都在 variables 列表中定义：

```yaml
template: "{{ name }} - {{ age }}"
variables:
- variable: "name"
  value_selector: ["env", "userName"]
- variable: "age"  # 不要遗漏任何变量
  value_selector: ["env", "userAge"]
```

### 3. 换行符不生效

**原因**：
- 使用了单行字符串而非多行字符串

**解决**：
使用 YAML 的多行字符串语法：

```yaml
# 使用 | 保留换行
template: |
  第一行
  第二行
  第三行

# 或使用 > 将多行折叠为一行（保留最后换行）
template: >
  这是一段
  很长的文本
```

### 4. JSON 格式错误

**原因**：
- 变量值包含未转义的引号
- JSON 格式不正确

**解决**：
```yaml
# 注意引号的转义
template: '{"name":"{{ name }}","desc":"{{ desc }}"}'

# 或考虑使用代码节点构建 JSON
# 在代码节点中使用 JsonOutput.toJson() 方法
```

### 5. 特殊字符显示异常

**原因**：
- 字符编码问题
- 特殊字符未正确处理

**解决**：
- 确保所有文本使用 UTF-8 编码
- 对特殊字符进行转义或使用代码节点处理

## 注意事项

1. ⚠️ 模板中的变量名必须与 variables 配置中的 variable 字段完全一致
2. ⚠️ value_selector 路径必须准确指向数据源，否则变量为空
3. ⚠️ 生成 JSON、XML 等结构化数据时注意格式正确性
4. ⚠️ 不要在模板中硬编码敏感信息（密码、密钥等）
5. ⚠️ 大量文本处理建议使用代码节点而非模板转换
