# 通用触发节点文档

## 1. 节点概述

触发节点是系统中的事件触发入口，相当于在代码中进行埋点触发。当特定事件发生时，触发节点会启动相应的动作流。该节点支持多种业务场景的事件触发配置。

## 2. 节点结构模板

```yaml
---

  use_icon_as_answer_icon: false
  icon: ""
  name: "触发节点"
  description: ""
  icon_background: ""

  features:
    text_to_speech:
      voice: ""
      language: ""
      enabled: false
    speech_to_text:
      enabled: false
    suggested_questions: []
    retriever_resource:
      enabled: true
    opening_statement: ""
    suggested_questions_after_answer:
      enabled: false
    sensitive_word_avoidance:
      enabled: false
    file_upload:
      image:
        number_limits: 3
        transfer_methods:
        - "local_file"
        - "remote_url"
        enabled: false
      allowed_file_types:
      - "image"
      number_limits: 3
      allowed_file_extensions:
      - ".JPG"
      - ".JPEG"
      - ".PNG"
      - ".GIF"
      - ".WEBP"

      - "local_file"
      - "remote_url"
      enabled: false
  environment_variables: []
  conversation_variables: []
  graph:
    nodes:
    - data:
        variables: []
        actionData:
          panelPath: "common/biz-trigger"

          formData:
          - name: "triggerApp"

            valueMode: "fixed"
            value: []
          category: "biz_event"
          panelType: ""

      x: -1
      y: -1
```

## 3. 核心配置参数说明

### 3.1 动作配置（actionData）

| 参数 | 类型 | 必填 | 默认值 | 描述 | 示例值 |
|------|------|------|--------|------|--------|
| panelPath | string | 是 | common/biz-trigger | 动作面板路径 | common/biz-trigger |
| action | string | 是 | - | 动作类型标识符 | answerque, OuSupplyPolicyInitEvent |
| category | string | 是 | biz_event | 动作分类 | biz_event |
| panelType | string | 否 | 空 | 面板类型 | - |

### 3.2 表单数据（formData）

#### 3.2.1 触发应用（triggerApp）

| 参数 | 类型 | 必填 | 描述 | 示例值 |
|------|------|------|------|--------|
| valueMode | string | 是 | 值模式 | fixed |
| value.name | string | 是 | 应用名称 | twhf, tyrz |
| value.label | string | 是 | 应用显示标签 | 提问回复, 统一认证 |

#### 3.2.2 触发事件（triggerEvent）

| 参数 | 类型 | 必填 | 描述 | 示例值 |
|------|------|------|------|--------|
| valueMode | string | 是 | 值模式 | fixed |
| value[0].opportunityType | array | 否 | 触发时机列表 | ["before"], [] |
| value[0].eventText | string | 是 | 事件描述文本 | 回复, 部门供应策略初始化事件 |
| value[0].event | string | 是 | 事件标识符 | answer-add, OuSupplyPolicyInitEvent |
| value[0].group | string | 是 | 事件分组 | answerque, OuSupplyPolicyInitEvent |

#### 3.2.3 触发流程（triggerFlow）

| 参数 | 类型 | 必填 | 描述 | 示例值 |
|------|------|------|------|--------|
| valueMode | string | 是 | 值模式 | fixed |
| value | array | 是 | 流程列表 | [] |

### 3.3 节点基本配置

| 参数 | 类型 | 必填 | 默认值 | 描述 | 示例值 |
|------|------|------|--------|------|--------|
| type | string | 是 | start | 节点类型 | start |
| title | string | 是 | - | 节点显示标题 | 提问回复, 部门供应策略初始化事件 |
| isAction | boolean | 是 | true | 是否为动作节点 | true |
| selected | boolean | 是 | true | 是否选中 | true |
| desc | string | 否 | 空 | 节点描述 | - |

### 3.4 节点布局配置

| 参数 | 类型 | 必填 | 默认值 | 描述 | 示例值 |
|------|------|------|--------|------|--------|

| width | number | 是 | 244 | 节点宽度 | 244 |
| sourcePosition | string | 是 | right | 源连接点位置 | right |
| positionAbsolute.x | number | 是 | 80 | 绝对位置X坐标 | 80 |
| positionAbsolute.y | number | 是 | 282 | 绝对位置Y坐标 | 282 |
| id | string | 是 | - | 节点唯一ID | 1768553812401 |
| position.x | number | 是 | 80 | 相对位置X坐标 | 80 |

## 5. 配置差异分析

| 配置项 | 示例1（提问回复） | 示例2（部门供应策略初始化） | 说明 |
|--------|------------------|----------------------------|------|
| action | answerque | OuSupplyPolicyInitEvent | 动作类型标识，根据业务场景自定义 |
| triggerApp.name | twhf | tyrz | 触发应用名称，与系统中注册的应用一致 |
| triggerApp.label | 提问回复 | 统一认证 | 应用显示标签，用于UI展示 |
| triggerEvent.opportunityType | ["before"] | [] | 触发时机，before表示事件前触发，after表示事件后触发 |
| triggerEvent.eventText | 回复 | 部门供应策略初始化事件 | 事件描述文本，用于UI展示 |
| triggerEvent.event | answer-add | OuSupplyPolicyInitEvent | 事件标识符，与系统埋点事件一致 |
| triggerEvent.group | answerque | OuSupplyPolicyInitEvent | 事件分组，用于事件分类管理 |

## 6. 配置注意事项

### 6.1 必须配置项

1. **actionData.action**：动作类型标识符，必须与系统中定义的动作类型一致
2. **triggerApp**：触发应用信息，包含name和label两个必填字段
3. **triggerEvent**：触发事件信息，包含eventText、event和group三个必填字段
4. **node.title**：节点显示标题，用于流程画布展示

### 6.2 可选配置项

1. **triggerEvent.opportunityType**：触发时机，可设置为["before"]、["after"]或空数组
2. **app.id**：节点ID，可使用默认值或自定义UUID
3. **node.id**：节点在画布中的唯一ID，通常由系统自动生成

### 6.3 固定配置项

1. **app.mode**：固定为"actflow"
2. **app.name**：固定为"触发节点"
3. **actionData.panelPath**：固定为"common/biz-trigger"
4. **actionData.category**：固定为"biz_event"
5. **data.type**：固定为"start"（开始节点）
6. **isAction**：固定为true
7. **selected**：固定为true
8. **targetPosition**：固定为"left"
9. **sourcePosition**：固定为"right"
10. **width**：固定为244
11. **height**：固定为53

## 7. 系统事件触发机制

### 7.1 事件触发流程

1. 系统代码中埋点触发指定事件
2. 事件系统根据event标识符匹配对应的触发节点
3. 验证触发时机（如果配置了opportunityType）
4. 启动对应的动作流执行后续节点

### 7.2 事件命名规范

- 事件标识符（event）：使用驼峰式命名或下划线命名，如answer-add、OuSupplyPolicyInitEvent
- 事件分组（group）：通常与事件标识符或动作类型保持一致
- 事件描述（eventText）：使用清晰的中文描述，便于UI展示

## 8. 应用场景

### 8.1 业务事件触发

- **用户操作事件**：用户登录、注册、提交表单等

- **系统状态事件**：定时任务执行、数据同步完成等

## 9. 扩展配置

### 9.1 启用文件上传功能

如需启用文件上传功能，可修改以下配置：

```yaml

  image:
    number_limits: 5  # 修改上传数量限制
    transfer_methods:
    - "local_file"
    - "remote_url"
    enabled: true  # 启用图片上传

  - "image"
  - "document"  # 添加文档类型
  number_limits: 5  # 修改总上传数量限制

  - ".JPG"
  - ".JPEG"
  - ".PNG"
  - ".GIF"
  - ".WEBP"
  - ".SVG"

  - ".DOCX"
  - ".PDF"
  allowed_file_upload_methods:
  - "local_file"
  - "remote_url"
```

如需启用敏感词过滤功能，可修改以下配置：

```yaml
sensitive_word_avoidance:
  enabled: true  # 启用敏感词过滤
```

## 10. 故障排查

### 10.1 事件未触发

**可能原因：**
1. 事件标识符（event）与系统埋点不一致
2. 触发应用名称（triggerApp.name）与系统注册的应用名称不一致
3. 触发时机（opportunityType）配置错误
4. 动作流未正确发布或启用

**解决方案：**
1. 检查事件标识符是否与系统埋点完全一致
2. 确认触发应用名称是否在系统中正确注册
3. 验证触发时机配置是否符合业务需求
4. 检查动作流状态是否为已发布和启用

### 10.2 节点配置错误

**可能原因：**
1. 缺少必填配置项
2. 配置值格式错误
3. 配置值与系统定义不匹配

**解决方案：**
1. 检查是否包含所有必须配置项
2. 验证配置值格式是否符合YAML语法规范
3. 确认配置值是否与系统定义的动作类型、事件标识符等一致

## 11. 最佳实践

### 11.1 事件命名规范

1. 使用清晰、简洁的事件描述
2. 事件标识符与业务场景保持一致
3. 事件分组便于事件分类管理

### 11.2 触发时机选择

1. **before**：适合在事件执行前进行验证、权限检查等操作
2. **after**：适合在事件执行后进行通知、日志记录等操作
3. 空数组：适合不需要特定时机的事件触发

### 11.3 配置文件管理

1. 使用版本控制管理配置文件
2. 对不同业务场景的配置文件进行分类存放
3. 定期备份配置文件，防止配置丢失

## 12. 总结

触发节点是系统事件驱动架构的核心组件，通过灵活的配置可以支持多种业务场景的事件触发需求。配置时需要注意以下几点：

1. 确保必填配置项完整且正确
2. 事件标识符与系统埋点保持一致
3. 根据业务需求选择合适的触发时机
4. 遵循系统配置规范和命名约定

通过正确配置触发节点，可以实现业务流程的自动化执行，提高系统的灵活性和可扩展性。
