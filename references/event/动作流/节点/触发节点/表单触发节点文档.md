# 表单触发节点文档

## 1. 节点概述

表单触发节点是系统中基于表单操作的事件触发入口，当表单发生特定操作（如初始化、保存、删除等）时，触发节点会启动相应的动作流。表单触发节点与业务表单紧密集成，支持多种表单操作事件的监听和响应。

## 3. 核心配置参数说明

### 3.1 动作配置（actionData）

| 参数 | 类型 | 必填 | 默认值 | 描述 | 示例值 |
|------|------|------|--------|------|--------|
| panelPath | string | 是 | frame-sform/frame-sform | 表单面板路径 | frame-sform/frame-sform |
| action | string | 是 | sformTrigger | 动作类型标识符 | sformTrigger |
| category | string | 是 | sformTrigger | 动作分类 | sformTrigger |
| panelType | string | 否 | 空 | 面板类型 | - |

### 3.2 表单数据（formData）

| formModel | string | 是 | 表单模型ID | 9F3ED654-FE76-4287-A229-D5233D7D0401, 3A087AD0-FE29-4DA0-A56E-5910ADAF6279 |

#### 3.2.2 表单关联信息（可选）

| 参数 | 类型 | 必填 | 描述 | 示例值 |
|------|------|------|------|--------|
| tableName | string | 否 | 表名 | - |
| sqlTableName | string | 否 | SQL表名 | - |
| tableId | string | 否 | 表ID | - |
| versionGuid | string | 否 | 版本GUID | - |

| 参数 | 类型 | 必填 | 描述 | 示例值 |

| valueMode | string | 是 | 值模式 | fixed |

| 配置项 | 示例1（采购任务分派） | 示例2（LC测试表单） | 说明 |

| formModel | 9F3ED654-FE76-4287-A229-D5233D7D0401 | 3A087AD0-FE29-4DA0-A56E-5910ADAF6279 | 表单模型ID，与系统中表单模型一致 |

| triggerEvent数量 | 2个事件（初始化、删除） | 1个事件（保存） | 可配置多个触发事件 |

| triggerEvent[0].event | pageInit | save | 事件标识符，对应表单的具体操作 |

## 6. 支持的表单事件

| 事件标识符 | 事件描述 | 触发时机 | 应用场景 |
|------------|----------|----------|----------|
| pageInit | 初始化 | before | 表单加载前执行初始化逻辑 |
| save | 保存 | after | 表单数据保存后执行后续操作 |
| delete | 删除 | after | 表单数据删除后执行后续操作 |

## 7. 配置注意事项

### 7.1 必须配置项

1. **formData.formName**：表单显示名称，与实际业务表单名称一致
2. **formData.formModel**：表单模型ID，必须与系统中注册的表单模型ID一致
3. **formData.formGuid**：表单GUID，必须与系统中表单的GUID一致
4. **formData.triggerEvent**：触发事件配置，包含opportunityType、eventText和event三个必填字段

### 7.2 可选配置项

1. **formData.tableName**：表单关联的表名（如果有）
2. **formData.sqlTableName**：表单关联的SQL表名（如果有）
3. **formData.tableId**：表单关联的表ID（如果有）
4. **formData.versionGuid**：表单版本GUID（如果需要版本控制）

### 7.3 固定配置项

1. **app.mode**：固定为"actflow"
2. **app.name**：固定为"表单触发"
3. **app.id**：固定为"5945fbac-388c-4600-a56d-e2c5819bac92"
4. **actionData.panelPath**：固定为"frame-sform/frame-sform"
5. **actionData.action**：固定为"sformTrigger"
6. **actionData.category**：固定为"sformTrigger"
7. **data.type**：固定为"start"（开始节点）
8. **isAction**：固定为true
9. **selected**：固定为true
10. **targetPosition**：固定为"left"
11. **sourcePosition**：固定为"right"
12. **width**：固定为244
13. **height**：固定为53

## 8. 表单事件触发机制

### 8.1 事件触发流程

1. 用户访问或操作业务表单
2. 表单系统执行相应操作（如初始化、保存、删除）
3. 表单系统触发对应的事件（如pageInit、save、delete）
4. 事件系统根据事件标识符匹配对应的表单触发节点
5. 验证触发时机（before/after）
6. 启动对应的动作流执行后续节点

### 8.2 事件触发时机说明

| 触发时机 | 触发时间 | 应用场景 |
|----------|----------|----------|
| before | 表单操作执行前 | 数据验证、权限检查、参数预处理 |
| after | 表单操作执行后 | 数据同步、通知发送、日志记录 |

## 9. 应用场景

## 10. 扩展配置

### 10.1 配置多个触发事件

一个表单触发节点可以配置多个触发事件，例如同时监听表单的保存和删除事件：

```yaml
triggerEvent:
  valueMode: "fixed"
  value:
  - opportunityType: ["after"]
    eventText: "保存"
    event: "save"
    group: "sformTrigger"
  - opportunityType: ["after"]
    eventText: "删除"
    event: "delete"
    group: "sformTrigger"
```

## 11. 故障排查

### 11.1 表单事件未触发

**可能原因：**
1. 表单模型ID（formModel）与系统中注册的表单模型ID不一致
2. 表单GUID（formGuid）与系统中表单的GUID不一致
3. 事件标识符（event）与表单实际支持的事件不匹配
4. 触发时机（opportunityType）配置错误
5. 动作流未正确发布或启用

**解决方案：**
1. 检查表单模型ID是否与系统中注册的表单模型ID完全一致
2. 确认表单GUID是否与系统中表单的GUID一致
3. 验证事件标识符是否为表单支持的事件类型
4. 检查触发时机配置是否符合业务需求
5. 确认动作流状态是否为已发布和启用

### 11.2 节点配置错误

**可能原因：**
1. 缺少必填配置项
2. 配置值格式错误
3. 配置值与系统定义不匹配

**解决方案：**
1. 检查是否包含所有必须配置项（formName、formModel、formGuid、triggerEvent）
2. 验证配置值格式是否符合YAML语法规范
3. 确认配置值是否与系统定义的表单信息一致

## 12. 最佳实践

### 12.1 表单事件配置

1. **合理选择触发时机**：根据业务需求选择合适的触发时机（before/after）
2. **事件分类管理**：对不同类型的表单事件进行分类管理，便于维护
3. **事件描述清晰**：使用清晰、简洁的事件描述文本，便于理解和维护

### 12.2 表单关联信息配置

1. **按需配置**：仅配置必要的表单关联信息，避免冗余配置
2. **保持一致**：确保表单关联信息与系统中实际配置一致

### 12.3 配置文件管理

1. **版本控制**：使用版本控制管理配置文件
2. **命名规范**：为配置文件使用清晰的命名规范，包含表单名称和功能描述
3. **定期备份**：定期备份配置文件，防止配置丢失

## 13. 总结

表单触发节点是系统中基于表单操作的重要事件触发机制，通过灵活配置可以实现对表单各种操作的监听和响应。正确配置表单触发节点需要注意以下几点：

1. 确保表单基本信息（formName、formModel、formGuid）与系统中实际表单一致
2. 根据业务需求选择合适的触发事件和触发时机
3. 可以配置多个触发事件来监听表单的不同操作
4. 定期检查和维护配置文件，确保配置的准确性和有效性

通过合理使用表单触发节点，可以实现表单操作与业务流程的紧密集成，提高系统的自动化程度和业务处理效率。
