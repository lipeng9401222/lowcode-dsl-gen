# 定时任务触发节点

## 概述

定时任务触发节点用于按照设定的时间规则自动触发工作流执行,适用于需要定期执行的业务场景,如数据同步、定时报表生成、周期性任务等。

## 节点配置

### 基本结构

```yaml
- data:
    variables: []
    actionData:
      panelPath: "simple-timer/simple-timer"
      action: "simpleTimer"
      formData:
      - name: "frequency"
        valueMode: "fixed"
        value: 5  # 频率类型
      - name: "startTime"
        valueMode: "fixed"
        value: "2025-12-17 02:33"
      - name: "frequencyData"
        valueMode: "fixed"
        value:
          monthWeek: []
          monthCheck: ["last"]
          monthWeekDay: []
          monthType: 1
          lastWeek: []
      - name: "endTime"
        valueMode: "fixed"
        value: "2026-12-24 00:00"
      - name: "month"
        valueMode: "fixed"
        value: []
      category: "timerOrSchedule"
      panelType: ""
    type: "start"
    title: "定时或指定时间触发"
    isAction: true
    selected: false
    desc: ""
  id: "1766021575240"
  type: "custom"
```

## 配置项详解

### 1. frequency (频率类型)

定义任务执行的周期频率。

| 值 | 含义 | 说明 |
|---|------|------|
| 1 | 按天 | 每天执行 |
| 2 | 按周 | 每周执行 |
| 3 | 按月(按日期) | 每月指定日期执行 |
| 4 | 按月(按周) | 每月指定周次和星期执行 |
| 5 | 按月(最后一天) | 每月最后一天执行 |
| 6 | 按年 | 每年执行 |

**示例**：

```yaml
- name: "frequency"
  valueMode: "fixed"
  value: 5  # 按月最后一天执行
```

### 2. startTime (开始时间)

任务首次执行的时间。

**格式**：`YYYY-MM-DD HH:mm`

**示例**：

```yaml
- name: "startTime"
  valueMode: "fixed"
  value: "2025-12-17 02:33"  # 从2025年12月17日凌晨2:33开始
```

### 3. endTime (结束时间)

任务停止执行的时间。

**格式**：`YYYY-MM-DD HH:mm`

**示例**：

```yaml
- name: "endTime"
  valueMode: "fixed"
  value: "2026-12-24 00:00"  # 到2026年12月24日停止
```

### 4. frequencyData (频率详细配置)

根据不同的频率类型,配置具体的执行规则。

#### 按天执行

```yaml
- name: "frequencyData"
  valueMode: "fixed"
  value:
    dayType: 1  # 1=每天, 2=每隔N天
    dayInterval: 1  # 间隔天数(当dayType=2时)
```

#### 按周执行

```yaml
- name: "frequencyData"
  valueMode: "fixed"
  value:
    weekDays: [1, 3, 5]  # 1-7表示周一到周日
```

#### 按月执行(按日期)

```yaml
- name: "frequencyData"
  valueMode: "fixed"
  value:
    monthType: 1  # 1=按日期
    monthCheck: [1, 15, 28]  # 每月的1号、15号、28号执行
```

#### 按月执行(按周)

```yaml
- name: "frequencyData"
  valueMode: "fixed"
  value:
    monthType: 2  # 2=按周
    monthWeek: [1, 3]  # 第1周和第3周
    monthWeekDay: [1, 5]  # 周一和周五
```

#### 按月执行(最后一天)

```yaml
- name: "frequencyData"
  valueMode: "fixed"
  value:
    monthType: 1
    monthCheck: ["last"]  # 每月最后一天
```

#### 按年执行

```yaml
- name: "frequencyData"
  valueMode: "fixed"
  value:
    yearMonths: [1, 6, 12]  # 1月、6月、12月
    yearDays: [1, 15]  # 每月的1号和15号
```

### 5. month (月份配置)

当频率为按年时,指定执行的月份。

**示例**：

```yaml
- name: "month"
  valueMode: "fixed"
  value: [3, 6, 9, 12]  # 每季度末月执行
```

## 常见问题

### 1. 如何设置每隔N天执行？

```yaml
- name: "frequency"
  valueMode: "fixed"
  value: 1  # 按天
- name: "frequencyData"
  valueMode: "fixed"
  value:
    dayType: 2  # 每隔N天
    dayInterval: 3  # 每隔3天
```

### 2. 如何设置每月第一个工作日执行？

```yaml
- name: "frequency"
  valueMode: "fixed"
  value: 4  # 按月(按周)
- name: "frequencyData"
  valueMode: "fixed"
  value:
    monthType: 2
    monthWeek: [1]  # 第1周
    monthWeekDay: [1]  # 周一
```

### 3. 时区如何设置？

定时任务使用服务器所在时区,请确保服务器时间设置正确。

### 4. 如何测试定时任务？

可以将开始时间设置为当前时间后的几分钟,观察任务是否按时触发。

## 最佳实践

1. **避免高峰期执行**：将大数据量任务安排在业务低峰期(如凌晨)执行
2. **合理设置执行时间**：预留足够的执行时间窗口,避免任务冲突
3. **监控任务执行**：配置适当的日志和告警,及时发现任务执行异常
4. **设置结束时间**：对于临时性定时任务,建议设置明确的结束时间
5. **错误处理**：在工作流中添加适当的错误处理逻辑

## 注意事项

1. ⚠️ 定时任务一旦启动会自动执行,请谨慎配置
2. ⚠️ 确保工作流的执行时间小于任务间隔,避免任务堆积
3. ⚠️ 修改定时配置后需要重启工作流才能生效
4. ⚠️ 建议在测试环境充分验证后再部署到生产环境
