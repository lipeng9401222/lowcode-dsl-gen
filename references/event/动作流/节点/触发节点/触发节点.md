# 触发节点文档

## 1. 节点概述

触发节点是系统中的事件触发入口，相当于在代码中进行埋点触发。当特定事件发生时，触发节点会启动相应的动作流。

## 2. 节点配置

### 2.1 基本信息

| 属性 | 值 | 描述 |
|------|-----|------|
| 应用模式 | actflow | 节点运行模式 |
| 节点名称 | 触发节点 | 节点显示名称 |
| 节点ID | d7e616b9-512b-42aa-a156-cc5ca3ebdba1 | 节点唯一标识符 |
| 图标 | 空 | 节点显示图标 |
| 图标背景 | 空 | 节点图标背景色 |

### 2.2 工作流特性

| 特性 | 状态 | 配置 |
|------|------|------|
| 文本转语音 | 禁用 | - |
| 语音转文本 | 禁用 | - |
| 推荐问题 | 禁用 | - |
| 检索资源 | 启用 | - |
| 开场白 | 空 | - |
| 回答后推荐问题 | 禁用 | - |
| 敏感词避免 | 禁用 | - |
| 文件上传 | 禁用 | 支持图片类型上传 |

### 2.3 文件上传配置

**允许的文件类型：**
- 图片类型（JPG, JPEG, PNG, GIF, WEBP, SVG）
- 上传数量限制：3个文件
- 上传方式：本地文件、远程URL

## 5. 事件触发说明

### 5.1 可用事件

| 事件组 | 事件标识 | 事件文本 | 时机类型 | 描述 |
|--------|----------|----------|----------|------|
| answerque | answer-add | 回复 | before | 在添加回复前触发 |

### 5.2 触发条件

当系统中发生"添加回复"操作时，会触发此节点。触发时机为操作执行前（before）。

### 5.3 触发流程

1. 系统检测到"添加回复"操作
2. 验证触发条件（时机：before）
3. 调用触发节点对应的动作流
4. 执行动作流中的后续节点

## 6. 应用场景

- 在用户添加回复前进行数据验证
- 在回复添加前记录操作日志
- 在回复添加前执行权限检查
- 在回复添加前触发通知机制

## 7. 注意事项

1. 触发节点必须作为动作流的起始节点
2. 确保触发事件与系统中的实际埋点事件一致
3. 可根据业务需求调整触发时机（before/after）
4. 多个动作流可使用相同的触发事件，但需注意执行顺序

## 8. 扩展配置

触发节点支持通过修改YAML配置文件进行扩展：

- 添加新的触发事件
- 调整文件上传限制
- 启用/禁用工作流特性
- 修改节点显示属性
