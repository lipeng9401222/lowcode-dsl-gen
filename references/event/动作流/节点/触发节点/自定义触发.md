# 自定义触发节点 (start)

## 概述

自定义触发节点是工作流的入口节点,用于接收来自其他动作流的调用触发。它允许一个动作流作为可复用的子流程,供其他动作流调用执行,实现动作流之间的编排和复用。

## 节点配置

### 基本结构

```yaml
data:
  variables: []
  actionData:
    panelPath: "custom-trigger/custom-trigger"  # 面板路径
    action: "customTrigger"  # 动作名称
    formData:
    - name: "parameters"  # 参数配置
      valueMode: "fixed"
      value:
      - UID: "VGuUbOtdQbGjFFB85HOzJArUWXnBZFZY"
        showName: "用户标识"
        defaultValue:
          valueMode: "fixed"
          value: ""
        name: "userguid"
        ParentTaskUID: -1
        type: "STRING"
        required: true
        desc: "用户标识"
      - UID: "3jLQWmO4lfjYXW1vDgpfnafRVyHNI2wr"
        showName: "用户姓名"
        defaultValue:
          valueMode: "fixed"
          value: ""
        name: "username"
        ParentTaskUID: -1
        type: "STRING"
        required: true
        desc: "用户姓名"
    category: "customTrigger"  # 类别
    panelType: ""
  type: "start"  # 节点类型
  title: "自定义触发"  # 节点标题
  isAction: true  # 是否为动作节点
  selected: true
  desc: ""
id: "1768553540467"
type: "custom"
```

## 字段说明

### actionData 配置

| 字段 | 类型 | 说明 |
|------|------|------|
| panelPath | string | 面板路径,固定为 "custom-trigger/custom-trigger" |
| action | string | 动作名称,固定为 "customTrigger" |
| category | string | 类别,固定为 "customTrigger" |

### formData 配置

#### parameters (参数定义)

自定义触发节点的核心配置,定义了该动作流对外暴露的输入参数。

```yaml
- name: "parameters"
  valueMode: "fixed"
  value: []  # 参数列表
```

## 参数配置详解

### 参数字段结构

每个参数包含以下字段:

| 字段 | 类型 | 说明 |
|------|------|------|
| UID | string | 参数唯一标识符 |
| name | string | 参数名称(用于变量引用) |
| showName | string | 参数显示名称 |
| type | string | 参数数据类型:STRING、NUMBER、BOOLEAN、OBJECT、ARRAY |
| required | boolean | 是否必填 |
| defaultValue | object | 默认值配置 |
| desc | string | 参数描述 |
| ParentTaskUID | number | 父任务ID,-1 表示根级参数 |

### 参数类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| STRING | 字符串类型 | "张三"、"user001" |
| NUMBER | 数字类型 | 100、3.14 |
| BOOLEAN | 布尔类型 | true、false |
| OBJECT | 对象类型 | {"name": "张三", "age": 25} |
| ARRAY | 数组类型 | [1, 2, 3] 或 [{"id": 1}, {"id": 2}] |

## 变量引用

自定义触发节点接收的参数可以在当前工作流的后续节点中引用:

### 引用参数值

```yaml
# 格式
{{#节点ID.parameters.参数名#}}

# 示例
{{#1768553540467.parameters.userguid#}}
{{#1768553540467.parameters.username#}}
```

### 在上下文中引用

```yaml
# 格式
s{节点ID}Context.data.parameters.{参数名}

# 示例
s1768553540467Context.data.parameters.userguid
s1768553540467Context.data.parameters.username
```

## 调用方式

### 从其他动作流调用

在主动作流中,使用"调用子流程"节点来调用配置了自定义触发的动作流:

```yaml
# 调用子流程节点配置
actionData:
  action: "callWorkflow"
  formData:
  - name: "workflowId"
    value: "70386935-5f0d-4ec9-aa8b-1272d783c906"  # 目标动作流ID
  - name: "parameters"
    value:
      userguid: "{{#上游节点ID.output.userId#}}"
      username: "{{#上游节点ID.output.userName#}}"
```

### 参数传递流程

1. **主动作流** → 调用子流程节点 → 传入参数
2. **子动作流** → 自定义触发节点 → 接收参数
3. **子动作流** → 执行业务逻辑 → 返回结果
4. **主动作流** → 接收返回结果 → 继续执行

## 使用场景

### 场景一:用户信息处理子流程

创建一个可复用的用户信息处理流程:

```yaml
parameters:
- UID: "VGuUbOtdQbGjFFB85HOzJArUWXnBZFZY"
  showName: "用户标识"
  name: "userguid"
  type: "STRING"
  required: true
  desc: "用户标识"
- UID: "3jLQWmO4lfjYXW1vDgpfnafRVyHNI2wr"
  showName: "用户姓名"
  name: "username"
  type: "STRING"
  required: true
  desc: "用户姓名"
```

**应用**: 多个主流程需要进行用户信息验证、补全、格式化等操作时,可以统一调用此子流程。

### 场景二:数据校验子流程

```yaml
parameters:

  showName: "待校验数据"
  name: "data"
  type: "OBJECT"
  required: true

  showName: "校验规则"
  name: "rules"
  type: "ARRAY"
  required: true
  desc: "校验规则列表"
```

**应用**: 统一的数据校验逻辑,多个入口流程调用进行数据合规性检查。

### 场景三:通知发送子流程


## 最佳实践

1. **明确参数定义**: 清晰定义所有输入参数及其类型、是否必填、描述等
2. **合理使用必填项**: 对关键参数设置 required: true,避免空值导致流程异常
3. **提供默认值**: 为非必填参数提供合理的默认值,提高流程健壮性
4. **参数命名规范**: 使用有意义的参数名称,遵循驼峰命名法
5. **添加参数描述**: 在 desc 字段详细说明参数用途,便于其他开发者理解
6. **复用性设计**: 将通用的业务逻辑封装为子流程,避免重复配置
7. **参数验证**: 在流程内部添加参数验证逻辑,确保数据合法性
8. **文档维护**: 为自定义触发的动作流编写使用文档,说明参数含义和调用方式

## 注意事项

1. ⚠️ 自定义触发节点必须是工作流的第一个节点
2. ⚠️ 参数的 UID 必须在整个工作流中唯一
3. ⚠️ ParentTaskUID 用于构建嵌套参数结构,根级参数使用 -1
4. ⚠️ 修改参数定义后,需要同步更新所有调用该动作流的主流程
5. ⚠️ 建议为每个参数添加清晰的描述和类型定义,避免调用时出错
6. ⚠️ 参数类型要与实际传入的数据类型匹配,否则可能导致执行错误
7. ⚠️ 动作流之间不要形成循环调用,避免死循环

## 与其他节点的区别

### vs Webhook 触发

| 特性 | 自定义触发 | Webhook 触发 |
|------|-----------|-------------|
| 触发方式 | 动作流内部调用 | 外部 HTTP 请求 |
| 调用者 | 其他动作流 | 外部系统/应用 |
| 参数传递 | 直接传递对象 | HTTP 请求体/参数 |
| 网络要求 | 无需网络 | 需要网络访问 |
| 使用场景 | 流程编排、逻辑复用 | 系统集成、外部触发 |

### vs 定时任务触发

| 特性 | 自定义触发 | 定时任务触发 |
|------|-----------|-------------|
| 触发方式 | 主动调用 | 定时自动触发 |
| 执行时机 | 被调用时立即执行 | 按时间规则执行 |
| 参数来源 | 调用方传入 | 无外部参数 |
| 使用场景 | 子流程复用 | 周期性任务 |
