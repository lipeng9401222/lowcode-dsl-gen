# 结束节点 (end-vue)

## 概述

结束节点是工作流的终点，用于返回处理结果、发送通知或触发其他操作。

## 节点配置

```yaml
data:
  error_handle_mode: "terminated"  # 错误处理模式
  start_node_id: ""                # 开始节点ID
  type: "end-vue"                  # 节点类型
  title: "结束"                     # 节点标题
  actionContext: []                # 动作上下文
  output_selector: []              # 输出选择器
  iterator_selector: []            # 迭代选择器
  actionData:                      # 动作数据
    panelPath: "common/end"        # 面板路径
    action: "end"                  # 动作名称
    formData:                      # 表单数据
    - name: "trigerType"           # 触发类型
      valueMode: "fixed"           # 值模式
      value: "webhook"             # 值
    - name: "reminderType"         # 提醒类型
      valueMode: "fixed"
      value: ""
    - name: "responseParams"       # 响应参数
      valueMode: "fixed"
      value:
      - UID: "root"                # 根节点
        showName: "根节点"
        children: []
        defaultValue:
          formula:                 # 公式
            expression: "RuleParamUtil.parserParam(s1756283014816Context,\"data.result\")"
            header: ""
            text: "拼接大模型输出结果.result"
            value:
            - name: "拼接大模型输出结果.result"
              id: "90094a35-a93b-46a7-bed0-a5d1ff553a8d;s1756283014816Context.data.result"
              value: "s1756283014816Context.data.result"
          valueMode: "formula"     # 值模式
          value: ""
        name: "根节点"
        ParentTaskUID: -1
        type: "OBJECT"             # 类型
        required: true
  isAction: true                   # 是否为动作节点
  is_parallel: false               # 是否并行
  selected: false
  desc: ""
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 节点类型，固定为 "end-vue" |
| title | string | 节点显示标题 |
| actionData.action | string | 动作名称，固定为 "end" |
| error_handle_mode | string | 错误处理模式 |

## formData 配置

### 1. trigerType（触发类型）

指定工作流的触发类型，影响返回方式。

```yaml
- name: "trigerType"
  valueMode: "fixed"
  value: "webhook"          # 触发类型
```

| 值 | 说明 |
|----|------|
| `webhook` | Webhook 触发，返回 HTTP 响应 |
| `manual` | 手动触发 |
| `schedule` | 定时触发 |
| `event` | 事件触发 |

### 2. reminderType（提醒类型）

配置是否发送提醒通知。

```yaml
- name: "reminderType"
  valueMode: "fixed"
  value: ""                 # 空表示不发送提醒
```

| 值 | 说明 |
|----|------|
| `""` | 不发送提醒 |
| `email` | 发送邮件通知 |
| `sms` | 发送短信通知 |
| `push` | 推送通知 |

### 3. responseParams（响应参数）

定义返回给调用方的数据结构。

```yaml
- name: "responseParams"
  valueMode: "fixed"
  value:
  - UID: "root"              # 根节点UID
    name: "根节点"            # 字段名
    showName: "根节点"        # 显示名称
    type: "OBJECT"           # 数据类型
    required: true           # 是否必填
    ParentTaskUID: -1        # 父节点UID
    children: []             # 子字段
    defaultValue: {}         # 默认值配置
```

## 响应参数配置

### 参数结构

每个响应参数包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| UID | string | 参数唯一标识 |
| name | string | 参数名称（返回的字段名） |
| showName | string | 显示名称 |
| type | string | 数据类型 |
| required | boolean | 是否必填 |
| ParentTaskUID | number | 父参数UID，-1表示根级 |
| children | array | 子参数列表（对于OBJECT类型） |
| defaultValue | object | 默认值配置 |

### 数据类型 (type)

| 类型 | 说明 | 示例 |
|------|------|------|
| `STRING` | 字符串 | "Hello World" |
| `NUMBER` | 数字 | 123, 45.67 |
| `BOOLEAN` | 布尔值 | true, false |
| `OBJECT` | 对象 | {"key": "value"} |
| `ARRAY` | 数组 | [1, 2, 3] |

### 默认值配置

#### 固定值模式

```yaml
defaultValue:
  valueMode: "fixed"
  value: "success"
```

#### 公式模式（引用节点输出）

```yaml
defaultValue:
  valueMode: "formula"
  value: ""
  formula:
    expression: "RuleParamUtil.parserParam(s{节点ID}Context,\"data.result\")"
    header: ""
    text: "节点名称.result"
    value:
    - name: "节点名称.result"
      id: "uuid;s{节点ID}Context.data.result"
      value: "s{节点ID}Context.data.result"
```

## 配置示例

### 场景一：返回简单字符串

```yaml
responseParams:
- UID: "root"
  name: "message"
  showName: "消息"
  type: "STRING"
  required: true
  ParentTaskUID: -1
  defaultValue:
    valueMode: "fixed"
    value: "处理成功"
```

**返回结果**：
```json
{
  "message": "处理成功"
}
```

### 场景二：返回代码节点的输出

```yaml
responseParams:
- UID: "root"
  name: "result"
  showName: "处理结果"
  type: "STRING"
  required: true
  ParentTaskUID: -1
  defaultValue:
    valueMode: "formula"
    formula:
      expression: "RuleParamUtil.parserParam(s1756283014816Context,\"data.result\")"
      text: "代码节点.result"
      value:
      - value: "s1756283014816Context.data.result"
```

**返回结果**：
```json
{
  "result": "代码节点输出的内容"
}
```

### 场景三：返回复杂对象

```yaml
responseParams:
- UID: "root"
  name: "data"
  showName: "数据"
  type: "OBJECT"
  required: true

    name: "code"

    defaultValue:
      valueMode: "fixed"
      value: "1"
  - UID: "field2"
    name: "message"
    showName: "消息"
    type: "STRING"
    required: true
    ParentTaskUID: "root"
    defaultValue:
      valueMode: "fixed"
      value: "成功"
  - UID: "field3"
    name: "result"
    showName: "结果"
    type: "STRING"
    required: false

    defaultValue:

        expression: "RuleParamUtil.parserParam(s{code_node_id}Context,\"data.result\")"
        text: "代码节点.result"
```

**返回结果**：
```json
{
  "data": {
    "code": "1",
    "message": "成功",
    "result": "处理结果内容"
  }
```


## 最佳实践

### 1. 返回结构设计

- **统一格式**：使用统一的返回格式（如 code、message、data）
- **清晰字段名**：使用有意义的字段名称
- **适当嵌套**：合理组织数据结构，避免过深嵌套

### 2. 状态码规范

```yaml
# 建议的状态码规范
- code: "1" 或 "200"      # 成功
- code: "0" 或 "400"      # 失败（客户端错误）
- code: "-1" 或 "500"     # 失败（服务端错误）
```

### 3. 错误信息

- **提供详细错误信息**：在失败分支返回具体的错误原因
- **不暴露敏感信息**：错误信息不要包含系统内部信息
- **国际化支持**：考虑多语言错误消息

### 4. 数据返回

- **只返回必要数据**：避免返回过多不需要的字段
- **注意数据大小**：大数据量考虑分页或提供下载链接
- **数据格式化**：确保日期、数字等格式符合约定

### 5. 多分支场景

- **不同分支设置不同结束节点**：清晰区分成功和失败
- **统一返回格式**：即使不同分支，也保持基本格式一致
- **提供上下文信息**：返回足够信息帮助调用方判断结果

## 实际应用场景

### 场景一：Webhook API 响应

```
处理业务逻辑
    ↓
条件判断：处理是否成功
    ↓ True              ↓ False
返回成功结果        返回错误信息
```

**成功返回**：
```json
{
  "code": "1",
  "message": "处理成功",
  "data": {...}
}
```

**失败返回**：
```json
{
  "code": "0",
  "message": "处理失败",
  "error": "具体错误信息"
}
```

### 场景二：批量处理结果汇总

```
迭代批量处理
    ↓
代码节点汇总结果
    ↓
结束节点返回汇总
```

**返回示例**：
```json
{
  "total": 100,
  "success": 95,
  "failed": 5,
  "errors": [...]
}
```

### 场景三：流式处理结果

```
LLM 流式生成
    ↓
代码节点拼接
    ↓
结束节点返回完整结果
```

**返回示例**：
```json
{
  "content": "完整的生成内容",
  "token_count": 1234
}
```

## 常见问题

### 1. 返回数据为空

**原因**：引用的节点输出路径错误或节点未执行

**解决**：
- 检查引用路径是否正确
- 确认前置节点已成功执行
- 使用代码节点打印调试信息

### 2. 返回格式错误

**原因**：参数类型配置不正确

**解决**：
- 检查 type 字段配置
- 确保返回数据类型与配置一致

### 3. 返回数据过大

**原因**：返回了大量数据或大文件

**解决**：
- 只返回必要字段
- 大数据考虑分页
- 文件提供下载链接而非直接返回
