# 字段约定

本文档定义动作流 DSL 中的关键字段约定，作为唯一权威参考。

## valueMode 取值模式

每个 `formData` 项的取值方式由 `valueMode` 字段决定：

| valueMode | 含义 | 数据来源 |
|-----------|------|----------|
| `fixed` | 固定值 | 直接写在 `value` 字段 |
| `formula` | 表达式取值 | 写在 `formula.expression` 字段 |

### fixed 模式

```yaml
formData:
- name: "variableClassKey"
  valueMode: "fixed"
  value: "purchaseproject"
```

### formula 模式

```yaml
formData:
- name: "context"
  valueMode: "formula"
  value: ""                    # formula 模式下 value 为空字符串
  formula:
    expression: 'RuleParamUtil.parserParam(s1770776813956Context,"data.purchaseproject")'
    header: ""
    text: "初始化上下文.招标方案上下文"
    value:
    - name: "初始化上下文.招标方案上下文"
      id: "da259d63-3e0c-448d-bf08-e220ca2370fb;s1770776813956Context.data.purchaseproject"
      value: "s1770776813956Context.data.purchaseproject"
```

## formula.expression 常见模式

`formula.expression` 最常见的模式是通过 `RuleParamUtil.parserParam()` 从前面节点的输出取值：

```
RuleParamUtil.parserParam(s<节点ID>Context, "data.<字段路径>")
```

示例：
- `RuleParamUtil.parserParam(s1770776813956Context, "data.purchaseproject")` — 从 ID 为 `1770776813956` 的节点输出取 `data.purchaseproject` 字段
- `RuleParamUtil.parserParam(s1770776847781Context, "data.result")` — 从业务节点取 `data.result`

### formula.value 数组

`formula.value` 是一个数组，每项包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | 显示名称（中文） | `"初始化上下文.招标方案上下文"` |
| `id` | UUID + 分号 + 变量路径 | `"da259d63-...;s1770776813956Context.data.purchaseproject"` |
| `value` | 变量路径（与 expression 中引用一致） | `"s1770776813956Context.data.purchaseproject"` |

## 节点引用 ID 前缀 `s`

业务节点上下文变量名的命名规则为 `s<节点ID>Context`：

- 前缀固定为小写字母 `s`
- 中间是节点的 `id`（毫秒时间戳字符串）
- 后缀固定为 `Context`

示例：
- 节点 ID `1770776813956` → 上下文变量名 `s1770776813956Context`
- 节点 ID `1770776847781` → 上下文变量名 `s1770776847781Context`

取值路径格式：`s<节点ID>Context.data.<字段路径>`

## is_parallel / parallel_nums

| 字段 | 类型 | 说明 |
|------|------|------|
| `is_parallel` | boolean | `false` = 普通顺序节点；`true` = 并行节点（迭代器场景） |
| `parallel_nums` | integer | 并发数，默认 `10` |

- 标准三段式骨架中所有节点均为 `is_parallel: false`
- 仅在迭代（iteration）场景中，迭代节点内部子节点可能设置 `is_parallel: true`

## 校验规则

| 校验项 | 校验方式 |
|--------|----------|
| `app.mode` 是 `actflow` | 字符串精确匹配 |
| `app.id` == `app.rowguid` | 一致性校验 |
| 至少有一个 `start` 节点 + 一个 `end-vue`/`end` 节点 | 结构完整性 |
| 每个 edge 的 source/target 都能在 nodes 中找到 | 引用闭合 |
| 每个节点 `id` 唯一 | 唯一性 |
| `formula.expression` 中引用的节点 ID 都存在 | 跨节点引用 |
| `commonInitContext` 的 `businessContext` 是合法 Java 全限定名 | 字符串校验 |

## 常见错误

| 错误 | 后果 | 修复 |
|------|------|------|
| 节点 `id` 重复 | 渲染报错 | 重新生成唯一 ID |
| edge 的 source 指向不存在的节点 | 流程断裂 | 修正 source/target |
| `formula.expression` 中引用了已删除节点的 ID | 运行时取不到值 | 重新连接到现存节点 |
| 缺少 `app.sign` | 无法识别接口 | 必填 |
| `app.name` 与文件名差异过大 | 设计器列表混乱 | 保持一致 |
