# 交互模式：工作流触发 → 更新数据表

## 场景说明

当工作流（审批流程）到达某个活动节点时，自动更新业务数据表的状态字段。这是业务流程自动化最常见的模式：**流程驱动数据状态变更**。

**典型场景**：
- 审批通过后，将业务表状态更新为"已审批"
- 流程到达某节点时，自动回写流程标识到业务表
- 流程结束时，将业务表状态更新为"已完成"

## 流程结构

```
1. [工作流触发] 接收流程上下文数据
   ↓
2. [业务动作-更新数据] 根据流程标识匹配业务表，更新状态字段
   ↓
3. [结束] 返回成功
```

## 关键配置说明

### 1. 工作流触发节点

| 字段 | 说明 | 获取方式 |
|------|------|---------|
| `processGuid` | 流程定义 GUID | 从工作流设计器获取 |
| `operationGuid` | 操作 GUID | 从工作流设计器获取 |
| `eventType` | 事件类型编码 | 如 `55` 表示活动到达 |
| `activityGuid` | 活动节点 GUID | 从工作流设计器获取 |
| `eventSource` | 事件源编码 | 如 `22` |

**流程上下文数据**（触发后可引用）：

| 路径 | 说明 |
|------|------|
| `s{触发节点ID}Context.data.ProcessInstanceData.WorkflowPvi.processversioninstanceguid` | 流程版本实例标识（最常用，用于匹配业务表） |
| `s{触发节点ID}Context.data.ProcessInstanceData.WorkflowPvi.processguid` | 流程定义标识 |
| `s{触发节点ID}Context.data.ProcessInstanceData` | 流程实例数据（完整对象） |

### 2. 更新数据节点

- **updateType**: `"condition"` — 按条件更新（非主键更新）
- **conditions**: 用 `pviguid`（流程版本实例标识）匹配业务表中的流程标识字段
- **updateFields**: 要更新的字段及值（如 `status: "4"`）

**条件引用工作流上下文**：

```yaml
right:
- dataType: "string"
  formula: {}
  label: "工作流触发.流程实例数据.流程版本实例.流程版本实例标识"
  type: "field"
  value: "s{触发节点ID}Context.data.ProcessInstanceData.WorkflowPvi.processversioninstanceguid"
```

### 3. 结束节点（工作流触发专用）

与 Webhook 触发的结束节点不同：

| 字段 | Webhook触发 | 工作流触发 |
|------|-----------|-----------|
| `trigerType` | `"webhook"` | `"biz"` |
| `status` | 无 | `"success"` |
| `responseParams` | 有（定义返回结构） | 无 |

## 复用方法

遇到"工作流触发→更新数据表"的需求时：

1. 复制本模板
2. 替换 `{占位符}`：
   - `{业务名称}` / `{动作流标识}` / `{UUID}` — app 和 workflow 基础信息
   - `{目标表名}` — 要更新的业务表名（如 `zh_story`、`cg_project`）
   - `{匹配字段名}` — 业务表中存储流程标识的字段名（通常为 `pviguid`）
   - `{更新字段名}` / `{更新值}` — 要更新的状态字段和值
   - `{触发节点ID}` / `{更新节点ID}` / `{结束节点ID}` — 节点 ID（13位时间戳格式）
   - `{processGuid}` / `{operationGuid}` / `{activityGuid}` / `{eventSource}` / `{eventType}` — 工作流配置参数
3. 修改 `conditions` 和 `updateFields` 适配具体业务

## 变体

### 变体一：工作流触发 → 更新多个字段

```yaml
- name: "updateFields"
  valueMode: "fixed"
  value:
  - name: "status"
    valueMode: "fixed"
    value: "4"
  - name: "approver"
    valueMode: "fixed"
    value: "审批人名称"
  - name: "approvetime"
    valueMode: "fixed"
    value: "2026-01-01"
```

### 变体二：工作流触发 → 条件分支 → 不同更新

```
1. [工作流触发] 接收流程上下文
   ↓
2. [条件分支] 事件类型是否为"通过"？
   ↓ 是              ↓ 否
3a. [更新数据] 状态=已审批   3b. [更新数据] 状态=已驳回
   ↓                    ↓
4. [结束]              [结束]
```

### 变体三：工作流触发 → SQL查询 → 更新

当需要根据流程数据先查询再更新时：

```
1. [工作流触发] 接收流程上下文
   ↓
2. [SQL查询] 根据流程标识查询业务数据
   ↓
3. [条件分支] 查询结果是否为空？
   ↓ 非空              ↓ 为空
4. [更新数据] 更新状态   [结束] 返回提示
   ↓
5. [结束] 返回成功
```
