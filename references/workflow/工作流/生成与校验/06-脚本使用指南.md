# 脚本使用指南

> 本文档详细说明 `add_workflow.py`（生成）和 `validate_yml.py`（校验）两个脚本的使用方法。

## 快速生成脚本（add_workflow.py）

### 基本用法

```bash
# 最小骨架（开始 + 申请 + 1 个审批 + 结束 + 浏览）
python scripts/add_workflow.py --metadata <metadata 路径> --name "<流程名>" --material "<表单名>"

# 多审批节点 + 关联表单数据表（完整入参）
python scripts/add_workflow.py \
  --metadata <metadata 路径> \
  --name "费用报销审核流程" \
  --approvers "部门审核,主管审核,财务审核" \
  --material "费用报销申请单" \
  --form-id 159 --table-id 2024072378 --sql-tablename formtable20260112164507245
```

### 完整参数

| 参数 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `--metadata` | ✅ | 路径 | metadata 目录路径 |
| `--name` | ✅ | 字符串 | 流程名称（中文） |
| `--approvers` | 否 | 逗号分隔 | 审批节点列表（不含开始/申请/结束/浏览），默认 `"审批"` |
| `--material` | 否 | 字符串 | 流程材料名称，默认 `"申请表单"` |
| `--form-id` | 否 | 字符串 | 表单 ID，用于拼 pageurl，留空后续在设计器中关联 |
| `--table-id` | 否 | **整数** | 数据表 ID（workflowPvMisTableSet.tableid），**必须为整数** |
| `--sql-tablename` | 否 | 字符串 | 数据库表名 |
| `--author` | 否 | UUID | 创建人 GUID，默认自动生成 |
| `--process-mode` | 否 | 枚举 | `normal/custom/free/subprocess`，默认 `normal` |
| `--methods-json` | 否 | JSON 数组 | 外部方法列表，可含 `ruleguid` 关联动作流 |
| `--events-json` | 否 | JSON 数组 | 流程事件列表，可含 `ruleGuid` 关联动作流 |
| `--contexts-json` | 否 | JSON 数组 | 相关数据 workflowContext 列表 |
| `--conditions-json` | 否 | JSON 数组 | 流转条件列表，可按 `transition` / `transitionGuid` / `transitionIndex` 定位 |
| `--free-activities` | 否 | 逗号分隔 | 自由流转节点名；`process-mode=free` 且不传时默认所有审批节点 |
| `--custom-activities` | 否 | 逗号分隔 | 增加“流程设计”按钮的节点名；`process-mode=custom` 且不传时默认第一个审批节点 |
| `--subprocess-guid` | 否 | UUID | `process-mode=subprocess` 时必填，指向子流程 `processguid` |
| `--revoke-option` / `--revoke-allow-day` | 否 | 整数 | 发起人撤销策略 |
| `--no-participator-option` / `--default-user-guid` | 否 | 整数/UUID | 预设处理人不存在时的处理策略 |
| `--filename` | 否 | 字符串 | 自定义文件名（不含扩展名） |
| `--force` | 否 | 标志 | 覆盖已存在文件 |

> ⚠️ `--table-id` 现在强制为整数类型。传入非数字（如 `car_apply`）会直接报错。

### 工作流能力示例

```bash
# 自定义流程：第一个审批节点自动增加“流程设计”按钮
python scripts/add_workflow.py --metadata <metadata> --name "合同审批流程" --process-mode custom

# 自由流程：审批节点禁用退回按钮，并设置自由流转模式
python scripts/add_workflow.py --metadata <metadata> --name "领导批示流程" --process-mode free --approvers "领导批示"

# 主子流程：主流程插入 activitytype=90 子流程节点
python scripts/add_workflow.py \
  --metadata <metadata> \
  --name "报销主流程" \
  --process-mode subprocess \
  --subprocess-guid <子流程processguid>

# 事件或外部方法关联动作流 ruleGuid
python scripts/add_workflow.py \
  --metadata <metadata> \
  --name "审批流程" \
  --events-json '[{"eventName":"审批后更新状态","belongTo":22,"eventType":55,"sourceOperation":"审批:送下一步","ruleGuid":"updateStatusActionFlow"}]'
```

### 对话脚本（创建简单顺序流）

#### 第 1 步：流程基本信息

```text
你要创建什么工作流？我需要知道：
1. 流程名称（中文，如 采购立项审核流程）
2. 是否 Vue 流程？建议 0（非 Vue）
3. 流程级时限要不要限制？默认 -1 不限
```

#### 第 2 步：审批节点（不含开始/申请/结束/浏览）

```text
请描述审批节点（按顺序，逗号分隔，不需要写"开始""申请""结束""浏览"）：
节点 1 名称（如 部门审核）：
节点 2 名称（如 主管审批）：
节点 3 名称（如 财务审核）：

简单顺序流：每个相邻节点直接连接，无条件
条件分支（可选）：金额 <= 500 走"主管审批"；金额 > 500 走"总监审批"（手写）
```

#### 第 3 步：表单材料

```text
该流程关联的表单是？我需要：
1. 表单名称（如 采购立项申请表）
2. 表单 ID（数字，会拼成 epointtemp/epointsform/form/<ID>/V1/... 路径；不知道可留空，后续在表单设计器中关联）
3. 数据表 ID + 数据库表名（用于 workflowPvMisTableSet；不知道可留空）
```

#### 第 4 步：创建计划 + 明确批准

- 所有 GUID 自动生成 UUIDv4
- 节点坐标自动按规范布局
- 自动生成开始 + 申请 + 审批节点 + 结束 + 浏览 5 类节点
- 每个审批节点自动生成"通过 + 退回"按钮
- 每个退回按钮自动生成对应 workflowConfig
- 先展示生成清单和目标文件路径
- 用户明确回复"批准创建"后，再调用脚本一键生成并校验

## 校验脚本（validate_yml.py）

### 基本用法

```bash
# 校验单个工作流文件
python scripts/validate_yml.py <metadata 路径>/workflow/<流程名>.workflow.yml

# 校验整个应用（含跨引用关系）
python scripts/validate_yml.py --check-refs <metadata>
```

### 通过条件

**错误数 = 0** 才算交付。出现任何 error 必须修复后重新校验，警告（warn）也建议处理。
