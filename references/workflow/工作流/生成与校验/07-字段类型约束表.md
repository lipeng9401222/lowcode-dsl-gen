# 字段类型约束表

> ⚠️ **生成 workflow 前必读！** YAML 中数字和字符串的写法不同会导致解析类型不同，这是最常见的错误来源。

## 核心规则

1. **字符串类型字段必须加引号**：`tag: '20'`（字符串 `"20"`），不加引号的 `tag: 20` 会被解析为整数
2. **整数类型字段不要加引号**：`multitransactormode: 10`（整数），加引号的 `multitransactormode: '10'` 会被解析为字符串

## 校验脚本自动检测

`scripts/validate_yml.py` 已内置以上所有类型校验：
- `tableid` 非整数 → **error**
- `iconx/icony` 非字符串 → **warn**
- `multitransactormode` 等为字符串 → **warn**
- `direction/tag` 为整数 → **warn**
- `workflowProcessVersion` 工作流字段值域 → **warn**
