# 05 校验报告

目标：生成后用简洁报告告诉用户结果和风险。

## 必跑校验

- 单文件生成后：`python scripts/validate_yml.py <文件>`
- 整应用完成后（spec v2 严格校验）：`python scripts/validate_yml.py --strict --check-refs <app-root>`
  - `--strict` 会把以下问题升级为 error：旧驼峰键（`isDefault` / `mobileHandleType` / `isLockWhenMultiTransactor` / `subProMultiTransctorMode` / `isShowLineGraph` 等）、`tableid` 残留、`fromMisTableId` / `fromFieldId` 残留、`<apptag>/metadata/` 老结构残留。
- 需要列资产清单时：`python scripts/inventory_metadata.py --app-root <app-root>`

## 报告格式

```text
已完成：
- 应用：<applicationname> (<apptag>)
- 新增/修改资产：<数量和类型>
- 关键文件：<最多列 5 个>
- 校验：通过 / 有警告 / 有错误
- 后续建议：<下一步>
```

如果校验失败，先修复；无法修复时列出阻断项，不要报告“完成”。
