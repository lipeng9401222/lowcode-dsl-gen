# 01 需求识别

目标：判断用户输入是“新建应用”“已有应用补充”“单资产生成”还是“整应用生成”。

## 识别信号

- 出现“需求文档、会议纪要、完整应用、整个线下应用、从文档生成”等，走整应用生成。
- 已给出 `apptag` 或应用根目录路径，优先定位已有应用。
- 只提到“加一个代码项/数据表/页面/流程”，走资产级快速通道。
- 只说“生成请假工作流/审批流/流程”时，不得默认生成整应用；先确认目标是“加到已有应用、新建完整应用、仅生成计划”。

## 必须确认的信息

每轮最多问 3 个问题，只问无法从工程和材料中推断的信息：

- 应用中文名称和英文标识。
- 是否新建应用，还是补充已有应用。
- 需求材料是否覆盖页面、表、流程、字典和接口。
- 任务模式 `task.mode` 及来源：`asset-fast`、`whole-app`、`greenfield-app`、`existing-app` 或 `plan-only`。
- 工作流实现效果：流程目标、节点链路、处理人来源、条件分支、表单/详情页、关联 MIS、字段权限、会签/通过率、超时提醒、状态联动/event。

## 输出

输出一段自然语言理解结果：

```text
我理解这次是从需求材料生成一个低代码应用。已识别出：应用、数据表、页面和流程；代码项与动作流信息还需要补充。
```
