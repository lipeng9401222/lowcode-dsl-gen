# 自定义流程

## 适用场景

在流程流转过程中，临时发现后续步骤需要调整时，**无需对已审批流转的步骤进行退回和重新处理**，办理人可以直接在当前节点上自行调整后续流转路径。

适合的诉求：

- 流程"主干"相对固定，但末段节点（审核 / 复核 / 终审）需要灵活追加或替换处理人
- 不希望走"驳回 → 重新提交"的传统回退流程
- 业务部门希望在审批中临时分流，且变更只对当前实例生效

## 关键属性

### 1) `workflowProcess` 开启自定义流程

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `isnewversion` | 流程类型 | `10`（自定义流程）、`20`（自由流程） | 是 |
| `customtype` | 新/老自定义标记（`isnewversion=10` 场景下生效） | `1`（老）、非 1（新，常用 `0`） | 是 |

### 2) 在允许调整的节点增加 `operationtype=80` 流程设计按钮

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `operationtype` | 按钮类型 | `80`（自定义流程设计） | 是 |
| `operationpageurl` | 按钮触发后打开的流程设计页 | 见下方默认值 | 是 |

`operationpageurl` 默认值（直接复制即可）：

```text
{"url":"frame/pages/epointworkflow/client/flowsetting?processVersionInstanceGuid=[#=ProcessVersionInstance_ProcessVersionInstanceGuid#]&workItemGuid=[#=WorkItem_WorkItemGuid#]&processversionguid=[#=ProcessVersion_ProcessVersionGuid#]","dialog":{"fullscreen": true}}
```

## 生成要点

- `isnewversion=10` + `customtype=0` 是开启自定义流程的最小组合，缺一不可。
- 至少有 **1 个普通人工节点**配置 `operationtype=80` 流程设计按钮，否则用户无法进入流程设计页。
- `operationpageurl` 建议直接复制示例中的平台默认地址，里面的 `[#=...#]` 占位符由引擎运行时填充。
- 流程设计按钮和"送下一步"按钮可共存于同一节点；通常 `送下一步` 的 `ordernumber=0`、`流程设计` 的 `ordernumber=1`。
- 自定义流程仍受工作流 5 类节点完整性红线约束（开始/申请/审批/结束/浏览缺一不可）。
