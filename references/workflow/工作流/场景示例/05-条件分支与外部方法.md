# 条件分支与外部方法

本场景把表单字段提取为相关数据，通过流转条件控制走向，并在流程事件中调用外部方法或动作流。

## 关键配置

```yaml
workflowContext:
  - fieldguid: <UUID-金额参数>
    fieldName: jine
    note: "金额"
    fieldType: 30
    valueSource: 30
    fromMaterialGuid: <材料UUID>
    fromMisTableId: '2'
    fromFieldId: '1707'
    fromFieldName: jine
    fieldValue: '[#=EPOINTFORM(formtable20250704162233623,jine,10)#]'
    belongTo: 10
    processversionguid: <版本UUID>

method:
  - methodGuid: <UUID-更新状态方法>
    processversionguid: <版本UUID>
    typename: com.epoint.demo.workflow.ExternalMethods
    methodname: updateStatus
    returnvaluetype: Void
    note: "更新业务状态"
    ordernum: 10
    ruleguid: updateStatusActionFlow

workflowEvent:
  - eventGuid: <UUID-事件>
    eventName: "审批后更新状态"
    belongTo: 22
    eventType: 55
    sourceGuid: <送下一步按钮UUID>
    processversionguid: <版本UUID>
    syncType: 0
    orderNumber: 100
    ruleGuid: updateStatusActionFlow

transition:
  - transitionguid: <UUID-金额审批流转>
    transitionname: "大额审批"
    workflowTransitionCondition:
      - conditionGuid: <UUID-条件>
        conditionName: "金额大于6000"
        transitionGuid: <UUID-金额审批流转>
        conditionExpressionType: 10
        leftValue: '[#=jine#]'
        compareOperation: ">"
        rightValue: '6000'
        valueType: 30
        processversionguid: <版本UUID>
        orderNum: 0
```

## 生成要点

- `workflowContext.fieldName` 可通过 `[#=字段名#]` 被流转条件和方法参数引用。
- `method.ruleguid`、`workflowEvent.ruleGuid`、`workflowTransitionCondition.ruleGuid` 都可以填写动作流标识。
- 动作流文件应在 event 阶段生成，并确保 `app.id`、`rowguid` 或 `sign` 与 ruleGuid 对得上。
