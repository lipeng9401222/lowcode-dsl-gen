# 条件分支与外部方法

本场景把表单字段提取为相关数据，通过流转条件控制走向，并在流程事件中调用外部方法或动作流。

## 关键配置

```yaml
workflowContext:
  - fieldguid: <UUID-金额参数>
    fieldname: jine
    note: "金额"
    fieldtype: 30
    valuesource: 30
    frommaterialguid: <材料UUID>
    fromsqltablename: formtable20250704162233623
    fromfieldname: jine
    fieldvalue: '[#=EPOINTFORM(formtable20250704162233623,jine,10)#]'
    belongto: 10
    processversionguid: <版本UUID>

transition:
  - transitionguid: <UUID-金额审批流转>
    transitionname: "大额审批"
    workflowTransitionCondition:
      - conditionguid: <UUID-条件>
        conditionname: "金额大于6000"
        transitionguid: <UUID-金额审批流转>
        conditionexpressiontype: 10
        leftvalue: '[#=jine#]'
        compareoperation: ">"
        rightvalue: '6000'
        valuetype: 30
        processversionguid: <版本UUID>
        ordernum: 0
    
method:
  - methodguid: <UUID-更新状态方法>
    processversionguid: <版本UUID>
    methodname: 更新业务状态
    note: "更新业务状态"
    ordernum: 10
    ruleguid: updateStatusActionFlow

workflowEvent:
  - eventguid: <UUID-事件>
    eventname: "审批后更新状态"
    belongto: 22
    eventtype: 55
    sourceguid: <送下一步按钮UUID>
    processversionguid: <版本UUID>
    synctype: 0
    ordernumber: 100
    ruleguid: updateStatusActionFlow


```

## 生成要点

- `workflowContext.fieldname` 可通过 `[#=字段名#]` 被流转条件表达式引用。
- “否则 / 不满足条件”必须生成显式互补条件，例如一条 `=` 与一条 `!=`，不要只设置默认变迁。
- `--conditions-json` 支持 `fromActivity` / `toActivity` 定位变迁；当源目标之间没有 transition 时，脚本会自动补线。
- 条件可选分支按 `A → B → C` 且同时存在 `A → C` 识别，`B` 放在分支泳道 `icony='20'`，主线节点保持 `icony='180'`。
- `method.ruleguid`、`workflowEvent.ruleguid`、`workflowTransitionCondition.ruleguid` 都可以填写动作流标识。
- 动作流文件应在 event 阶段生成，并确保 `app.id`、`rowguid` 或 `sign` 与 ruleguid 对得上。

## 典型示例：满足条件先评审，否则直接执行

```json
[
  {
    "fromActivity": "申请",
    "toActivity": "总负责人评审",
    "transitionname": "总负责人评审",
    "conditionname": "需求来源为内部优化",
    "leftvalue": "[#=demandsource#]",
    "lefttext": "需求来源",
    "compareoperation": "=",
    "rightvalue": "4",
    "righttext": "内部优化",
    "valuetype": 10
  },
  {
    "fromActivity": "申请",
    "toActivity": "指定人员执行",
    "transitionname": "直接执行",
    "conditionname": "需求来源非内部优化",
    "leftvalue": "[#=demandsource#]",
    "lefttext": "需求来源",
    "compareoperation": "!=",
    "rightvalue": "4",
    "righttext": "非内部优化",
    "valuetype": 10
  }
]
```
