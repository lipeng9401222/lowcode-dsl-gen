# 表单材料与数据表映射

> 本文档定义 workflowPvMaterial（流程版本材料）和 workflowPvMisTableSet（表单数据表映射）。

## workflowPvMaterial（流程版本材料）

**1 个流程通常 1 个表单**。

```yaml
workflowPvMaterial:
  - materialguid: <UUID-表单>
    processversionguid: <版本UUID>
    materialname: "请假表"
    type: 10                         # 10:表单 20:附件
    status: 10
    submittype: 10                   # 10:电子件 20:纸质
    pageurl_read: epointtemp/epointsform/form/159/V1/recorddetail
    pageurl_readandwrite: epointtemp/epointsform/form/159/V1/record
    mobilepageurl_read: epointtemp/epointsform/form/159/V1/mobilerecorddetail
    mobilepageurl_readandwrite: epointtemp/epointsform/form/159/V1/mobilerecord
```

## workflowPvMisTableSet（表单数据表映射）

**1 个表单 1 条**。

> ⚠️ **spec v2**：`tableid` 字段已**废弃**——mis 表会先于工作流入库，引擎在工作流安装时根据 `sql_tablename` 自动绑定 mis 表 ID，yml 里**不要**再写 `tableid`。

```yaml
workflowPvMisTableSet:
  - mistablesetguid: <UUID>
    processversionguid: <版本UUID>
    materialguid: <对应表单 materialguid>
    tableguid: <UUID>
    sql_tablename: formtable20260112164507245
```

## 相关数据（workflowContext）

> **工作流扩展** — 本节内容由原 `references/workflow/工作流扩展节点/03-相关数据.md` 合并而来，属于工作流扩展字段。

用表达式方式将业务字段定义成参数，供流程中使用（如流转条件的 `[#=jine#]`）。

### 数据来源方式（`valuesource`）

| `valuesource` | 含义 | 关键字段 |
|--------------|------|---------:|
| `10` | 直接赋值 | `fieldvalue` 直接填常量 |
| `20` | 方法获取 | 通过外部方法获取值 |
| `30` | 表单字段 | `frommaterialguid` + `fromsqltablename` + `fromfieldname` + `fieldvalue`（EPOINTFORM 表达式） |

> **spec v2 调整**：`fromMisTableId` / `fromFieldId` 已废弃，由 `fromsqltablename` + `fromfieldname` 替代——引擎安装工作流时会根据 `fromsqltablename` + `fromfieldname` 自动绑定 mis 表 ID 和字段 ID。

### 字段一览

| 字段 | 含义 | 类型 | 必填 |
|------|------|------|------|
| `fieldguid` | 唯一标识 | UUID | 是 |
| `fieldname` | 参数名称 | 字符串（如 `jine`） | 是 |
| `belongto` | 所属 | 数字，默认 `10` | 是 |
| `fieldtype` | 数据类型 | `10`（字符）、`20`（整形）、`30`（数字）、`40`（时间）、`50`（布尔） | 是 |
| `valuesource` | 来源方式 | `10`（直接赋值）、`20`（方法获取）、`30`（表单字段） | 是 |
| `frommaterialguid` | 来源材料标识 | UUID（= workflowPvMaterial.materialguid） | 否（valuesource=30 时填） |
| `fromsqltablename` | 来源表 sql_tablename（spec v2 新增） | 字符串（= workflowPvMisTableSet.sql_tablename，如 `formtable20250704162233623`） | 否（valuesource=30 时填） |
| `fromfieldname` | 来源字段名 | 字符串 | 否（valuesource=30 时填） |
| `fieldvalue` | 数据值 | 字符串（如 `[#=EPOINTFORM(formtable20250704162233623,jine,10)#]`） | 否 |
| `processversionguid` | 版本标识 | UUID | 是 |
| `note` | 说明 | 字符串 | 是 |

### 示例

```yaml
workflowContext:
  # 来源方式：表单字段（spec v2: 用 fromsqltablename 取代 fromMisTableId/fromFieldId）
  - fieldguid: <UUID>
    fieldname: jine
    note: "金额"
    fieldtype: 10
    valuesource: 30
    frommaterialguid: <UUID-材料>
    fromsqltablename: formtable20250704162233623
    fromfieldname: jine
    fieldvalue: '[#=EPOINTFORM(formtable20250704162233623,jine,10)#]'
    belongto: 10
    processversionguid: <版本UUID>
  # 来源方式：直接赋值
  - fieldguid: <UUID>
    fieldname: size
    note: "数量"
    fieldtype: 10
    valuesource: 10
    fieldvalue: '11'
    belongto: 10
    processversionguid: <版本UUID>
```

> ⚠️ `workflowContext` 定义的参数可在流转条件（`workflowTransitionCondition`）中通过 `[#=参数名#]` 引用，也可在外部方法参数（`workflowMethodParameter`）中使用。
