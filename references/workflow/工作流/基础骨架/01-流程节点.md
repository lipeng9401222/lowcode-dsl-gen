# 流程节点定义

> 本文档详细定义了工作流中 5 类必须节点的字段规范。完整的设计红线和检查清单见 [工作流入口](../index.md)。

## workflowProcess（流程基本信息）

```yaml
workflowProcess:
  processguid: 4b05e824-a18d-4b87-a0d4-a35a6a4c70a7   # 流程唯一标识，UUIDv4
  processname: "请假申请流程20260429"                    # 流程名称（中文）
  note: "员工请假审批流程"                               # 备注
  isvue: 1                          # 1:Vue（默认）；0:非Vue
  designversion: senior             # senior:高级；simple:简易
  ordernum: 50
  status: 10                        # 10:启用；20:停用
  tag: '20'                         # 监听开关 '10':启用；'20':停用（注意是字符串）
  isnewversion: 10                  # 自定义流程标记
  customtype: null
  cansimpledisplay: null
  statemachinetag: null
  appguid: null
  baseouguid: null
  tenantguid: ''
```

## activity（活动 = 流程节点）

> ⚠️ activity 数组项**直接平铺**字段，**没有** `WorkflowActivity:` 子层包装。

### activitytype 枚举（必填）

| 值 | 含义 | vmlid |
|----|------|-------|
| `10` | 开始 | `-1` |
| `20` | 结束 | `1` |
| `30` | 普通人工活动（最常用：申请、审批、终审） | `≥2` 递增 |
| `40` | 路由活动 | - |
| `50` | 外部活动 | - |
| `90` | 子流程 | - |
| `100` | 浏览（孤立节点） | `-2` |
| `110` | 加办活动 | - |

### 通用属性

```yaml
- activityguid: <UUIDv4>             # 活动唯一标识
  activityname: <中文名>              # 名称
  activitydispname: <中文名>          # 显示名（同 activityname）
  processversionguid: <版本UUID>     # 关联版本（所有节点必须一致）
  activitytype: 10/20/30/100         # 活动类型
  splittype: 30                      # 分支方式 10:全部 20:多路 30:单一
  jointype: 30                       # 会合方式 10/20/30/40
  vmlid: -1/1/-2/2..N                # 活动序号
  iconx: '-159'                      # 横坐标（字符串）
  icony: '123'                       # 纵坐标（字符串）
  multitransactormode: 10            # 多人处理模式（10/20/30/25/50/60/70）
  handleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag>'  # PC 处理页地址，pagetag 取自同应用 pagedesigne yml 顶层
  mobilehandleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag>'
  isallowaddattachfile: 20           # 附件操作权限 10/20/30/40
  timelimitenable: 20                # 时限开关
  timelimit: -1.0
  timelimitunit: 55
  earlywarning_enable: 20            # 预警开关
  earlywarning_time: -1.0
  earlywarning_timeunit: 55
  is_passwhennotransactor: 20         # 是否共用活动实例（spec v2 已小写化）
  is_lockwhenmultitransactor: 0
  locktimewhenmultitransactor: '60'   # [可选] 多人锁定持续时间（分钟），>1
  mobilehandletype: app
  mobileappletrules: ''               # [可选] 移动端小程序规则
  smscontent: ''                      # [可选] 短信通知内容
  overtimeremindcontent: ''           # [可选] 超时提醒内容
  addattachfilesource: ''             # [可选] jointype=20 时多路汇合选择的分支节点 guid
  joinmethodguid: ''                  # [可选] jointype=40 时自定义会合方法 guid
  note: <说明>
```

### activity 通用扩展字段（spec v2 补充）

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `addattachfilesource` | 多路汇合场景选择的分支节点标识 | jointype=20 时填对应分支节点 guid | 否 |
| `joinmethodguid` | 自定义汇合方法标识 | jointype=40 时填外部方法 guid | 否 |
| `locktimewhenmultitransactor` | 多人处理时锁定持续时间 | 字符串（分钟，>1） | 否 |
| `mobileappletrules` | 移动端小程序规则 | 字符串 | 否 |
| `smscontent` | 短信通知内容 | 字符串 | 否 |
| `overtimeremindcontent` | 超时提醒内容 | 字符串 | 否 |

> **生成策略**：默认骨架不输出这些扩展字段。当用户提到"超时提醒 / 短信通知 / 多人锁定时间 / 小程序规则 / 多路汇合"等关键词时，由 skill 主动追问后通过 `add_workflow.py --activity-extra-json` 注入。

### 1. 开始节点（`activitytype=10`）

**纯启动，不能填表单**。

```yaml
- activityguid: <UUID-开始>
  activityname: "开始"               # 必须是 "开始"
  activitydispname: "开始"
  processversionguid: <版本UUID>
  activitytype: 10
  splittype: 30
  jointype: null                     # 应为空
  multitransactormode: null          # 应为空
  handleurl: null                    # 应为空
  mobilehandleurl: null              # 应为空
  isallowaddattachfile: null         # 应为空
  timelimitenable: null              # 应为空
  earlywarning_enable: null          # 应为空
  is_passwhennotransactor: null       # 应为空
  iconx: '-159'
  icony: '123'
  vmlid: -1
  workflowActivityOperation:
    - operationguid: <UUID>
      activityguid: <UUID-开始>
      processversionguid: <版本UUID>
      operationname: "送下一步"
      operationtype: 10              # 通过
      is_requireopinion: 20
      is_checkmaterialsubmit: 20
      is_showoperationpage: 0
      operationvisiablecase: '10'
      is_shownextactivity: 1
```

### 2. 申请节点（`activitytype=30`，`activityname=申请`）

**填报提交**：必须绑定表单（`workflowPvMaterial`），必须有 `handleurl`，支持附件，可退回。

```yaml
- activityguid: <UUID-申请>
  activityname: "申请"               # 必须是 "申请"
  activitytype: 30
  splittype: 30
  jointype: 30
  multitransactormode: 10
  handleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag_form>'
  mobilehandleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag_form>'
  isallowaddattachfile: 20
  timelimitenable: 20
  timelimit: -1.0
  timelimitunit: 55
  earlywarning_enable: 20
  earlywarning_time: -1.0
  earlywarning_timeunit: 55
  is_passwhennotransactor: 20
  is_lockwhenmultitransactor: 0
  mobilehandletype: app
  iconx: '26.5'
  icony: '99'
  vmlid: 2
  workflowActivityOperation:
    - operationguid: <UUID>
      activityguid: <UUID-申请>
      processversionguid: <版本UUID>
      operationname: "送下一步"
      operationtype: 10
      is_requireopinion: 10
      is_checkmaterialsubmit: 10     # 提交前校验表单必填项
      is_showoperationpage: 1
      operationvisiablecase: '10'
      is_shownextactivity: 1
```

### 3. 审批节点（`activitytype=30`，`activityname` 自定义）

**普通人工审批节点**：含 1 个通过 + 1 个退回按钮。

```yaml
- activityguid: <UUID-审批>
  activityname: "部门审核"            # 自定义
  activitytype: 30
  splittype: 30
  jointype: 30
  multitransactormode: 10            # 多人处理模式 10/20/30/25/50/60/70
  handleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag_form>'
  mobilehandleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag_form>'
  isallowaddattachfile: 20
  timelimitenable: 20
  timelimit: -1.0
  timelimitunit: 55
  earlywarning_enable: 20
  earlywarning_time: -1.0
  earlywarning_timeunit: 55
  is_passwhennotransactor: 20
  is_lockwhenmultitransactor: 0
  mobilehandletype: app
  iconx: '345.5'
  icony: '99'
  vmlid: 3
  nopasshandlevalue: '10'            # '10':退回上一步 '20':退回开始
  passrate: '60'                     # [可选] 通过率 0-100；spec v2 普通节点专属
  passratecalculatemode: 10          # [可选] 通过率计算模式：10 完成数计算 / 20 每次完成都计算
  workflowActivityOperation:
    # 通过按钮
    - operationguid: <UUID-通过>
      operationname: "送下一步"
      operationtype: 10
      ...
    # 退回按钮（必有对应 workflowConfig）
    - operationguid: <UUID-退回>
      operationname: "退回"
      operationtype: 30              # 退回
      is_requireopinion: 10
      is_checkmaterialsubmit: 20
      is_showoperationpage: 1
      operationvisiablecase: '1020'  # '10'正常+'20'退回时
      is_shownextactivity: 0
      backtargetscope: '1'           # '1':仅提交人 '2':所有人
      targetactivity: '[#=AllBeforeActivity#]'
      multitransctormode: OR         # OR/AND/OrAndRead
```

### 普通节点专属字段（spec v2 补充）

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `passrate` | 通过率 | 字符串（0-100，如 `'60'`） | 否 |
| `passratecalculatemode` | 通过率计算模式 | `10`（按完成数计算）/ `20`（每次完成都计算） | 否 |
| `nopasshandlevalue` | 未通过处理 | `'10'`（退回上一步）/ `'20'`（退回开始） | 否，默认 `'10'` |

> **生成策略**：默认骨架不输出 `passrate` / `passratecalculatemode`。用户提到"通过率 / 多人会签 60% 通过"等关键词时，skill 主动追问后通过 `add_workflow.py --approve-pass-rate-json` 注入。

### 5. 浏览节点（`activitytype=100`，孤立节点）

**孤立节点，不参与流转**。位置置于整个流程的上方（icony 为负值）。

```yaml
- activityguid: <UUID-浏览>
  activityname: "浏览"
  activitytype: 100
  splittype: 30
  jointype: null
  multitransactormode: null          # 建议为空
  handleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag_detail>' # 浏览节点允许填，pagetag 取详情页
  mobilehandleurl: 'home/vuepagedesigner/renderer/add?pagetag=<page_tag_detail>'
  isallowaddattachfile: null
  timelimitenable: null
  earlywarning_enable: null
  is_passwhennotransactor: null
  iconx: '253'
  icony: '-20'
  vmlid: -2
```

### 6. 路由节点（`activitytype=40`）

用于自定义会合判断。当前节点和后续节点之间可增加路由节点，当前普通人工节点可将 `multitransactormode` 设为 `25`，路由节点用 `jointype=40` 并通过 `joinmethodguid` 指向会合判断外部方法。

```yaml
- activityguid: <UUID-路由>
  activityname: "单位会合判断"
  activitydispname: "单位会合判断"
  processversionguid: <版本UUID>
  activitytype: 40
  splittype: 30
  jointype: 40
  joinmethodguid: <外部方法UUID>
  iconx: '545.5'
  icony: '99'
  vmlid: 4
```

### 7. 子流程节点（`activitytype=90`）

用于主流程调用可复用的子流程。子流程可同步或异步执行，并通过 `callsubprocessguid` 绑定被调用流程的 `processguid`。

```yaml
- activityguid: <UUID-子流程节点>
  activityname: "发票核验子流程"
  activitydispname: "发票核验子流程"
  processversionguid: <版本UUID>
  activitytype: 90
  splittype: 20
  jointype: 30
  callsubprocessguid: <子流程processguid>
  subprocesssynctype: 20            # 10 异步；20 同步
  multitransactormode: 10           # 10 全部子流程完成；20 任一子流程完成
  subProMultiTransctorMode: 10      # 多处理人传递给子流程第一个活动
  iconx: '545.5'
  icony: '99'
  vmlid: 4
```


## 附：handleurl 的 pagetag 来源规则

需要填 `handleurl` / `mobilehandleurl` 的活动类型：**activitytype ∈ {30, 50, 100, 110}**——普通人工活动（30）、外部活动（50）、浏览活动（100）、加办活动（110）。开始（10）/ 结束（20）/ 路由（40）/ 子流程（90）这几类不填。

默认值形如：

```
home/vuepagedesigner/renderer/add?pagetag=<page_tag>
```

`<page_tag>` 取自同应用 `<apptag>/pagedesigne/<...>.pagedesigne.yml` 顶层 `pagetag` 字段。

| 节点类型 | 默认 pagetag |
|---------|-------------|
| 申请 / 审批（30） | 表单页 pagetag |
| 浏览（100） | 详情页 pagetag；无独立详情页时回落到表单页 pagetag |
| 外部活动 / 加办活动（50 / 110） | 按业务实际，通常用表单页 pagetag |

### 取值流程

1. 扫同应用 `<apptag>/pagedesigne/` 目录的 `*.pagedesigne.yml`。
2. **仅 1 份** → 自动取该 pagedesigne 的 pagetag。
3. **多份** → 按编号列表与用户对话确认（"哪个用作表单页 / 哪个用作详情页"）。
4. 脚本：`scripts/add_workflow.py --pagetag-form <表单页 pagetag> --pagetag-detail <详情页 pagetag>`，未传时按上述规则自动推断；多份且未传则报错退出。

> ⚠️ 校验：自定义 handleurl（不以 `home/vuepagedesigner/renderer/` 开头）时 `validate_yml.py` 会输出 warn，但不阻断生成。
