# 活动按钮与退回配置

> 本文档定义活动按钮（workflowActivityOperation）、退回配置（workflowConfig）及 工作流扩展的活动材料/权限字段。

## workflowActivityOperation（活动按钮）

每个普通节点含 1-2 个按钮（送下一步、退回）。

### operationtype 枚举

| 值 | 含义 |
|----|------|
| `10` | 通过 |
| `30` | 退回 |
| `2` | 退回到开始 |
| `20` | 转交 |
| `50` | 收回 |
| `60` | 保存 |
| `120` | 抄送 |
| `125` | 协办 |
| `130` | 加办 |
| `80` | 自定义流程设计 |
| `85` | 送至退回节点 |

### 按钮通用字段（工作流扩展标记 ）

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `controlvisiablemethodguid`  | 控制按钮显隐的外部方法标识 | 外部方法 GUID | 否 |
| `defaultopinion`  | 默认意见 | 字符串 | 否 |
| `is_showopiniontemplete`  | 是否显示意见模板 | `1`（是）、`0`（否），默认 `0` | 是 |
| `statelevel`  | 按钮重要程度 | `0`（默认）、`10`（主按钮，蓝色）、`20`（警告，黄色）、`30`（危险，红色） | 否 |
| `ordernumber`  | 排序号 | 正整数，默认 `0` | 是 |

> ⚠️ `is_showopiniontemplete` 与 `ordernumber` 在引擎层视为**必填**，缺失会被拦截。`scripts/add_workflow.py` 已默认输出这两个字段。

### 按钮 工作流字段完整示例

带主按钮 / 退回按钮 工作流字段的完整审批节点写法：

```yaml
- activityguid: <UUID-审批>
  activityname: "部门审核"
  activitytype: 30
  ...
  workflowActivityOperation:
    # 通过按钮（statelevel=10 主按钮蓝色 / 默认意见 / 排序 0）
    - operationguid: <UUID-通过>
      activityguid: <UUID-审批>
      processversionguid: <版本UUID>
      operationname: "送下一步"
      operationtype: 10
      is_requireopinion: 10
      is_checkmaterialsubmit: 20
      is_showoperationpage: 1
      operationvisiablecase: '10'
      is_shownextactivity: 1
      is_showopiniontemplete: 0       # 必填：是否显示意见模板，0=否
      ordernumber: 0                  # 必填：排序号
      statelevel: 10                  # 主按钮（蓝色）
      defaultopinion: "同意"          # 可选：默认意见，点击按钮自动填入
      controlvisiablemethodguid: null # 可选：控制按钮显隐的外部方法
    # 退回按钮（statelevel=30 危险红色 / 排序 1）
    - operationguid: <UUID-退回>
      activityguid: <UUID-审批>
      processversionguid: <版本UUID>
      operationname: "退回"
      operationtype: 30
      is_requireopinion: 10
      is_checkmaterialsubmit: 20
      is_showoperationpage: 1
      operationvisiablecase: '1020'
      is_shownextactivity: 0
      backtargetscope: '1'
      targetactivity: '[#=AllBeforeActivity#]'
      multitransctormode: OR
      is_showopiniontemplete: 0
      ordernumber: 1
      statelevel: 30                  # 危险按钮（红色）
```

### 退回按钮专属字段

| 字段 | 含义 | 取值 |
|------|------|------|
| `targetactivity` | 退回目标 | `[#=AllBeforeActivity#]`（所有前序） |
| `backtargetscope` | 退回范围 | `'1'`（仅提交人）、`'2'`（所有处理人） |
| `multitransctormode` | 退回时多人处理模式 | `OR`、`AND`、`OrAndRead` |
| `operationvisiablecase` | 显示场合 | `'10'`正常、`'20'`退回时、`'30'`已办时，可拼接如 `'1020'` |

### 自定义流程设计按钮（`operationtype=80`）

自定义流程需要在允许用户调整后续流转的普通人工节点上增加“流程设计”按钮，同时 `workflowProcess.isnewversion=10`、`customtype=0`。

```yaml
- operationguid: <UUID-流程设计按钮>
  activityguid: <UUID-当前活动>
  processversionguid: <版本UUID>
  operationname: "流程设计"
  operationtype: 80
  is_requireopinion: 20
  is_checkmaterialsubmit: 20
  is_showoperationpage: 1
  operationvisiablecase: '10'
  is_shownextactivity: 0
  is_showopiniontemplete: 0
  ordernumber: 2
  statelevel: 10
  operationpageurl: '{"url":"frame/pages/epointworkflow/client/flowsetting?processVersionInstanceGuid=[#=ProcessVersionInstance_ProcessVersionInstanceGuid#]&workItemGuid=[#=WorkItem_WorkItemGuid#]&processversionguid=[#=ProcessVersion_ProcessVersionGuid#]","dialog":{"fullscreen": true}}'
```

### 自由流程按钮红线

自由流程（`workflowProcess.isnewversion=20`）中不允许配置退回按钮（`operationtype=30`）。需要反向处理时，用普通通过按钮配置反向流转。

## workflowConfig（退回按钮配置，必须与退回按钮一一对应）

```yaml
workflowConfig:
  - rowguid: <UUID>
    processversionguid: <版本UUID>
    belongto: 22                     # 22 表示按钮
    configname: backTargetScope
    configvalue: '1'                 # '1' 仅提交人；'2' 所有处理人
    sourceguid: <对应退回按钮的 operationguid>
```

> ⚠️ 校验脚本会**严格校验**退回按钮 ↔ workflowConfig 一一对应，缺一报错。

## workflowActivityMaterial（活动材料，可选）

> **说明**：每个普通节点关联 ≥0 个流程版本材料，用于控制各节点可操作的表单材料。
>
> **生成策略**：默认骨架不输出此节点；当用户明确要求"为某个审批节点配置表单字段权限 / 哪些字段只读 / 哪些字段隐藏"等需求时，由 skill 主动追问后通过 `add_workflow.py --activity-materials-json` 注入。

### 字段表

| 字段 | 含义 | 类型 | 可选值 / 默认 | 必填 |
|------|------|------|--------------|------|
| `activitymaterialguid` | 唯一标识 | UUID | 自动生成 | 是 |
| `processversionguid` | 版本标识 | UUID | 同 workflowProcessVersion.processversionguid | 是 |
| `activityguid` | 活动标识 | UUID | 对应 activity.activityguid | 是 |
| `materialguid` | 材料标识 | UUID | 对应 workflowPvMaterial.materialguid | 是 |
| `accessright` | 权限 | 整数 | `10` 只读 / `20` 读写，默认 `20` | 是 |
| `is_mustsubmit` | 是否必须提交 | 整数 | `10` 是 / `20` 否，默认 `20` | 是 |
| `submittype` | 提交方式 | 整数 | `10` 提交方表单 | 是 |
| `configdata` | 权限配置数据（对接表单全部配置） | 字符串 | - | 否 |
| `controlvisiablemethodguid` | 控制显隐外部方法标识 | UUID | - | 否 |

## workflowActivityFieldAccess（活动材料字段权限，可选）

> 每个活动材料关联 ≥0 条字段权限配置，控制各节点对表单字段的读写权限。

### 字段表

| 字段 | 含义 | 类型 | 可选值 / 默认 | 必填 |
|------|------|------|--------------|------|
| `rowguid` | 唯一标识 | UUID | 自动生成 | 是 |
| `activitymaterialguid` | 所属活动材料标识 | UUID | 同 workflowActivityMaterial.activitymaterialguid | 是 |
| `processversionguid` | 版本标识 | UUID | 同 workflowProcessVersion.processversionguid | 是 |
| `activityguid` | 活动标识 | UUID | 对应 activity.activityguid | 是 |
| `materialguid` | 材料标识 | UUID | 对应 workflowPvMaterial.materialguid | 是 |
| `sqltablename` | 表单 sqlTableName | 字符串 | 同 workflowPvMisTableSet.sql_tablename，如 `formtable20250704162233623` | 是 |
| `fieldname` | 表单字段名 | 字符串 | 如 `jine` | 是 |
| `fieldchinesename` | 表单字段中文名 | 字符串 | 如 `金额` | 是 |
| `accessright` | 权限 | 整数 | `10` 只读 / `20` 读写 / `30` 隐藏 / `40` 必填 | 是 |
| `accesstype` | 字段类型 | 整数 | `10`（属性字段） | 是 |
| `isallowattachwrite` | 附件字段权限 | 字符串 | `add;delete;edit;view`，逗号或分号分隔 | 否 |

### 嵌套示例（活动 → 活动材料 → 字段权限）

```yaml
# 审批节点：金额字段只读、备注字段隐藏
- activityguid: <UUID-审批>
  activityname: "部门审核"
  activitytype: 30
  ...
  workflowActivityMaterial:
    - activitymaterialguid: <UUID-活动材料>
      processversionguid: <版本UUID>
      activityguid: <UUID-审批>
      materialguid: <UUID-表单>
      accessright: 20
      is_mustsubmit: 20
      submittype: 10
      workflowActivityFieldAccess:
        - rowguid: <UUID-字段权限-1>
          activitymaterialguid: <UUID-活动材料>
          processversionguid: <版本UUID>
          activityguid: <UUID-审批>
          materialguid: <UUID-表单>
          sqltablename: formtable20250704162233623
          fieldname: jine
          fieldchinesename: "金额"
          accessright: 10              # 只读
          accesstype: 10
        - rowguid: <UUID-字段权限-2>
          activitymaterialguid: <UUID-活动材料>
          processversionguid: <版本UUID>
          activityguid: <UUID-审批>
          materialguid: <UUID-表单>
          sqltablename: formtable20250704162233623
          fieldname: remark
          fieldchinesename: "备注"
          accessright: 30              # 隐藏
          accesstype: 10
```

> 校验脚本对 `workflowActivityMaterial.activityguid / materialguid` 引用闭合做强校验；`workflowActivityFieldAccess.activitymaterialguid` / `sqltablename` 引用闭合做 warn 提示。
