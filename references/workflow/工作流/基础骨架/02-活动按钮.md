# 活动按钮与退回配置

> 本文档定义活动按钮（workflowActivityOperation）、退回配置（workflowConfig）及 工作流扩展的活动材料/权限字段。

## workflowActivityOperation（活动按钮）

每个普通节点含 1-2 个按钮（送下一步、退回）。

### operationtype 枚举

| 值 | 含义 |
|----|------|
| `10` | 通过 |
| `30` | 退回 |
| `2` | 退回到开始 |
| `20` | 转交 |
| `50` | 收回 |
| `60` | 保存 |
| `120` | 抄送 |
| `125` | 协办 |
| `130` | 加办 |
| `80` | 自定义流程设计 |
| `85` | 送至退回节点 |

### 按钮通用字段（工作流扩展标记 ）

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `controlVisiableMethodGuid`  | 控制按钮显隐的外部方法标识 | 外部方法 GUID | 否 |
| `defaultOpinion`  | 默认意见 | 字符串 | 否 |
| `is_ShowOpinionTemplete`  | 是否显示意见模板 | `1`（是）、`0`（否），默认 `0` | 是 |
| `stateLevel`  | 按钮重要程度 | `0`（默认）、`10`（主按钮，蓝色）、`20`（警告，黄色）、`30`（危险，红色） | 否 |
| `ordernumber`  | 排序号 | 正整数，默认 `0` | 是 |

> ⚠️ `is_ShowOpinionTemplete` 与 `ordernumber` 在引擎层视为**必填**，缺失会被拦截。`scripts/add_workflow.py` 已默认输出这两个字段。

### 按钮 工作流字段完整示例

带主按钮 / 退回按钮 工作流字段的完整审批节点写法：

```yaml
- activityguid: <UUID-审批>
  activityname: "部门审核"
  activitytype: 30
  ...
  workflowActivityOperation:
    # 通过按钮（stateLevel=10 主按钮蓝色 / 默认意见 / 排序 0）
    - operationguid: <UUID-通过>
      activityguid: <UUID-审批>
      processversionguid: <版本UUID>
      operationname: "送下一步"
      operationtype: 10
      is_requireopinion: 10
      is_checkmaterialsubmit: 20
      is_showoperationpage: 1
      operationvisiablecase: '10'
      is_shownextactivity: 1
      is_ShowOpinionTemplete: 0       # 必填：是否显示意见模板，0=否
      ordernumber: 0                  # 必填：排序号
      stateLevel: 10                  # 主按钮（蓝色）
      defaultOpinion: "同意"          # 可选：默认意见，点击按钮自动填入
      controlVisiableMethodGuid: null # 可选：控制按钮显隐的外部方法
    # 退回按钮（stateLevel=30 危险红色 / 排序 1）
    - operationguid: <UUID-退回>
      activityguid: <UUID-审批>
      processversionguid: <版本UUID>
      operationname: "退回"
      operationtype: 30
      is_requireopinion: 10
      is_checkmaterialsubmit: 20
      is_showoperationpage: 1
      operationvisiablecase: '1020'
      is_shownextactivity: 0
      backtargetscope: '1'
      targetactivity: '[#=AllBeforeActivity#]'
      multiTransctorMode: OR
      is_ShowOpinionTemplete: 0
      ordernumber: 1
      stateLevel: 30                  # 危险按钮（红色）
```

### 退回按钮专属字段

| 字段 | 含义 | 取值 |
|------|------|------|
| `targetactivity` | 退回目标 | `[#=AllBeforeActivity#]`（所有前序） |
| `backtargetscope` | 退回范围 | `'1'`（仅提交人）、`'2'`（所有处理人） |
| `multiTransctorMode` | 退回时多人处理模式 | `OR`、`AND`、`OrAndRead` |
| `operationvisiablecase` | 显示场合 | `'10'`正常、`'20'`退回时、`'30'`已办时，可拼接如 `'1020'` |

### 自定义流程设计按钮（`operationtype=80`）

自定义流程需要在允许用户调整后续流转的普通人工节点上增加“流程设计”按钮，同时 `workflowProcess.isnewversion=10`、`customtype=0`。

```yaml
- operationguid: <UUID-流程设计按钮>
  activityguid: <UUID-当前活动>
  processversionguid: <版本UUID>
  operationname: "流程设计"
  operationtype: 80
  is_requireopinion: 20
  is_checkmaterialsubmit: 20
  is_showoperationpage: 1
  operationvisiablecase: '10'
  is_shownextactivity: 0
  is_ShowOpinionTemplete: 0
  ordernumber: 2
  stateLevel: 10
  operationpageurl: '{"url":"frame/pages/epointworkflow/client/flowsetting?processVersionInstanceGuid=[#=ProcessVersionInstance_ProcessVersionInstanceGuid#]&workItemGuid=[#=WorkItem_WorkItemGuid#]&processVersionGuid=[#=ProcessVersion_ProcessVersionGuid#]","dialog":{"fullscreen": true}}'
```

### 自由流程按钮红线

自由流程（`workflowProcess.isnewversion=20`）中不允许配置退回按钮（`operationtype=30`）。需要反向处理时，用普通通过按钮配置反向流转。

## workflowConfig（退回按钮配置，必须与退回按钮一一对应）

```yaml
workflowConfig:
  - rowguid: <UUID>
    processversionguid: <版本UUID>
    belongto: 22                     # 22 表示按钮
    configname: backTargetScope
    configvalue: '1'                 # '1' 仅提交人；'2' 所有处理人
    sourceguid: <对应退回按钮的 operationguid>
```

> ⚠️ 校验脚本会**严格校验**退回按钮 ↔ workflowConfig 一一对应，缺一报错。

## workflowActivityMaterial（活动材料）

> ⚠️ **待补充**：每个普通节点关联 ≥0 个流程版本材料，用于控制各节点可操作的表单材料。

## workflowActivityFieldAccess（活动材料权限）

> ⚠️ **待补充**：每个活动材料关联 ≥0 条字段权限配置，用于控制各节点对表单字段的读写权限。
