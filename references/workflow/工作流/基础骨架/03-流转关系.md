# 流转关系（transition）

> 本文档定义节点间流转关系（transition）及 工作流扩展的流转条件字段。

## transition（节点流转）

```yaml
transition:
  - transitionguid: <UUID>
    processversionguid: <版本UUID>
    fromactivityguid: <源节点 activityguid>
    toactivityguid: <目标节点 activityguid>
    transitionname: "申请"            # 业务语义命名
    transitiondispname: "申请"
    is_sendtomessagecenter: 10       # 10:生成待办；20:不生成
    priority: 60                     # 100/80/60/40/20
    type: 10                         # 默认 10
    targetActivityTransactorSource: 10  # 10:用目标活动设置；20:用变迁定义；30:通过方法获取
    is_targettransactor_editable: 10   # 10:允许；15:可删不可加；20:不允许
    isDefault: 20                    # 20:不是源活动默认变迁
    is_showasoperationbutton: 20
    vmlid: 2                         # 必填，≥2 不重复
```

> ⚠️ vmlid 必填，从 2 开始递增；transition 必须形成开始→申请→审批→...→结束的闭合链路。

### transition 扩展字段

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `selecttransctormethodguid`  | 处理人来源为外部方法时的方法标识 | 外部方法 GUID（`targetActivityTransactorSource=30` 时必填） | 否 |
| `transactorallowmaxcount`  | 处理人数量限制 | ≥0，`null` 表示不限制 | 否 |
| `type` 值新增 | 变迁类型 | `10`（变迁，可见）、`20`（跳转，不可见的变迁） | 是 |

> ⚠️ 工作流字段命名规范化：`isTargetTransactorEditable` → `is_targettransactor_editable`，`isShowAsOperationButton` → `is_showasoperationbutton`。

## 流转条件（workflowTransitionCondition）

> **工作流扩展** — 本节内容由原 `工作流扩展节点/04-流转条件.md` 合并而来。流转条件写在 `transition` 子节点中，与上方 `transition` 字段表形成绑定关系：每条 transition 可关联 ≥0 个流转条件。

### 条件表达式类型

支持三种条件表达式类型：

| `conditionExpressionType` | 含义 | 关键字段 |
|--------------------------|------|----------|
| `10` | 单一表达式 | `leftValue`（如 `[#=jine#]`）+ `compareOperation`（如 `>`）+ `rightValue`（如 `6000`）+ `valueType` |
| `20` | 复杂表达式 | `conditionExpression`（如 `[1] AND [2]`），由多个单一条件组合 |
| `30` | 外部方法 | `methodGuid`（外部方法标识） |

### 字段一览

| 字段 | 含义 | 类型 | 必填 | 字段来源 |
|------|------|------|------|----------|
| `conditionGuid` | 条件唯一标识 | UUID | 是 | 工作流扩展 |
| `conditionName` | 条件名称 | 字符串 | 是 | 工作流扩展 |
| `transitionGuid` | 所属变迁标识 | UUID（= transition.transitionguid） | 是 | 工作流扩展 |
| `conditionExpressionType` | 表达式类型 | `10`/`20`/`30` | 是 | 工作流扩展 |
| `leftValue` | 左值 | 字符串，如 `[#=jine#]` | 否（type=10 时填） | 工作流扩展 |
| `leftText` | 左值文本 | 字符串 | 否 | 工作流扩展 |
| `compareOperation` | 比较操作 | `<`/`<=`/`>`/`>=`/`=`/`!=`/`in`/`notin` | 否（type=10 时填） | 工作流扩展 |
| `rightValue` | 右值 | 字符串 | 否（type=10 时填） | 工作流扩展 |
| `rightText` | 右值文本 | 字符串 | 否 | 工作流扩展 |
| `valueType` | 值类型 | `10`（字符）、`20`（整形）、`30`（数字）、`40`（日期） | 否（type=10 时填） | 工作流扩展 |
| `conditionExpression` | 复杂表达式 | 字符串，如 `[1] AND [2]` | 否（type=20 时填） | 工作流扩展 |
| `methodGuid` | 外部方法标识 | UUID | 否（type=30 时填） | 工作流扩展 |
| `ruleGuid` | 规则标识 | UUID（动作流标识） | 否 | 工作流扩展 |
| `processversionguid` | 版本标识 | UUID | 是 | 工作流扩展 |
| `orderNum` | 排序号 | ≥0 | 是 | 工作流扩展 |
| `remark` | 备注 | 字符串 | 否 | 工作流扩展 |

### 示例（条件分支：金额 > 6000 才走审批）

```yaml
transition:
  - transitionguid: <UUID>
    fromactivityguid: <UUID-申请>
    toactivityguid: <UUID-审批>
    transitionname: "审批"
    processversionguid: <版本UUID>
    vmlid: 3
    workflowTransitionCondition:
      - conditionGuid: <UUID-条件>
        conditionName: "金额大于6000"
        transitionGuid: <UUID>       # 同所属 transition
        conditionExpressionType: 10  # 单一表达式
        leftValue: '[#=jine#]'
        leftText: "金额"
        compareOperation: ">"
        rightValue: '6000'
        rightText: '6000'
        valueType: 30                # 数字类型
        processversionguid: <版本UUID>
        orderNum: 0
```

> ⚠️ 流转条件中的 `[#=jine#]` 参数必须先在 `workflowContext` 中定义。详见 [04-表单材料与数据表.md](./04-表单材料与数据表.md) 的「相关数据(workflowContext)」小节。
