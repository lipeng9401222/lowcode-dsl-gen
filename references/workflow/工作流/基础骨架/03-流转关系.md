# 流转关系（transition）

> 本文档定义节点间流转关系（transition）及 工作流扩展的流转条件字段。

## transition（节点流转）

```yaml
transition:
  - transitionguid: <UUID>
    processversionguid: <版本UUID>
    fromactivityguid: <源节点 activityguid>
    toactivityguid: <目标节点 activityguid>
    transitionname: "申请"            # 业务语义命名
    transitiondispname: "申请"
    is_sendtomessagecenter: 10       # 10:生成待办；20:不生成
    priority: 60                     # 100/80/60/40/20
    type: 10                         # 默认 10
    targetactivitytransactorsource: 10  # 10:用目标活动设置；20:用变迁定义；30:通过方法获取
    is_targettransactor_editable: 10   # 10:允许；15:可删不可加；20:不允许
    is_default: 20                    # 20:不是源活动默认变迁
    is_showasoperationbutton: 20
    vmlid: 2                         # 必填，≥2 不重复
```

> ⚠️ vmlid 必填，从 2 开始递增；transition 必须形成开始→申请→审批→...→结束的闭合链路。条件可选分支允许出现 `A → B → C` 且同时存在 `A → C`，用于表达“满足条件走 B，不满足直接到 C”。

### transition 扩展字段

| 字段 | 含义 | 取值 | 必填 |
|------|------|------|------|
| `selecttransctormethodguid`  | 处理人来源为外部方法时的方法标识 | 外部方法 GUID（`targetactivitytransactorsource=30` 时必填） | 否 |
| `transactorallowmaxcount`  | 处理人数量限制 | ≥0，`null` 表示不限制 | 否 |
| `type` 值新增 | 变迁类型 | `10`（变迁，可见）、`20`（跳转，不可见的变迁） | 是 |

> ⚠️ 工作流字段命名规范化：`isTargetTransactorEditable` → `is_targettransactor_editable`，`isShowAsOperationButton` → `is_showasoperationbutton`。

### 条件分支生成规则

用 `scripts/add_workflow.py` 生成条件分支时，优先在 `--conditions-json` 中写 `fromActivity` / `toActivity`。若该源目标之间没有 transition，脚本会自动补一条变迁并挂载条件。

“否则 / 不满足条件”必须生成显式互补条件，不用 `is_default=10` 代替。例如“需求来源为内部优化走总负责人评审，否则直接指定人员执行”应生成两条从 `申请` 出发的条件线：

```json
[
  {
    "fromActivity": "申请",
    "toActivity": "总负责人评审",
    "transitionname": "总负责人评审",
    "conditionname": "需求来源为内部优化",
    "leftvalue": "[#=demandsource#]",
    "lefttext": "需求来源",
    "compareoperation": "=",
    "rightvalue": "4",
    "righttext": "内部优化",
    "valuetype": 10
  },
  {
    "fromActivity": "申请",
    "toActivity": "指定人员执行",
    "transitionname": "直接执行",
    "conditionname": "需求来源非内部优化",
    "leftvalue": "[#=demandsource#]",
    "lefttext": "需求来源",
    "compareoperation": "!=",
    "rightvalue": "4",
    "righttext": "非内部优化",
    "valuetype": 10
  }
]
```

布局上，`A → B → C` 且同时存在 `A → C` 时，`B` 识别为可选分支节点，坐标放到分支泳道 `icony='20'`；`A`、`C` 仍在主线 `icony='180'`。

## 流转条件（workflowTransitionCondition）

> **工作流扩展** — 本节内容由原 `工作流扩展节点/04-流转条件.md` 合并而来。流转条件写在 `transition` 子节点中，与上方 `transition` 字段表形成绑定关系：每条 transition 可关联 ≥0 个流转条件。

### 条件表达式类型

支持三种条件表达式类型：

| `conditionexpressiontype` | 含义 | 关键字段 |
|--------------------------|------|----------|
| `10` | 单一表达式 | `leftvalue`（如 `[#=jine#]`）+ `compareoperation`（如 `>`）+ `rightvalue`（如 `6000`）+ `valuetype` |
| `20` | 复杂表达式 | `conditionexpression`（如 `[1] AND [2]`），由多个单一条件组合 |
| `30` | 外部方法 / 动作流规则 | `methodguid` 或 `ruleguid` |

强校验规则：

- `conditionexpressiontype=10` 必须包含 `leftvalue`、`compareoperation`、`rightvalue`、`valuetype`。
- `conditionexpressiontype=20` 必须包含 `conditionexpression`，且 `[1]`、`[2]` 等序号只能引用同一条 transition 下的 type=10 条件。序号按 type=10 条件的 `ordernum` 升序、再按列表顺序计算。
- `conditionexpressiontype=30` 必须包含 `methodguid` 或 `ruleguid`。

### 字段一览

| 字段 | 含义 | 类型 | 必填 | 字段来源 |
|------|------|------|------|----------|
| `conditionguid` | 条件唯一标识 | UUID | 是 | 工作流扩展 |
| `conditionname` | 条件名称 | 字符串 | 是 | 工作流扩展 |
| `transitionguid` | 所属变迁标识 | UUID（= transition.transitionguid） | 是 | 工作流扩展 |
| `conditionexpressiontype` | 表达式类型 | `10`/`20`/`30` | 是 | 工作流扩展 |
| `leftvalue` | 左值 | 字符串，如 `[#=jine#]` | 否（type=10 时填） | 工作流扩展 |
| `lefttext` | 左值文本 | 字符串 | 否 | 工作流扩展 |
| `compareoperation` | 比较操作 | `<`/`<=`/`>`/`>=`/`=`/`!=`/`in`/`notin` | 否（type=10 时填） | 工作流扩展 |
| `rightvalue` | 右值 | 字符串 | 否（type=10 时填） | 工作流扩展 |
| `righttext` | 右值文本 | 字符串 | 否 | 工作流扩展 |
| `valuetype` | 值类型 | `10`（字符）、`20`（整形）、`30`（数字）、`40`（日期） | 否（type=10 时填） | 工作流扩展 |
| `conditionexpression` | 复杂表达式 | 字符串，如 `[1] AND [2]` | 否（type=20 时填） | 工作流扩展 |
| `methodguid` | 外部方法标识 | UUID | 否（type=30 时填） | 工作流扩展 |
| `ruleguid` | 规则标识 | UUID（动作流标识） | 否 | 工作流扩展 |
| `processversionguid` | 版本标识 | UUID | 是 | 工作流扩展 |
| `ordernum` | 排序号 | ≥0 | 是 | 工作流扩展 |
| `remark` | 备注 | 字符串 | 否 | 工作流扩展 |

### 示例（条件分支：金额 > 6000 才走审批）

```yaml
transition:
  - transitionguid: <UUID>
    fromactivityguid: <UUID-申请>
    toactivityguid: <UUID-审批>
    transitionname: "审批"
    processversionguid: <版本UUID>
    vmlid: 3
    workflowTransitionCondition:
      - conditionguid: <UUID-条件>
        conditionname: "金额大于6000"
        transitionguid: <UUID>       # 同所属 transition
        conditionexpressiontype: 10  # 单一表达式
        leftvalue: '[#=jine#]'
        lefttext: "金额"
        compareoperation: ">"
        rightvalue: '6000'
        righttext: '6000'
        valuetype: 30                # 数字类型
        processversionguid: <版本UUID>
        ordernum: 0
```

> ⚠️ 流转条件中的 `[#=jine#]` 参数必须先在 `workflowContext` 中定义。详见 [04-表单材料与数据表.md](./04-表单材料与数据表.md) 的「相关数据(workflowContext)」小节。
> ⚠️ 条件表达式引用的是 `workflowContext.fieldname`，不是中文标签。中文名称写 `lefttext` / `note`；不要生成 `contextname` 之类字段。

### 示例（复杂条件：年龄 > 20 且性别为男）

```yaml
workflowTransitionCondition:
  - conditionguid: <UUID-条件1>
    conditionname: "年龄大于20"
    transitionguid: <UUID>
    conditionexpressiontype: 10
    leftvalue: '[#=age#]'
    lefttext: "年龄"
    compareoperation: ">"
    rightvalue: '20'
    righttext: '20'
    valuetype: 30
    processversionguid: <版本UUID>
    ordernum: 0
  - conditionguid: <UUID-条件2>
    conditionname: "性别是男"
    transitionguid: <UUID>
    conditionexpressiontype: 10
    leftvalue: '[#=gender#]'
    lefttext: "性别"
    compareoperation: "="
    rightvalue: 'male'
    righttext: '男'
    valuetype: 10
    processversionguid: <版本UUID>
    ordernum: 1
  - conditionguid: <UUID-条件3>
    conditionname: "年龄大于20且性别男"
    transitionguid: <UUID>
    conditionexpressiontype: 20
    conditionexpression: "[1] AND [2]"
    processversionguid: <版本UUID>
    ordernum: 2
```
